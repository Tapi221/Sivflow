## 🎯 このドキュメントの使い方

### 新人の方へ
1. まず [`07_Guides/新人向けクイックスタート.md`](07_Guides/新人向けクイックスタート.md) を読んでください
2. 環境構築後、[`07_Guides/機能開発フロー.md`](07_Guides/機能開発フロー.md) を確認してください
3. 困ったら [`07_Guides/トラブルシューティング.md`](07_Guides/トラブルシューティング.md) を見てください

### 機能追加・修正の方へ
1. [`02_Features/`](02_Features/) で関連する機能仕様を確認
2. [`03_Logic/`](03_Logic/) でビジネスロジックを確認
3. [`07_Guides/機能開発フロー.md`](07_Guides/機能開発フロー.md) に沿って実装

### トラブル発生時
1. [`07_Guides/トラブルシューティング.md`](07_Guides/トラブルシューティング.md) で該当する症状を検索
2. 解決しない場合は [`07_Guides/デバッグガイド.md`](07_Guides/デバッグガイド.md) を参照
3. それでも解決しない場合はチームに相談

## 📝 ドキュメント更新ルール

### いつ更新するか
- 新機能を追加したとき
- 既存機能を変更したとき
- バグ修正で仕様が変わったとき

### どこを更新するか
1. **機能仕様書** (`02_Features/`) - UI/UXの変更
2. **ロジック仕様書** (`03_Logic/`) - アルゴリズムの変更
3. **データベース設計書** (`01_Database/`) - スキーマの変更
4. **完了レポート** (`05_Plans/`) - 実装完了時

### 更新の優先度
🔴 **必須**: 仕様書 (02_Features, 03_Logic, 01_Database)  
🟡 **推奨**: 完了レポート (05_Plans)  
🟢 **任意**: コメント、README
