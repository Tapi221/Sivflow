# システムアーキテクチャ概要

## 1. コンセプト
「Flashcard Master」は **Offline-First PWA** です。
ユーザーの操作はすべてローカル（IndexedDB）に対して即座に行われ、ネットワーク接続時にバックグラウンドでクラウド（Firebase）と同期されます。これにより、通信環境に依存しない高速なレスポンスを実現しています。

## 3. 主要モジュールの役割

### A. データ永続化層
- **LocalDB (`src/services/localDB.ts`)**:
  - アプリケーションの正本データ（Single Source of Truth）。
  - `cards`, `folders`, `studyLogs`, `levelHistories` などのテーブルを管理。
- **Firestore (`src/services/firebase.ts`)**:
  - バックアップおよびマルチデバイス同期用の中央リポジトリ。
  - セキュリティルールにより、`userId` に基づくアクセス制御を強制。

### B. 同期システム (`src/services/syncService.ts`)
- **双方向同期**:
  - **Push**: ローカルでの変更（`updatedAt` 更新）を検知し、クラウドへ送信（楽観的ロック併用）。
  - **Pull**: クラウド上の新しい変更を取得し、ローカルへ反映（スマートマージ）。
- **競合解決**:
  - フィールド単位の自動マージを行い、修復不能な文字レベルの競合が発生した場合は `hasSyncConflict: true` を立ててカード単位で警告。
  - 基本原則は LWW (Last Write Wins) だが、画像やタグは結合 (Union) して保持。

### C. 復習アルゴリズム (`src/services/reviewAlgorithm.ts`)
- **責務の分離**:
  - **Scheduling**: `memoryStability` を更新し、次回の復習日を決定。
  - **Metrics (`src/utils/reviewMetrics.ts`)**: UI表示用の「定着度 (Retention)」と「耐性スコア (Resistance)」の計算のみを担当。

### D. アセット管理 (`src/Components/card/MediaUploader.tsx`)
- **アップロード処理**:
  - クリップボードからの貼り付けやD&Dに対応。
  - **HEIC変換**: iOS写真を自動的にJPEGに変換してアップロード。
  - **Optimistic UI**: 画像のアップロード完了を待たずに、一時的なBlob URLで即座にカードを表示。

## 4. 技術スタック詳細

### フロントエンド
- **Framework**: React 19
- **Build**: Vite + TypeScript
- **State**: @tanstack/react-query
- **Routing**: react-router-dom
- **UI**: Tailwind CSS / Radix UI / Lucide React

### バックエンド (Firebase)
- **Auth**: Google Sign-In
- **Firestore**: NoSQL Database
- **Storage**: Object Storage (画像/音声)
- **Hosting**: CDN配信

---

## 5. セキュリティモデル
- **Authentication Context**: 全てのデータアクセスは `userId` に紐付きます。
- **Firestore Rules**: 
  - `match /cards/{cardId}` -> `allow read, write: if request.auth.uid == resource.data.userId;`
  - 他人のデータへのアクセスは厳格に遮断されます。
