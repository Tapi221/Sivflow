# プロジェクト用語集 (Glossary)

## 1. コア概念 (Core Concepts)

| 日本語 | 英語 (Internal) | 説明 |
| :--- | :--- | :--- |
| **カード** | `Card` | 学習の最小単位。表面（問題）と裏面（解答）を持つ。 |
| **フォルダ** | `Folder` | カードをまとめるコンテナ。ファイルシステムのような階層構造を持つ。他のアプリでいう「デッキ (Deck)」。 |
| **復習** | `Review` | 既存のカードをテストすること。新規学習 (`Learn`) と区別する場合もあるが、広義にはすべて `Review`。 |
| **定着度** | `Retention` | 「現在、記憶に残っている確率」。UI上のメイン指標。信号機カラーで表現される。 |
| **ストリーク** | `Streak` | 連続学習日数。1日でもサボると0になる（が、厳密には「0時切り替え」ではなく猶予を持たせる場合がある）。 |

## 2. アルゴリズム・数値 (Metrics)

| 用語 | 変数名 | 定義 |
| :--- | :--- | :--- |
| **安定度** | `Stability (S)` | 記憶が保持される期間（日数）。アルゴリズム内部の値であり、ユーザーには直接見せない（または「耐性スコア」として抽象化する）。 |
| **難易度** | `Difficulty (D)` | カードごとの固有の難しさ。初期設定や評価によって変動する。 |
| **耐性スコア** | `Resistance Score` | Stabilityを対数変換し、0-100+のスコアにしたもの。「積み上げ」を可視化する指標。 |
| **王冠** | `Crown` | 定着度が極めて高い（例: 85%以上）状態。マスタリーの証。 |

## 3. システム・インフラ (System)

| 用語 | 英語 | 説明 |
| :--- | :--- | :--- |
| **Source of Truth** | - | データの正本。本アプリでは常に **「ローカル端末 (LocalDB)」** が正本であり、サーバーはコピーである。 |
| **コンフリクト** | `Conflict` | 複数の端末で同時に異なる変更が行われた状態。原則 `Last Write Wins` だが、文章系は `Manual Merge`。 |
| **下書き** | `Draft` | 作成中だが、まだ復習スケジュールに乗っていない状態。 |
| **はてな** | `Uncertainty` | カードに付ける「？」フラグ。正解しても自信がない場合に付ける。 |

## 4. 開発用語 (Dev)

| 用語 | 説明 |
| :--- | :--- |
| **Vendor Chunk** | `node_modules` 内のライブラリをまとめたJSファイル。初期ロード高速化のために分割される。 |
| **Optimistic Update** | サーバーレスポンスを待たずに、UIを先に更新すること。「サクサク感」の源泉。 |
| **Graceful Degradation** | オフラインやエラー時でも、アプリをクラッシュさせずに機能を制限して動かし続けること。 |
