## アーキテクチャ

### データ保存方針（ハイブリッド設計）

本アプリは、データの性質に応じて保存先を分離する。

#### ユーザークラウドストレージに保存
- フォルダ構造
- 単語カードコンテンツ
- 学習ログ（studyLogs, trainingLogs, levelHistory）
- ごみ箱スナップショット

形式: JSON / バイナリファイル（アプリ専用フォルダ配下）

#### Firebase（Firestore）に保存
- 認証情報（users）
- ユーザー設定（userSettings）
- 統計データ（userStats）
- 共有関連（shareLinks, sharedFolders）
- ログイン履歴（loginHistory）

## 設計原則

### 統計処理方針

- **クライアントサイドで統計計算を行わない**
- 学習完了時にサマリー（正解数・不正解数等）を Cloud Functions に送信
- Cloud Functions が userStats をインクリメンタル更新
- 統計画面は userStats の read のみで構成（最大1〜2回の read）

### セキュリティ原則

- 未認証ユーザーは一切のデータにアクセス不可
- すべてのデータは userId に完全に紐づく
- Firestore セキュリティルールでアクセス制御
- HTTPS 通信必須

### パフォーマンス原則

- Firestore read 回数を最小化
- 統計画面は事前集計済みデータを使用
- コード分割（React.lazy）で初期読み込み時間短縮
- 画像の遅延読み込み

## パフォーマンス目標

- ページ読み込み時間: 2秒以内（3G環境想定）
- フォルダツリー展開: 200ms以内
- 画像アップロード: 1MB ファイルで 3秒以内
- 検索・フィルタリング: 500ms以内
- Firestore read 回数（統計画面）: 最大1〜2回

## セキュリティ要件

### Firestore セキュリティルール
- すべてのコレクションで userId チェック必須
- 未認証ユーザーは読み取り不可
- 他ユーザーのデータはアクセス不可

### 認証要件
- メール認証必須
- パスワードは Firebase が暗号化管理
- OAuth は HTTPS 必須

### 通信セキュリティ
- HTTPS 通信必須
- CORS ポリシー適切に設定
