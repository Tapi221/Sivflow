## 2. ドキュメント層 (`docs/`)

役割: 設計仕様、運用手順、実装メモ、技術的な検討記録を格納します。サブディレクトリでカテゴリ分けされています（例: 00_Meta, 01_ImplementationPlans, 03_Specifications, 06_Operations など）。

- 代表的ファイル・用途:
  - `API_Documentation.md`: 公開・内部API の仕様
  - `GEMINI.md` / `KATEX.md`: 外部ツールやライブラリ利用のガイド
  - `implementation_plan_*.md`: 大きな実装変更の計画書
  - 監査・運用系ドキュメント（障害対応、マイグレーション手順、ローカルの安全な運用方法）

注意: ドキュメントは実装と同期して古くなるため、主要な変更時に更新してください。

## 3. フロントエンド層 (`src/`)

概要: React + TypeScript + Vite を用いたクライアントアプリのソースコード。機能別にディレクトリが分かれています。

- `src/App.tsx`, `src/main.tsx`, `src/index.css` 等: アプリのエントリとグローバルスタイル

- `src/services/` — ビジネスロジックとアプリケーションコア
  - 役割: 同期（Sync）、ローカルDB（Dexie）やバックグラウンド処理、スナップショット/バックアップ、テレメトリの集約
  - 代表的モジュール:
    - `SyncServiceV2.ts`: 同期アルゴリズム（差分取得・マージ・競合解決の実装）
    - `operationQueue.ts`: オフラインで発生した操作のキューイングと再送制御
    - `localDB.ts`: IndexedDB（Dexie）スキーマとマイグレーション
    - `telemetry.ts`: エラーログやパフォーマンス指標の収集

## 4. サーバーサイド／Cloud Functions (`functions/`)

概要: Firebase Cloud Functions（または Node ベースのサーバー処理）の実装。`functions/src/` に TypeScript ソースがあり、ビルド済み成果物は `lib/` や `index.js` に配置されます。

- 代表的構成:
  - `functions/package.json`, `functions/tsconfig.json` — 関数単位の依存管理とコンパイル設定
  - `functions/src/index.ts`: Cloud Functions のエントリ（Callable 関数やトリガーのエクスポート）
  - `functions/src/triggers/`: `onUserCreate.ts`, `onSyncRequest.ts` などの Firestore トリガー
  - `functions/src/services/`: サーバー側の DB クライアント、外部 API 連携、認証検証ロジック

運用上の注意:
  - Cloud Functions の環境変数やシークレットは Git に入れない（`serviceAccountKey.json` は取り扱いに注意）。

## 5. スクリプト (`scripts/`)

- `dist_final_check/`, `dist_test/`: ビルド結果のスナップショット。CIや手動検査で利用する。

### ディレクトリ詳細: `src/Components/`

機能ドメインごとに分割されています。

- `ui/`: 汎用 UI コンポーネント（Button, Dialog, Checkbox 等）。ロジックを持たず、純粋な表示を担当。
- `card/`: カードエディタ、ブロックエディタ、カード表示コンポーネント。
- `folder/`: フォルダツリー (`FolderTree`), フォルダダイアログ, カラムビュー (`ColumnNavigator`) 等。
- `explorer/`: サイドバー拡張機能（タブ切り替え、Recent, Favorites, `TagFilter`）。
- `tag/`: タグ管理 (`TagManagerDialog`)、タグ入力コンポーネント。
- `settings/`: 設定ダイアログ、ショートカット一覧、データ管理画面。
- `stats/`: 統計グラフ (`Recharts` ラッパー)、学習ヒートマップ。
- `services/` (非推奨): 一部の古いサービスロジックが残存している場合がありますが、基本は `src/services/` (ルート直下) を参照。

### ディレクトリ詳細: `src/hooks/`

ビジネスロジック、状態管理、データアクセスをカプセル化したカスタムフック群。

- **データアクセス**:
  - `useCards.ts`, `useFolders.ts`, `useTags.ts`, `useAllDescendantCards.ts`
- **UI状態管理**:
  - `useExplorerStore.ts`: サイドバーの状態（幅、タブ、開閉、フィルタ）を一元管理。
  - `useFolderDnD.ts`: フォルダのドラッグ＆ドロップロジック。
  - `useCommandPalette.tsx`: コマンドパレットの制御。
- **機能**:
  - `useKeyboardShortcuts.ts`: キーボードショートカットの登録と管理。
  - `useSync.ts`, `useSyncSettings.ts`: 同期設定と状態のフック。

## 詳細目録: `src/services/`（主要ファイルと短い説明）

- `SyncServiceV2.ts` — 同期オーケストレーター。差分取得（Pull）→ 適用 → ローカル変更の Push を制御し、フルリシンクや競合検出・記録を行います。実装は Telemetry/NetworkMonitor/QueueManager/CloudAdapter に依存しています。 [src/services/SyncServiceV2.ts](src/services/SyncServiceV2.ts#L1)
- `operationQueue.ts` — ローカルの操作キュー（圧縮ルール、優先度、再試行・DLQ の扱い）。バッチ処理と並列処理の制御ロジックを含みます。 [src/services/operationQueue.ts](src/services/operationQueue.ts#L1)
- `localDB.ts` — `Dexie` を使った IndexedDB 層。テーブル定義、データ正規化/非正規化、移行・インポート・フォレンジック用ユーティリティを含む大型モジュールです。 [src/services/localDB.ts](src/services/localDB.ts#L1)
- `SnapshotService.ts` — アプリ全体のスナップショット作成・比較・エクスポート機能（世代管理、Firestore への保存、部分エクスポート）。 [src/services/SnapshotService.ts](src/services/SnapshotService.ts#L1)
- `reviewAlgorithm.ts` — 復習・間隔調整ロジック（アルゴリズム本体）。学習間隔や復習タイミング計算に関連する関数群。 [src/services/reviewAlgorithm.ts](src/services/reviewAlgorithm.ts#L1)
- `telemetry.ts` / `ProductionLogger.ts` — テレメトリ収集・トランザクション計測・ログ出力の実装。運用観測の基盤。 [src/services/ProductionLogger.ts](src/services/ProductionLogger.ts#L1)
- `SnapshotService.ts`, `IndexedDBRebuildOrchestrator.ts`, `IndexedDBMetadataService.ts` — データ整合性・再構築やメタ情報管理に関わる補助モジュール。

（注）`src/services/` はアプリケーションのコアロジックが集まるため、ここを変更する際はユニットテストと `localDB` のマイグレーション手順を必ず更新してください。

## 詳細目録: `functions/`（主要トリガーとロジック）

- `functions/src/index.ts` — Functions のエントリポイント。現在はセキュリティ関連トリガーをエクスポートします。 [functions/src/index.ts](functions/src/index.ts#L1)
- `functions/src/security/index.ts` — Firestore の `users/{userId}/securityLogs/{logId}` 作成トリガー。異常検知ルールに基づくリスクスコア計算、通知作成、デバイスの revoke、アカウントロック等の自動対処ロジックを実装しています。 [functions/src/security/index.ts](functions/src/security/index.ts#L1)

運用注意:
- `functions/` 配下の `serviceAccountKey.json` の扱いに注意。CI/デプロイでは Secrets 管理を使ってください。

## 詳細目録: `scripts/`（実行スクリプトと簡易手順）

- `card_simulation.ts` — 復習アルゴリズムやカードの状態遷移をシミュレーションする TypeScript スクリプト。ローカルで `ts-node` で実行して挙動確認が可能です。 [scripts/card_simulation.ts](scripts/card_simulation.ts#L1)
- `card_data_patch.ts` — データ補正/パッチ用スクリプト（既存データのバルク変換や修正に使用）。 [scripts/card_data_patch.ts](scripts/card_data_patch.ts#L1)
- `migrateBase64.ts` — Base64 形式からの移行処理など、データマイグレーション用ユーティリティ。 [scripts/migrateBase64.ts](scripts/migrateBase64.ts#L1)
- `verify_migration.js` / `verify_security_signal.ts` — マイグレーションやセキュリティシグナルの検証用。CI／ローカルでの検証に使います。 [scripts/verify_migration.js](scripts/verify_migration.js#L1)

実行例（ローカル）:

```powershell
npx ts-node scripts/card_simulation.ts
node scripts/verify_migration.js
```

（注）スクリプトにより DB を書き換えるものがあるため、実行前にバックアップ（スナップショット）を取得してください。
