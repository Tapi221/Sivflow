# ディレクトリ構成図 (超詳細版・マスターマップ)

本ドキュメントは、プロジェクト配下の全ディレクトリおよび主要なファイルの役割を階層別に網羅したものです。

---

## 📁 1. プロジェクトルート (Root)

| ファイル/ディレクトリ | 役割・技術スタック |
|---|---|
| `.github/workflows/` | CI/CD (GitHub Actions) 自動ビルド・デプロイ設定 |
| `.agent/` | AI Agent専用ワークフロー(`workflows/`)、一時作業領域(`scratchpad/`) |
| `docs/` | **[詳細後述]** ドキュメント管理 |
| `functions/` | **[詳細後述]** サーバーサイド (Firebase Cloud Functions) |
| `src/` | **[詳細後述]** フロントエンド (React + Vite + Typescript) |
| `tests/` | **[詳細後述]** テストスイート |
| `public/` | `manifest.json`, `sw.js` (PWA用), `favicon`, `sounds/` (通知音) |
| `scripts/` | メンテナンス・デプロイ・データ移行用スクリプト群 |
| `firebase.json` | Firebase サービスの中心的な構成設定 |
| `firestore.rules` | データベースセキュリティルール |
| `vite.config.ts` | ビルド設定、エイリアス設定、プラグイン構成 |

## 📁 2. ドキュメント層 (`docs/`)

プロジェクトの意思決定と仕様が詰まった階層です。

### 📌 03_Specifications (技術仕様)
- `同期システム仕様書.md`: 同期エンジン (V2) の整合性アルゴリズム。
- `マップ機能_技術仕様書.md`: d3-force を用いた関係性可視化の設計。
- `ルーティング定義書.md`: `react-router-dom` のパスと権限管理。
- `画面遷移図.md`: Mermaidによるアプリケーションフロー。

### 📌 06_Operations (運用管理)
- `Phase2運用ガイド.md`: 本番環境での監視・運用フロー。
- `トラブルシューティングとデータ復旧.md`: 障害発生時の復旧プロトコル。
- `ローカルDBアーキテクチャ監査レポート.md`: Dexie/IndexedDB の健全性評価。
- `開発環境での安全な運用手順.md`: ローカル開発でのデータ破壊防止策。

## 📁 3. フロントエンド層 (`src/`)

### 📌 `src/services/` (ビジネスロジック・コア)
- `SyncServiceV2.ts`: クラウド/ローカルの全同期をオーケストレーション。
- `operationQueue.ts`: オフライン操作をキューイングし、衝突を解消(Matrix)。
- `localDB.ts`: `Dexie.js` によるデータベーススキーマ・マイグレーション定義。
- `AppInitializer.ts`: 起動時のヘルスチェック、マイグレーション、初期同期。
- `SnapshotService.ts`: データのバックアップ/リストア、履歴管理。
- `DataIntegrityService.ts`: IndexedDB の破損検知と自己修復ロジック。

### 📌 `src/hooks/` (データ・ロジック連携)
- `useSync.ts`: 同期状態の監視と手動同期実行。
- `useCards.ts` / `useFolders.ts`: CRUD 操作の抽象化とキャッシュ管理。
- `useGraphSimulation.ts`: マップ機能の物理演算 (d3-force) をカプセル化。
- `useProjectMap.ts`: マップの座標保存・読み込み。
- `useNetworkStatus.ts`: オフライン/オンライン切り替えのリアルタイム検知。

### 📌 `src/Components/` (UIモジュール)
各サブディレクトリ内の代表的なファイルです。

- **`map/`**:
  - `WorldMap.tsx`: 全体俯瞰地図、ズーム/パン対応。
  - `LocalMap.tsx`: 特定カード中心の相関図。
  - `HandTray.tsx`: マップ上に配置するカードのドラッグ元トレイ。
- **`card/`**:
  - `Flashcard.tsx`: カードの基本表示（表面・裏面）。
  - `CardEditor.tsx`: 記述式・画像等を含む高度なエディタ。
  - `MediaUploader.tsx`: Firebase Storage への画像/音声アップロード。
- **`study/`**:
  - `StudyMode.tsx`: 学習セッションのメインループ。
  - `StampRally.jsx`: 学習達成時の演出。
- **`sync/`**:
  - `SyncStatusIndicator.tsx`: 右上の雲アイコン（状態表示）。
  - `ConflictResolutionDialog.tsx`: 競合発生時のユーザー選択UI。
- **`ui/`**: 共通の `button`, `dialog`, `dropdown`, `input` 等（Atomic設計）。

### 📌 `src/utils/` (汎用ユーティリティ)
- `queueUtils.ts`: カード操作を透過的にキュー処理に変える Facade。
- `imageUtils.ts`: 画像のリサイズ、圧縮、Blob管理。
- `statistics.ts`: 忘却曲線や学習効率の計算アルゴリズム。

---

## 📁 4. バックエンド層 (`functions/`)

### 📌 `functions/src/`
- `index.ts`: Firebase 関数 (Callable/Triggers) のエクスポート。
- `v1/`: APIのバージョン管理。
- `triggers/`: 
  - `onUserCreate.ts`: 初期フォルダ作成などのプロビジョニング。
  - `onSyncRequest.ts`: クラウド側での差分計算とマージ。
- `services/`: サーバーサイドの DB クライアント、認証検証。

---

## 📁 5. テスト層 (`tests/` & `src/__tests__/`)

| ディレクトリ | 役割 |
|---|---|
| `tests/unit/` | 純粋関数の単体テスト (Vitest) |
| `tests/logic/` | 同期エンジンやキューの複雑な状態遷移テスト |
| `tests/e2e/` | 実際のブラウザ操作 (Playwright) による結合テスト |
| `src/services/__tests__/` | サービス層（OperationQueue 等）のモックを用いたテスト |
