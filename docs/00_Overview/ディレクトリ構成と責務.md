# プロジェクト構成とディレクトリの責務

## 1. 目的
この文書は、リポジトリ内の各ディレクトリの責務を明確にし、実装のガードレールを提供します。実際のワークスペース構成に合わせて更新しています。

## 2. ディレクトリ詳細定義

```
.
├── src/                        # Webアプリケーション（React + TypeScript）
│   ├── Components/             # Feature-based コンポーネント群
│   │   ├── ui/                 # 汎用UI（Button, Input 等）— ドメイン知識なし
│   │   ├── layout/             # アプリケーションのテンプレート（Header, Sidebar）
│   │   ├── card/               # カード編集・表示ドメイン（Editor, Block, MathBlock等）
│   │   │   └── blocks/         # ブロック実装（MathBlock.tsx 等）
│   │   ├── stats/              # 統計・グラフ関連
│   │   ├── export/             # エクスポート/インポート機能
│   │   └── review/             # 学習モード固有のUI
│   ├── Pages/                  # ルーティング対応（画面ごとの上位コンポーネント）
│   ├── contexts/               # グローバルなコンテキスト（Auth, Dialogs）
│   ├── hooks/                  # カスタムフック（useCards, useFolders 等）
│   ├── __tests__/              # 単体/コンポーネントテスト
│   ├── assets/                 # 画像等のアセット
│   ├── App.tsx
│   └── main.tsx
├── functions/                  # Firebase Functions（Node/TypeScript）
│   ├── src/
│   └── lib/
├── public/                     # 静的公開ファイル（manifest, offline.html 等）
├── docs/                       # 仕様・設計・運用ドキュメント（このディレクトリ）
├── scripts/                    # スクリプト・マイグレーション・ユーティリティ
├── services/                   # 外部サービス用の小さいサービス群（pptx-converter 等）
├── tests/                      # e2e / 結合テスト
├── test-results/               # テスト出力結果
├── build_*.txt                 # ビルドログや出力の履歴ファイル群
├── package.json
├── tsconfig.json
├── vite.config.ts
└── README.md
```

## 3. ガードレール（更新版）

### UI vs Feature
- `src/Components/ui` のコンポーネントは表示のみを担当し、`services` や `localDB` を直接参照してはいけません。

### データとビジネスロジックの分離
- データ取得・永続化・ビジネスロジックは `src/hooks`、`src/services`、または `functions/` に実装し、UI側はフックやプロップスを通じて受け取ってください。

### Firebase / Local DB
- Firebase 初期化やクラウドのロジックは `src/services` と `functions/` にまとめ、IndexedDB（Dexie）等のローカルDBは `src/services/localDB.ts` や `src/hooks/useCards.ts` を通して参照してください。

## 4. 実装上の注意点
- 新しい機能フォルダを追加する際は、まず `docs/00_Overview/ディレクトリ構成と責務.md` に責務を追記してください。
- 大きな変更（フォルダ移動・API設計変更等）はドキュメントと `README.md` を同時に更新してください。
