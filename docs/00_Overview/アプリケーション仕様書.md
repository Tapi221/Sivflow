# アプリケーション仕様書 (Comprehensive Specification)

**最終更新日**: 2026-02-04

## 1. アプリケーション概要 (Overview)

### 1.1 コンセプト
効率的な学習を支援する**暗記カード（Flashcard）Webアプリケーション**。
忘却曲線に基づいた学習スケジュール管理（SRS）と、開発者や学習者に最適化されたリッチコンテンツ作成機能を特徴とします。

### 1.2 アーキテクチャ構成
*   **Frontend**: React 18, TypeScript, Vite, Tailwind CSS
*   **Framework**: Single Page Application (SPA) with React Router v6
*   **State Management**: React Query v5 (Server State), Context API (Auth, Theme, Toast, Notification)
*   **Database (Hybrid Storage)**:
    *   **Local (Master)**: IndexedDB (via **Dexie.js**) - リアクティブなデータ取得 (`useLiveQuery`) とオフライン動作を担当。
    *   **Cloud (Backup/Sync)**: Firestore - デバイス間同期とバックアップを担当。
    *   **ブラウザストレージ準拠**: 健全性チェック、自動再構築、CLEAN/DIRTY状態管理を実装。
*   **Storage**: Firebase Storage (画像・音声・PDF)
*   **Authentication**: Firebase Auth (Google Sign-In)
*   **UX System**: エラーレベル導入（INFO/WARNING/ERROR）、プログレッシブディスクロージャー、インラインバリデーション、コンテキスト活用

## 2. 詳細機能仕様 (Detailed Features)

### 2.1 高機能カードエディター (Card Editor)
学習コンテンツを作成する中核機能 (`CardEditor.tsx`)。

*   **マルチメディア対応**:
    *   **Text**: 自動リサイズ (`AutoResizeTextarea`)、Markdownライクな入力。
    *   **Images**: ドラッグ＆ドロップ対応 (`MediaUploader`)。HEIC形式の自動JPEG変換、圧縮機能搭載。アップロード中は「楽観的UI」としてローカルBlobを表示し、バックグラウンド処理。
    *   **Audio**: 音声ファイルアップロード対応。
    *   **Code**: `Prism.js` ベースの構文強調エディタ (`CodeBlockEditor`)。言語選択可。
    *   **Math**: LaTeX 数式レンダリング (`MathJax/KaTeX` wrapper)。
    *   **Real Card Preview**: 編集中の内容を実際のカードと同じ見た目で確認できるプレビュー機能（`Flashcard.tsx`）。
*   **バリデーション (Validation)**:
    *   **空カードの保存防止**: タイトル、タグ、および全種類のブロック（テキスト、コード、画像、音声、数式、リンク）のコンテンツがすべて未入力のカードは保存できません。UI上で「保存」および「続けて作成」ボタンが非活性化されます。
*   **属性管理**:
    *   **Draft**: 入力途中保存用フラグ。学習対象から除外。
    *   **Uncertainty**: 「曖昧」フラグ。重点学習モード (`UncertainMode`) で使用。
    *   **Tags**: タグ管理 (`TagInput`)。サジェスト機能付き。
    *   **Bookmark**: 特定のカードに印を付ける機能。

### 2.2 学習アルゴリズム (SRS Logic)
`src/services/reviewAlgorithm.ts` に実装された独自アルゴリズム。詳細は [復習アルゴリズム仕様書](../03_Logic/復習アルゴ仕様書.md) を参照。

#### 定着度 (Memory Stability)
*   **範囲**: `0.01` (忘却) 〜 `1.0` (完全定着)
*   **初期値の扱い**:
    *   **DB保存値**: `0` (新規作成時)
    *   **アルゴリズム計算値**: `0.30` (初回学習開始時のベースライン)

#### フェーズ分類
*   **Phase 1 (導入期)**: 0.00 - 0.30
*   **Phase 2 (習得期)**: 0.30 - 0.50
*   **Phase 3 (定着期)**: 0.50 - 0.70
*   **Phase 4 (熟達期)**: 0.70 - 1.00

#### 次回学習日の計算式 (Interval Calculation)
`Interval = 1 + 100 * Stability^2.5`
*   指数関数的な間隔増加を実現。最大 `90日` で頭打ち（定数 `MAX_INTERVAL_DAYS`）。

### 2.3 フォルダ・階層構造 (Folder Management)
*   **Recursive Structure**: 無限階層のフォルダ構造。
*   **Memo Function**: フォルダごとに「メモ」と「画像」を保持可能。

### 2.4 画像同期と最適化 (Image Sync System)
詳細は [画像管理設計指針](../02_Features/画像管理設計指針.md) を参照。

### 2.5 UXシステム (User Experience System)
詳細は [エラーメッセージングUX設計指針](エラーメッセージングUX設計指針.md) を参照。

#### エラーレベル導入
*   **INFO**: 状態通知（自動で消える／邪魔しない）
*   **WARNING**: 放置すると困る（閉じられる／軽い行動導線）
*   **ERROR**: 続行不可（明確なアクション必須）

#### プログレッシブディスクロージャー
*   時間軸ベースの表示切り替え
*   詳細は要求時のみ表示

#### インラインバリデーション
*   ストレージ容量の常時監視
*   オンライン/オフライン状態の常時表示 (Header)
*   エラーを予防するシステム

#### コンテキスト活用
*   状況別メッセージ（初回起動 / 通常起動 / 復旧）
*   ユーザーの不安を最小化

### 2.6 セキュリティ監視と自動制御 (Security & Enforcement)
バックエンド（Cloud Functions）およびフロントエンド（SecurityMonitor）による多層防御。詳細は [クライアントサイドセキュリティ設計仕様書](../03_Specifications/クライアントサイドセキュリティ設計仕様書.md) を参照。

*   **リアルタイム監視**: Firestore のメタデータ (`isAccountLocked`, `requires2FA`) および `notifications` サブコレクションを常時監視。
*   **アカウントロック**: 重大な異常検知時に `isAccountLocked` が ON となり、全画面ロック (`AccountLockedScreen`) を強制。
*   **警告バナー**: 軽微な異常検知時に `SecurityAlertBanner` を表示。ユーザーによる既読化が可能。
*   **2要素認証要求**: 不審なアクティビティに基づき、次回ログイン時の2FAを強制するフラグを管理。

## 3. 内部アーキテクチャ詳細 (Internal Architecture)

### 3.1 ルーティング定義 (`App.tsx`)
React Router v6 を使用。認証ガード (`ProtectedRoute`) で保護される。

### 3.2 データアクセス層
*   **LocalDB**: IndexedDB を Dexie.js でラップ
*   **SafeIndexedDBWriter**: 書き込みの安全性を保証
*   **StorageStateManager**: READ_ONLY モードの管理

### 3.3 データフロー
*   **Read**: `Dexie.useLiveQuery` によるリアクティブな取得。
*   **Write**: SafeIndexedDBWriter 経由での安全な書き込み。
*   **Sync**: 差分同期による効率的なクラウド同期。

### 3.4 初期化フロー (AppInitializer)
1. **Phase 1**: DIRTY マーク
2. **Phase 2**: 健全性チェック
3. **Phase 3**: 必要なら再構築
4. **Phase 4**: Snapshot 移行
5. **Phase 5**: 履歴圧縮（バックグラウンド）
6. **Phase 6**: CLEAN マーク

### 3.5 ブラウザストレージ準拠
詳細は [ブラウザストレージ設計指針](../01_Database/ブラウザストレージ設計指針.md) を参照。

*   **健全性チェック**: スキーマバージョン、DIRTY状態、エンティティ数の検証
*   **自動再構築**: 異常検知時の自動復旧
*   **無限ループ検知**: 再構築回数の監視とERROR通知

## 4. UI/UX 特記事項

### 4.1 学習モード UX
*   **Unified Flashcard**: `Flashcard.tsx` がすべての場所（一覧・編集・学習）での表示を統一。
*   **Flexible Slots**: モードに応じた追加UI（復習回数バッジ等）をスロット経由で挿入。

### 4.2 信頼性向上策
*   **Upload Progress**: アップロードプログレスの視覚化。
*   **Reliable Upload**: `useReliableFileUpload` によるリトライ・中断耐性。
*   **Storage Monitoring**: ストレージ容量の常時監視とWARNING通知。
*   **Header Indicators**: オンライン/オフライン状態、ストレージ容量の常時表示（Status Bar廃止に伴い移行）。

### 4.3 通知システム
*   **NotificationProvider**: グローバル通知管理
*   **InfoToast**: 自動で消える通知
*   **WarningDialog**: 閉じられる警告
*   **ErrorDialog**: アクション必須のエラー
