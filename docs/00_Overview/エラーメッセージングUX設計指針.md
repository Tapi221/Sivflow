必ず**セット**で機能します。

1. **エラーレベル導入** → 重要度を即判断させる ✅ **実装完了**
2. **プログレッシブディスクロージャー** → 情報過多を防ぐ ✅ **実装完了**
3. **インラインバリデーション** → エラー自体を減らす ✅ **実装完了**
4. **コンテキスト活用** → 不安を発生させない ✅ **実装完了**

## 1. エラーレベル導入（必須インフラ）✅

**実装状況**: 完了

### 実装内容

- ✅ 型定義 (`src/types/notification.ts`)
- ✅ 通知サービス (`src/services/NotificationService.ts`)
- ✅ UIコンポーネント
  - `NotificationProvider.tsx`
  - `InfoToast.tsx`
  - `WarningDialog.tsx`
  - `ErrorDialog.tsx`
- ✅ App.tsx への統合

### 目的

- ユーザーに「今すぐ行動が必要か？」を一瞬で判断させる
- すべてが同じ重さに見える地獄を終わらせる

### レベル定義（ユーザー視点）

| レベル | 意味 | UXルール |
|--------|------|----------|
| INFO | 状態通知 | 自動で消える／邪魔しない |
| WARNING | 放置すると困る | 閉じられる／軽い行動導線 |
| ERROR | 続行不可 | 明確なアクション必須 |

### 重要ポイント

- **技術的深刻度とUI深刻度は一致しない**
- IndexedDB破損でも、自動復旧中なら INFO でいい
- ユーザーが止まる瞬間だけ ERROR を出す

### 実装例

#### INFO（状態通知）
```
データを初期化しています。
この処理によってデータが失われることはありません。

⏱️ 通常は1〜2分で完了します。
```

**特徴**:
- 自動で進行
- ユーザーの操作不要
- 不安を軽減する一文のみ

#### WARNING（放置すると困る）
```
ブラウザのストレージ容量が不足しています。

既存のデータは保存されています。
新しいデータの保存ができません。

【対処法】
1. ブラウザの設定を開く
2. 「プライバシーとセキュリティ」→「Cookie とサイトデータ」
3. 不要なサイトのデータを削除
4. このページをリロード
```

**特徴**:
- 今すぐではないが、対処が必要
- 具体的な手順を提示
- 閉じることができる

#### ERROR（続行不可）
```
申し訳ございません。

通常は自動的に復旧しますが、
今回は自動復旧の上限を超えたため、起動できない状態です。

この問題はユーザー操作が原因ではありません。
システム側の調査が必要です。

サポートにお問い合わせください。
```

**特徴**:
- 続行不可
- サポート連絡が必須
- ユーザーの責任ではないことを明示

## 2. プログレッシブディスクロージャー（核心）✅

**実装状況**: 完了

### 実装内容

- ✅ `useProgressiveMessage` フック (`src/hooks/useProgressiveMessage.ts`)
- ✅ AppInitializer への適用

### 解決する問題

- 最初から全部説明して誰も読まない問題
- 「丁寧さ」と「冗長さ」を混同している問題

### 原則

**今この瞬間に必要な情報だけを出す**

### 時間軸ベース設計（例）

```
0〜3秒:
データを準備しています

30秒:
もう少しお待ちください

60秒:
時間がかかっています
[詳細を見る]
```

### 重要ポイント

- 「詳細」は**ユーザーが要求したときだけ**
- 最初から6行説明するのは、非常ベルを常に鳴らしているのと同じ
- 読ませる設計ではなく、**読ませなくて済む設計**

### 実装例

#### 初期表示（最小限）
```
データを初期化しています。
```

#### 30秒経過（進捗確認）
```
データを初期化しています。
もう少しお待ちください。
```

#### 60秒経過（詳細提供）
```
データを初期化しています。

⏱️ 通常は1〜2分で完了します。
5分以上続く場合は、異常の可能性があるためページをリロードしてください。

[詳細を見る]
```

#### 詳細展開（ユーザー要求時のみ）
```
データを初期化しています。

この処理によってデータが失われることはありません。

⏱️ 通常は1〜2分で完了します。
5分以上続く場合は、異常の可能性があるためページをリロードしてください。

【技術情報】
- 処理: IndexedDB 再構築
- 理由: データ破損検知
- エラーコード: rebuild_required
```

## 3. インラインバリデーション（理想形）✅

**実装状況**: 完了

### 実装内容

- ✅ StorageMonitor (`src/services/StorageMonitor.ts`)
- ✅ StatusBar (`src/Components/common/StatusBar.tsx`)
- ✅ Layout.tsx への統合
- ✅ SafeIndexedDBWriter への WARNING 通知追加

### 目的

- エラーを「発生イベント」から「予防システム」に変える
- メッセージを減らす最強の方法は、**そもそも出さないこと**

### 例

| 問題 | 従来 | 改善 |
|------|------|------|
| 容量不足 | 起動時エラー | 事前WARNING |
| オフライン | 操作後エラー | 常時ステータス表示 |
| 同期失敗 | ダイアログ | 自動リトライ＋失敗時のみ通知 |

### 重要ポイント

- ユーザーは「失敗した」ことより**「邪魔された」ことにキレる**
- エラー表示は最後の手段

### 実装例

#### 容量不足の予防
```typescript
// StorageMonitor による常時監視
const quota = await storageMonitor.checkQuota();

if (quota.percent > 80) {
  // WARNING レベルで事前通知
  notificationService.warning('ストレージ容量が不足しています。');
}
```

#### オフライン状態の常時表示
```typescript
// StatusBar に常時表示
<StatusBar>
  {isOnline ? <Cloud /> : <CloudOff />}
  {isOnline ? 'オンライン' : 'オフライン'}
</StatusBar>
```

#### 同期失敗の自動リトライ
```typescript
// 自動リトライ（ユーザーに見せない）
async function syncWithRetry(maxRetries = 3) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      await sync();
      return; // 成功したら終了
    } catch (error) {
      if (i === maxRetries - 1) {
        // 最後のリトライで失敗したときだけ通知
        notificationService.warning('同期に失敗しました。自動的に再試行します。');
      }
    }
  }
}
```

**実装状況**: 完了

### 実装内容

- ✅ ContextService (`src/services/ContextService.ts`)
- ✅ メッセージテンプレート (`src/utils/messageTemplates.ts`)
- ✅ AppInitializer への適用

### 目的

- 同じ処理でも「意味」を変える
- ユーザーの不安を最小化する

### 状況別表現の例

| 状況 | 表示 |
|------|------|
| 初回起動 | 初期設定中 |
| 通常起動 | データを更新中 |
| 復帰時 | 変更内容を同期中 |

### なぜ強いか

- 技術的には同じ処理
- **心理的には全く別の体験**
- 「壊れているのでは？」という疑念を未然に潰せる

### 実装例

#### 初回起動
```
初期設定中です。

アカウント情報を準備しています。
```

#### 通常起動
```
データを更新中です。

最新の情報を取得しています。
```

#### 復帰時（オフライン後）
```
変更内容を同期中です。

オフライン中の編集をクラウドに保存しています。
```

#### エラー後の復旧
```
データを復旧中です。

問題を自動的に修復しています。
```

## 4つを統合した設計原則

この4点は必ず**セット**で機能します。

### 統合例：データ初期化フロー

#### 1. エラーレベル（INFO）
```
データを初期化しています。
```

#### 2. プログレッシブディスクロージャー（時間軸）
```
0〜30秒: データを初期化しています。
30〜60秒: もう少しお待ちください。
60秒〜: [詳細を見る] ボタンを表示
```

#### 3. インラインバリデーション（予防）
```typescript
// 事前チェック
if (rebuildCount > 3) {
  // ERROR レベルで即座に停止
  notificationService.error('自動復旧の上限を超えました。');
  return;
}
```

#### 4. コンテキスト活用（状況別）
```
初回起動: 初期設定中です。
通常起動: データを更新中です。
エラー後: データを復旧中です。
```

### どれか一つ欠けると

- **エラーレベルなし**: 情報は整理されているのに多すぎる
- **プログレッシブディスクロージャーなし**: 短いけど意味が伝わらない
- **インラインバリデーションなし**: エラーが多すぎて邪魔
- **コンテキスト活用なし**: 正しいけど不安になる
