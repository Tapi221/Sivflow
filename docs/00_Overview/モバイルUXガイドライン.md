# フロントエンド設計記録：モバイルファーストUXガイドライン

## 1. 概要
FlashcardMasterは「スキマ時間での学習」を重視しているため、モバイル体験（Mobile Experience）がUXの核となります。
デスクトップ版の縮小ではなく、**「モバイルで最も快適に使える」**ことを最優先した設計ルールを定めています。

## 2. レイアウト適応（Layout Adaptation）

### ナビゲーション (Navigation)
*   **Desktop**: サイドバー固定。画面の広さを活かし、常にメニューを表示。
*   **Mobile**: ボトムナビゲーション固定。
    *   **理由**: スマートフォンを持った時、親指が届く範囲（Thumb Zone）に主要アクションを配置するため。
    *   **ルール**: 統計や設定など、頻度が低いものは「もっと見る」に隠さず、主要な4-5アイコンに絞る。

### グリッドシステム (Grid System)
*   **Folder/Card List**:
    *   Desktop: `grid-cols-4` (情報量を最大化)
    *   Mobile: `grid-cols-2` (視認性を確保しつつ一覧性を維持)
    *   **例外**: カード編集画面のプロパティなどは `grid-cols-1`（縦積み）にして、タップミスを防ぐ。

## 3. アクションボタンの設計

### "Text Hidden" 戦略
モバイル画面幅（<768px）では、ボタン内のテキストラベルを積極的に非表示にし、**アイコンのみ**で表現します。

*   **実装例**:
    ```jsx
    <Button>
      <Plus className="w-5 h-5" />
      <span className="hidden md:inline">新規作成</span>
    </Button>
    ```
*   **理由**:
    *   ラベルを表示するとボタンが肥大化し、ツールバーからはみ出す。
    *   学習アプリの特性上、アイコン（再生、編集、削除）の意味が共通認識されやすいため、テキストなしでも機能する。

### タップターゲット (Touch Target)
*   モバイルでのボタンサイズは最低 `44px` (または `h-10` 以上) を確保する。
*   密集するアクション（例：カード右上のメニュー）は、`DropdownMenu` に格納するか、十分なマージン (`gap-2`以上) を取る。

## 4. 入力とインタラクション

### 編集体験 (Editing)
*   **カード編集**:
    *   プレビュー画面はデフォルトで隠す（`Tabs` や `Switch` で切り替え）。
    *   キーボード出現時に画面が隠れないよう、モーダルではなく別ページ (`/CardEdit`) として遷移させる設計を推奨。

### フィードバック (Feedback)
*   **Toasts**: モバイルでは上部ではなく「下部（Bottom）」または「中央」に出す。
    *   理由: 上部は指で隠れやすいため。現在は `sonner` のデフォルト位置（右下/中央下）を採用。

---

## 5. 今後の追加実装ルール
新しいUIコンポーネントを追加する際は、以下のチェックリストを通してください。

1.  **"Thumb Friendly?"**: 片手持ちの親指で操作できるか？
2.  **"No Overflow?"**: `w-full` ではなく `max-w-[100vw]` や `overflow-hidden` が考慮されているか？
3.  **"Readable?"**: フォントサイズは `text-xs` (12px) 未満になっていないか？（バッジ等は例外）

このガイドラインを守ることで、「PCで作ってスマホで崩れる」という典型的な失敗を防ぎます。
