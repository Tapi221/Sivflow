# プロダクトデザイン：人格とマイクロコピー

## 1. プロダクトの人格 (Persona)

FlashcardMasterは単なるツールではなく、**「知的で、静かなる伴走者 (The Silent Mentor)」** です。
ユーザーの学習を邪魔せず、しかし必要な時には的確な称賛とガイダンスを提供します。

### キーワード
*   **Supportive (支援的)**: ユーザーを否定しない。「忘れた」は失敗ではなく、次の成長へのステップであると定義する。
*   **Professional (プロフェッショナル)**: 余計な装飾や馴れ馴れしい言葉遣いは避ける。信頼できる専門家のようなトーン。
*   **Calm (穏やか)**: 焦らせない。通知は控えめに。色は目に優しいTealとSlateを採用。

## 2. マイクロコピーの原則 (Microcopy Principles)

システムメッセージやボタンのラベルには、以下の原則を適用しています。

### 原則1: ユーザーの行動を肯定する
「学習しなかった」ではなく「休息をとった」と解釈できるような表現を心がけます。

*   **Bad**: "あなたは昨日学習しませんでした。"（事実の指摘・非難）
*   **Good**: "おかえりなさい。今日も着実に知識を積み上げましょう。"（再開の歓迎）

### 原則2: 結果よりプロセスを称賛する
単に正解数を褒めるだけでなく、Resistance Score（継続）やStreak（習慣）に焦点を当てます。

*   **Context**: 365日連続学習を達成した時
*   **Message**: "偉大な知性は、日々の小さな習慣から作られます。" (Great intellect involves habit.)

### 原則3: エラーはシステムの責任とする
ユーザーが何か失敗した時（例：オフラインでアップロード）、ユーザーを叱責せず、システムが対処することを伝えます。

*   **Bad**: "ネットワークエラー。接続を確認してください。"（突き放し）
*   **Good**: "オフラインのため、変更は端末に保存されました。接続回復時に自動で同期します。"（解決策の提示）

## 3. 具体的な用語集 (Wording)

| 画面/機能 | 用語 (JA) | 用語 (EN) | 意図 |
| :--- | :--- | :--- | :--- |
| **ホーム** | "今日の復習" | "Priority Task" | 「やらなきゃいけない」ではなく「これが優先事項だ」とプロっぽく提示。 |
| **評価ボタン** | "忘れた" | "Forgot" | 正直に申告するハードルを下げる。赤色だが攻撃的ではない淡い色。 |
| **評価ボタン** | "余裕" | "Easy" | 「簡単」ではなく「余裕」。マスタリーを感じさせる言葉。 |
| **完了画面** | "お疲れ様でした" | "Review Complete" | シンプルに労う。派手なファンファーレよりも静かな達成感を重視。 |

## 4. プレミアム感を出す工夫
*   **英語の併記**: 重要な見出しには小さく英語（例: `SEQUENCE`）を添えることで、知的な雰囲気を醸成する（装飾としての英語）。
*   **数字の扱い**: 数字は等幅フォントや特定のスタイル（例: `text-6xl`）で強調し、データドリブンな信頼感を演出する。
