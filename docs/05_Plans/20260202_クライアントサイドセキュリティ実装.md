# クライアントサイドセキュリティ実装計画

## 概要
バックエンド（Cloud Functions）で判定されたセキュリティステータス（アカウントロック、2FA要求、警告通知）をクライアントサイド（React）で受信し、UI制御（ロック画面表示、強制ログアウト、バナー表示）を行う実装計画です。

## 実装方針

### 1. SecurityMonitor の拡張
`src/services/logic/SecurityMonitor.ts` に以下の機能を追加します。
*   **メタデータ監視**: Firestore の `users/{userId}` ドキュメントをリアルタイム監視 (`onSnapshot`) し、`isAccountLocked`, `requires2FA` のフラグ変更を検知します。
*   **通知監視**: `users/{userId}/notifications` コレクションを監視し、未読のセキュリティ警告（`type: 'SECURITY_ALERT'`）を取得します。
*   **状態通知**: 変化があった際、コールバックを通じてアプリケーション（`AuthContext`）に通知します。

### 2. AuthContext の拡張
`src/contexts/AuthContext.tsx` でセキュリティ状態を管理します。
*   `securityState` ステートを追加（`isLocked`, `requires2FA`, `alerts`）。
*   `SecurityMonitor` からの通知を受け取り、ステートを更新します。
*   `isLocked` が `true` の場合、全画面を「アカウントロック画面」に切り替えるか、強制ログアウト処理を実行します。

### 3. UI コンポーネントの実装
*   **AccountLockedScreen**: アカウントロック時に表示する専用画面。「管理者によりロックされました。サポートにお問い合わせください」等のメッセージを表示。
*   **SecurityAlertBanner**: 画面上部に警告を表示するコンポーネント（Warningレベルの通知がある場合）。

## 検証計画

### 1. 自動テスト (Emulator)
既存の `functions/src/tools/verify_security.ts` を使用し、バックエンド側でフラグを立てる。
*   `ADMIN_ACCOUNT_LOCK` イベントを送信し、スコアを 81 以上にする。

### 2. 手動検証 (UI動作確認)
1.  アプリを起動し、ログイン状態にする。
2.  別ターミナルで `verify_security.ts` を実行、または Firestore Emulator UI で直接 `users/{userId}` の `isAccountLocked` を `true` に変更。
3.  アプリが即座に「アカウントロック画面」に遷移することを確認。
4.  Firestore で `isAccountLocked` を `false` に戻し、復帰（または再ログイン）できるか確認。
5.  `users/{userId}/notifications` に警告を追加し、バナーが表示されるか確認。
