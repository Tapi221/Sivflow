# 本番環境でのセキュリティテスト手順（安全版）

## 1. 準備フェーズ
*   **管理者アクセスの確保**
    *   Firebase コンソールで自分のユーザー情報を確認
    *   Firestore や Authentication に直接アクセスできる状態にしておく
*   **復旧用ドキュメントの作成**
    *   `users/{userId}/security/status` のバックアップコピーを作成
    *   端末リスト、スコア、アカウントロック状態をメモしておく
*   **テスト端末の限定**
    *   テスト用の1端末（自分のメイン端末）を対象にする
    *   不要な端末は一時的にログアウトしておく

## 2. ログ送信テスト
1.  クライアントで `securityLogs` を作成
    *   例: `SYNC_AUTH_ERROR` や `ACCESS_DENIED_REVOKED` を送信
2.  Cloud Functions のログ（エミュレータまたは本番コンソール）で処理が発動するか確認
3.  Firestore の `users/{userId}/security/status` を確認し、リスクスコアが加算されているか確認

## 3. スコア閾値テスト
21点 (Warning)、51点 (High)、81点 (Critical) の各閾値で以下を確認：

| スコア | 期待動作 | 確認方法 | 検証結果 (2026/02/02 実施) |
| :--- | :--- | :--- | :--- |
| **21点以上** | 通知作成 | `users/{userId}/notifications` を確認 | **OK** (30点で通知生成を確認) |
| **51点以上** | `requires2FA = true` | Firestore 上のユーザードキュメント確認 | **OK** (80点でメタデータ更新を確認) |
| **81点以上** | 全端末 Revoke + アカウントロック | `users/{userId}/devices/*` が `revoked`、`isAccountLocked = true` | **OK** (100点でロック・Revokeを確認) |

> **注意**: Critical テスト時は即座に復旧手順が必要

## 4. 復旧手順（必須）
1.  **端末 Revoke の解除**
    *   Firestore で `users/{userId}/devices/*` の `status` を `active` に戻す
2.  **アカウントロック解除**
    *   `users/{userId}/isAccountLocked = false`
3.  **リスクスコアリセット**
    *   `users/{userId}/security/status` のスコアを 0 にリセット
4.  **通知クリア（任意）**
    *   `users/{userId}/notifications` のテスト用通知を削除

## 5. 注意事項
*   本番ログがテストデータで汚染されるので、あとで解析する場合はテスト用ユーザーを分けるか、`metadata.test = true` のフラグを付けて区別すること
*   Functions のバグにより不可逆な状態になる可能性はゼロではないため、必ず復旧手順を事前に試すこと
*   外部サービス連携（FCM, SendGrid）を使う場合、テスト通知がユーザーに届くので注意

## 付録: エミュレータによる検証ログ (要約)
```
[Verify 1] Notifications count: 1 (Expected >= 1) -> OK
[Verify 1] Current Score: 30
[Verify 2] isAccountLocked: true (Expected: true) -> OK
[Verify 2] Device Status: revoked (Expected: revoked) -> OK
[Verify 2] Final Score: 100
```
