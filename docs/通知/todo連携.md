# 外部ToDo連携：復習タスク自動追加機能 仕様書

## 1. 概要

本仕様は、学習用カードアプリにおいて、ユーザーの復習予定を **外部ToDo（Google ToDo想定）へ自動で追加**する機能の設計を定義する。

本機能は大規模リリースを前提とし、以下を最優先とする。

* 一貫性のある挙動
* 二重生成・日付ズレの防止
* 朝型・夜型ユーザー双方への対応
* クライアント状態に依存しない安定動作

---

## 2. 基本方針

### 2.1 連携方針

* 外部ToDoとは **同期しない**
* 本アプリをマスターデータとする
* 外部ToDoへの操作は「追加のみ（エクスポート）」とする

### 2.2 吐き出し対象

* **その日に実行すべき復習タスクのみ**
* 未来日付のタスクは外部ToDoへ追加しない

---

## 3. 用語定義

### 3.1 論理的な「今日（Logical Day）」

ユーザーが設定した「1日の開始時刻」を基準に定義されるアプリ内の日付概念。

* 実日付とは一致しない場合がある
* 復習計算・タスク追加はすべて Logical Day 基準で行う

---

## 4. ユーザー設定

### 4.1 設定項目

```ts
UserSettings {
  exportEnabled: boolean;   // 外部ToDo連携 ON/OFF
  dayStartAt: string;       // 1日の開始時刻（HH:mm）
}
```

### 4.2 初期値

* exportEnabled: OFF
* dayStartAt: `07:00`

### 4.3 設定仕様

* dayStartAt は 00:00〜23:59 の任意時刻を指定可能
* 毎日同一の時刻で固定
* 日ごと・タスクごとの変更は不可
* 設定変更は **次の Logical Day から反映**

---

## 5. 復習タスク確定ルール

### 5.1 確定タイミング

* Logical Day の開始時刻を跨いだ瞬間に、その日の復習タスクを確定する

### 5.2 確定後の扱い

* 確定後に作成されたカードは **翌日の復習対象**とする
* 作成途中（入力中・未保存）状態は考慮しない

---

## 6. カード作成時の扱い

### 6.1 Logical Day の付与

```ts
Card {
  id: string;
  createdAt: Timestamp;
  logicalDay: string; // YYYY-MM-DD
}
```

* logicalDay は作成時点で確定
* 後から自動変更しない

### 6.2 開始時刻跨ぎ時の挙動

* 開始時刻を超えた後に保存されたカードは翌日の logicalDay を持つ

---

## 7. 外部ToDoへのタスク追加仕様

### 7.1 追加タイミング

* Logical Day 開始時刻に 1日1回のみ実行

### 7.2 追加条件

* exportEnabled = true
* 当該 Logical Day に対して未追加であること

### 7.3 タスク粒度

* **1日 = 1タスク**
* カード1枚単位では追加しない

---

## 8. タスク内容設計

### 8.1 タイトル例

```
数学I 復習（15枚）
```

### 8.2 メモ例

```
・一次関数
・平方完成
・所要目安：25分
```

### 8.3 期限

* 論理的な「今日」
* 時刻指定は行わない（または開始時刻固定）

---

## 9. 重複防止・ログ設計

### 9.1 エクスポートログ

```ts
ReviewExportLog {
  userId: string;
  logicalDay: string;   // YYYY-MM-DD
  googleTaskId: string;
  exportedAt: Timestamp;
}
```

### 9.2 ルール

* 同一 logicalDay につき1レコードのみ
* ログ存在時は再追加しない

---

## 10. バックエンド実行方式

### 10.1 実行主体

* サーバーサイド（Cloud Scheduler + Cloud Functions / Cloud Run）

### 10.2 実行頻度

* 定期チェック（例：5分〜15分間隔）
* Logical Day 開始判定はサーバー時刻 + ユーザー timezone を使用

---

## 11. UI要件

### 11.1 設定画面

* 外部ToDo連携 ON/OFF トグル
* 1日の開始時刻選択

説明文：

> この時刻に、その日の復習タスクが追加されます

### 11.2 プレビュー表示

```
次回追加予定：
今日 13:00
数学I 復習（15枚）
```

### 11.3 カード作成画面補足表示

* 開始時刻跨ぎ後：

```
このカードは「明日の復習」に含まれます
```

---

## 12. 明示的にやらないこと

* 外部ToDoとの双方向同期
* 作成中状態による確定遅延
* 未来日付タスクの自動追加
* 日付変更時刻（00:00）への固定依存

---

## 13. 設計原則まとめ

* 境界は「時刻」で切る
* 状態では切らない
* 自動処理は最小限・一貫性優先
* 柔軟性は設定に閉じ込める
