# 一問一答モードにおけるデータ永続化（リロード対策）

## 1. 目的
一問一答の連続作成モードにおいて、ユーザーがページを誤ってリロードしたり、ブラウザが意図せず閉じてしまった場合に、それまで入力したデータや構築したカードのスタック（履歴）が失われないようにすることを目的としています。

## 2. 永続化の仕組み
データ保護を確実にしつつ、完了時にはクリーンな状態を保つため、`sessionStorage` を採用しています。

### A. エディタスタックの保存 (`OneQAMode.jsx`)
- **何が保存されるか**: 現在画面に表示されているカードエディタのIDリスト、およびそれぞれの「保存済み/未保存」の状態。
- **保存タイミング**: カードが追加された時、または保存が完了した時。
- **復元タイミング**: `OneQAMode` コンポーネントがマウントされる際、`sessionStorage` にデータがあれば自動的に復元します。

### B. 入力内容の保存 (`CardEditor.tsx`)
- **何が保存されるか**: 各エディタ内に入力されたテキスト、タグ、各種設定（ブックマーク、不確実フラグなど）。
- **保存タイミング**: 入力内容が変更されるたびに、エディタIDをキーとして自動保存されます。
- **復元タイミング**: エディタが描画される際、自身のIDに紐づくデータがあれば自動的にフィールドに反映されます。

## 3. データの破棄（クリーンアップ）
不要なデータが残って次回の作成時に混ざるのを防ぐため、以下のタイミングで `sessionStorage` 内のデータを明示的に削除します。

- 「作成完了」ボタンを押して正常に完了した時。
- 「キャンセル」ボタンを押して意図的に中断した時。

## 4. ユーザー体験へのメリット
- **安心感**: 長い文章を入力している途中で接続が切れたりリロードしたりしても、即座に元の状態から再開できます。
- **シームレスな操作**: ページ遷移や更新を意識することなく、連続的な作業に集中できます。
