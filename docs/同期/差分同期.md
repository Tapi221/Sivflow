Firebase Differential Sync + Image + Device Tracking Implementation Plan (改訂版)
目的

ユーザーの学習データ（カード・フォルダ・設定・統計）を複数端末間で安全に同期

画像はサムネイル優先で転送し、必要時のみ高解像度を取得

端末ごとの同期状況・最終同期日をユーザーに確認可能にする

1. データモデル & スキーマ
BaseEntity

isDeleted: boolean（ソフトデリート管理）

updatedAt: timestamp（更新時刻）

UploadedImage

thumbnailUrl?: string

remoteId?: string

sizeBytes?: number（アップロード制御用）

SyncMetadata

deviceId: string

lastSyncTime: Date | Timestamp | null

lastHighResSync: Date | Timestamp | null（高解像度取得日時）

追加方針

userId + updatedAt インデックスを全テーブルに追加

競合解決は「Last Write Wins」＋「フィールド単位のマージ」を検討（カード本文と画像参照は個別に判断）

2. 画像同期
imageSyncService.ts

Blob 検出

local DB で remoteUrl がない画像を検出

サムネイル生成

Canvas / OffscreenCanvas で低解像度を生成

アップロード

高解像度とサムネイルを Firebase Storage にアップロード

sizeBytes を記録して無料枠・課金リスクを管理

オンデマンド取得

同期時はサムネイルのみ

カード閲覧時に高解像度をダウンロード

コスト制御

1ユーザーあたりの総容量・1枚画像の上限を設定

サブスク制限がある場合はアップロード権限を制御

3. 同期コアロジック
syncService.ts

Differential Pull

updatedAt > lastSyncTime のドキュメントを取得

isDeleted = true の場合はローカルも削除/非表示

Differential Push

ローカルで dirty なアイテムを検出

バッチでアップロード

競合解決

基本: updatedAt で勝者決定

フィールド単位マージ: 画像は別扱い、本文は文字単位で Last Write Wins

オフライン/オンライン

navigator.onLine または Firebase の接続イベントで自動同期

再接続時は差分のみを同期

4. 端末追跡 & UI

SyncMetadata を Firestore/LocalDB に保持

ユーザーに表示する内容：

端末名 / ID

最終差分同期日 (lastSyncTime)

高解像度同期日 (lastHighResSync)

ローカルデータが存在するかどうかを端末ごとに判定可能

プライバシー配慮: 他端末の内容を他ユーザーに見せない

5. ユーザー設定 & 学習進捗

userSettings と userStats は同期対象に明示

オフライン編集後も自動同期で最新状態に反映

端末間で差分が正確に反映されるよう、更新時刻を厳密に管理

6. テスト・検証計画
Automated Tests

isNewer 比較ロジック

Soft delete 伝播確認

モック Firebase Storage/Firestore での画像アップロード

大量画像・複数端末でのバッチ同期負荷テスト

Manual Verification

初回同期

ローカル DB をクリア → ログイン → 全データがクラウドから同期される

オフライン編集

オフラインでカード作成/編集/削除 → オンライン復帰で同期

マルチデバイス

端末 A/B で同時編集 → フィールド単位マージが正しく動作

画像同期

カード追加 → サムネイル同期のみ → 高解像度は閲覧時に取得

端末追跡 UI

全端末の最終同期日・高解像度同期日が正しく表示される

7. 開発環境 & コスト管理

Firebase Emulator で開発・テスト

Storage/Firestore の無料枠上限を意識して容量制御

高解像度画像は必要な場合のみ同期、無駄な課金を回避
