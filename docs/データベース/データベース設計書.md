# データベース設計書 (LocalDB Schema)

## 概要
本アプリケーションは `Dexie.js` を使用した IndexedDB ラッパー (`LocalDB`) をメインのデータストアとして採用しています。
"Offline First" の原則に基づき、すべてのデータは一旦ローカルに保存され、バックグラウンドでクラウド (Firebase) と同期されます。

## スキーマバージョン
現在の最新バージョン: **10**
定義: `LocalDB.ts` 参照

---

## 1. 主要エンティティ

### `cards` (学習カード)
アトミックな学習単位です。

| カラム名 | 型 | インデックス | 説明 |
| :--- | :--- | :--- | :--- |
| `id` | string | **PK** | UUID (v4)。 |
| `userId` | string | ○ | 所有者ID。 |
| `folderId` | string | ○ | 所属フォルダID。 |
| `isDeleted` | boolean | | 論理削除フラグ。 |
| `isDraft` | boolean | ○ | 下書きフラグ。 |
| `questionText` | string | | 表面テキスト。 |
| `answerText` | string | | 裏面テキスト。 |
| `questionImages` | URL[] | | 画像URL配列。 |
| `answerImages` | URL[] | | 画像URL配列。 |
| `questionAudios` | URL[] | | 音声URL配列。 |
| `questionCode` | object | | `{ code: string, language: string }` |
| `answerCode` | object | | `{ code: string, language: string }` |
| `questionMemo` | string | | 補足メモ。 |
| `answerMemo` | string | | 補足メモ。 |
| `tags` | string[] | ○ | タグ配列（MultiEntry）。 |
| `hasUncertainty`| boolean | | 「はてな」フラグ。 |
| `isBookmarked` | boolean | | ブックマークフラグ。 |
| `memoryStability`| number | | FSRS定着度 (0.01-1.0)。 |
| `difficulty` | number | | 難易度 (1-10)。 |
| `interval` | number | | 現在の復習間隔 (日)。 |
| `nextReviewDate` | Date | ○ | 次回予定日。 |
| `lastReviewDate` | Date | | 前回実施日。 |
| `updatedAt` | Date | ○ | 最終更新日時。 |
| `localUpdatedAt` | Date | | ローカル操作日時（同期競合判定用）。 |
| `reviewCount` | number | | 累積復習回数。 |

### `folders` (フォルダ)
階層構造を持つコンテナです。

| カラム名 | 型 | インデックス | 説明 |
| :--- | :--- | :--- | :--- |
| `id` | string | **PK** | UUID。 |
| `folderName` | string | | 表示名。 |
| `parentFolderId` | string | ○ | 親フォルダID。ルートは null/empty。 |
| `isDeleted` | boolean | | 論理削除フラグ。 |
| `isHidden` | boolean | | 統計・復習からの除外フラグ。 |
| `updatedAt` | Date | | 最終更新日時。 |

### `tags` (タグ管理)
タグごとの色設定などを保持します。

| カラム名 | 型 | インデックス | 説明 |
| :--- | :--- | :--- | :--- |
| `compoundId` | string | **PK** | `[rootFolderId]+[name]` の複合キー。 |
| `name` | string | | タグ名。 |
| `rootFolderId` | string | ○ | スコープ定義用フォルダID。 |
| `color` | string | | 表示色 (Tailwind class or Hex)。 |

## 2. ログ・履歴系

### `studyLogs` (学習記録)
UI上の「ストリーク」や「ヒートマップ」の元データとなります。

| カラム名 | 型 | 説明 |
| :--- | :--- | :--- |
| `id` | string (PK) | ログID。 |
| `cardId` | string | カードID。 |
| `userId` | string | ユーザーID。 |
| `folderId` | string | 実施時のフォルダID。 |
| `studiedAt` | Date | 実施日時。 |
| `subjectiveScore` | number | 0:Forgot, 1:Hard, 2:Good, 3:Easy。 |
| `nextReviewInterval`| number | 更新後の間隔 (日)。 |
| `stabilityBefore` | number | 実施前のStability。 |
| `stabilityAfter` | number | 実施後のStability。 |

### `levelHistories` (日次集計)
統計グラフ描画用スナップショットです。

| カラム名 | 型 | 説明 |
| :--- | :--- | :--- |
| `id` | string | `[userId]_[YYYY-MM-DD]` 形式のID。 |
| `date` | string | YYYY-MM-DD。 |
| `counts` | object | `{ unstable: 10, fragile: 5, ... }` フェーズ別枚数。 |

## 3. 設定・同期系

### `userSettings` (ユーザー設定)
| キー | 説明 |
| :--- | :--- |
| `id` | ユーザーID。 |
| `minStabilityForCrown` | 王冠条件(%)。 |
| `autoCarryOver` | 期限切れ繰越フラグ。 |
| `displayName` | 表示名。 |
| `photoURL` | アバター画像URL。 |

同期システムの制御テーブルです。詳細は `docs/03_Logic/同期システム仕様書.md` を参照。
