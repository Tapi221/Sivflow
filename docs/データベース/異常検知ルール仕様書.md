# 異常検知ルール仕様書 (Implemented)

## 1. 概要
本システムは、単一イベントの重み付けと、時間内の発生頻度（行動パターン）を組み合わせてリスクスコア (`riskScore`) を算出し、動的にセキュリティレベルを制御する。

## 2. セキュリティイベント定義

| イベントタイプ | 説明 |
| :--- | :--- |
| **LOGIN_FAILED** | ログイン試行失敗 |
| **SYNC_AUTH_ERROR** | 同期処理時の認証/権限エラー |
| **ACCESS_DENIED_REVOKED** | 無効化(Revoked)されたデバイスからのアクセス |
| **SENSITIVE_OP_REVOKED** | 権限のない重要操作の試行 |
| **ADMIN_DEVICE_REVOKE** | 管理者によるデバイス解除 |
| **ADMIN_ACCOUNT_LOCK** | 管理者によるアカウントロック |
| **LOCK_CONTENTION_EXCESS** | 同期ロックの過剰な競合 |
| **SYNC_CONFLICT_EXCESS** | データの競合多発 |

## 3. 検知ルールとスコアリング

各イベント発生時に以下のロジックでスコアを加算する。
`riskScore = min(100, currentScore + addedScore)`

| イベント | 加算スコア | 条件 (TimeWindow / Count) | 重大度 |
| :--- | :--- | :--- | :--- |
| **LOGIN_FAILED** | +5 | 5分間に1回 (頻度制限) | Low |
| **SYNC_AUTH_ERROR** | +10 | 即時 (Always) | Medium |
| **ACCESS_DENIED_REVOKED** | +30 | 即時 (Always) | High |
| **SENSITIVE_OP_REVOKED** | +20 | 即時 (Always) | Medium |
| **ADMIN_DEVICE_REVOKE** | +10 | 即時 (Always) | Medium |
| **ADMIN_ACCOUNT_LOCK** | +50 | 即時 (Always) | Critical |
| **LOCK_CONTENTION_EXCESS** | +7 | 5分間に10回以上 | Low |
| **SYNC_CONFLICT_EXCESS** | +3 | 5分間に10回以上 | Low |

## 4. 減衰ルール (Decay Logic)

誤検知や更生を考慮し、時間経過とともにスコアを自然減少させる。

*   **ロジック**: 1時間経過するごとに **-3点**
*   **式**: `currentScore = max(0, currentScore - floor(hoursElapsed * 3))`
*   **タイミング**: 新しいイベント発生時、または定期的な評価時

## 5. リスクレベルと自動対処

蓄積された `riskScore` に基づき、以下のステージで対応を行う。

| RiskScore | レベル | 自動対処 (Server Action) | クライアント挙動 |
| :--- | :--- | :--- | :--- |
| **0 - 20** | Safe | なし | 通常動作 |
| **21 - 50** | Warning | 通知ログ作成 (`notifications`) | **SecurityAlertBanner** 表示 |
| **51 - 80** | High | `requires2FA = true` | 次回ログイン時 2FA 要求 |
| **81 - 100** | Critical | `isAccountLocked = true`<br>全デバイス `revoked` | **AccountLockedScreen** 表示<br>操作不能化 |
