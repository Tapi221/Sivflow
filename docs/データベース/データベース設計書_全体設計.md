# データベース設計書

Flash Master のデータベース（Firestore + クラウドストレージ）設計です。

## データ保存方針

本アプリは**ハイブリッド設計**を採用しており、データの性質に応じて保存先を分離します。

### 保存先の分類

| データ種類 | 保存先 | 理由 |
|----------|--------|------|
| 認証情報・ユーザー設定・統計値・共有リンク | Firestore | メタデータ、検索・集計が必要 |
| カードコンテンツ・フォルダ構造・学習ログ | ユーザークラウドストレージ | 大容量データ、プライバシー |

## Firestore コレクション設計

### users/{userId}

ユーザー基本情報。

```typescript
{
  userId: string;                    // ドキュメントID（Firebase Auth UID）
  email: string;
  displayName?: string;
  profileImageUrl?: string;
  authMethods: Array<'email' | 'google' | 'apple' | 'microsoft'>;
  createdAt: Timestamp;
  lastLoginAt: Timestamp;
  isSuspended?: boolean;             // アカウント停止フラグ
  plan: 'free' | 'pro';              // サブスクリプションプラン
  baseQuotaMB: number;               // 基本容量（MB）
  extraQuotaMB: number;              // 追加容量（MB）
  usedQuotaMB: number;               // 使用容量（MB）
}
```

**インデックス**: なし（documentId でアクセス）

### userStats/{userId}

統計データ（事前集計済み）。

```typescript
{
  userId: string;                    // ドキュメントID（userId と同一）
  totalStudyCount: number;           // 総学習回数
  todayStudyCount: number;           // 本日の学習回数
  weeklyStudyCount: number;          // 今週の学習回数

  accuracyRate: number;              // 正答率（%）

  updatedAt: Timestamp;

```

**更新方法**: Cloud Functions が学習完了時のサマリーを受け取り、インクリメンタル更新

**インデックス**: なし（documentID でアクセス）

## セキュリティルール

### Firestore セキュリティルール

すべてのコレクションで `userId` チェックを必須とする。

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // ユーザー認証状態チェック
    match /users/{userId} {
      allow read, write: if request.auth.uid == userId;
    }
    
    match /userSettings/{userId} {
      allow read, write: if request.auth.uid == userId;
    }
    
    match /userStats/{userId} {
      allow read: if request.auth.uid == userId;
      // write は Cloud Functions のみ許可
    }
    
    match /shareLinks/{shareLinkId} {
      allow read: if 
        request.auth.uid == resource.data.ownerUserId;
      allow create: if request.auth.uid == request.resource.data.ownerUserId;
      allow update, delete: if request.auth.uid == resource.data.ownerUserId;
    }
    
    match /sharedFolders/{sharedFolderId} {
      allow read: if
        request.auth.uid == resource.data.originalOwnerUserId ||
        request.auth.uid == resource.data.importedByUserId;
      allow update, delete: if request.auth.uid == resource.data.importedByUserId;
    }
    
    match /loginHistory/{loginHistoryId} {
      allow read, write: if request.auth.uid == resource.data.userId;
    }
  }
}
```

## データ整合性

### 制約

1. **userId 一貫性**: すべてのデータは `userId` で紐づく
2. **カードとフォルダ**: カードは必ず有効な `folderId` を持つ
3. **統計データ**: `userStats` は Cloud Functions のみが更新可能

### トランザクション

- カード削除時: カード本体 + 関連ログを削除
- フォルダ削除時: フォルダ + 配下カード + 関連ログを削除（ごみ箱に移動）

---

## パフォーマンス考慮

### インデックス

- `shareLinks`: `ownerUserId` + `createdAt`（複合インデックス）
- `sharedFolders`: `importedByUserId` + `importedAt`（複合インデックス）
- `loginHistory`: `userId` + `loginDate`（複合インデックス）

### データ量制限

- Firestore ドキュメントサイズ: 最大 1MB
- クラウドストレージファイル: 画像 5MB、音声 10MB

### クエリ最適化

- 統計画面: `userStats` を1回の read で取得（事前集計済み）
- リスト表示: ページネーション対応（必要に応じて）
