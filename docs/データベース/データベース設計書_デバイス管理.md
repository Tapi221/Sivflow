# デバイス管理コレクション定義書

Firebase Firestoreにおけるデバイス管理（Device Management）のためのコレクション定義です。

## 概要

ユーザーが使用するデバイス（クライアント端末）の情報を管理し、同期の制御、セキュリティ監査、および古いセッションのクリーンアップに使用します。

## コレクションパス

```plain
sync_metadata/{userId}/devices/{deviceId}
```

- **userId**: Firebase Auth UID
- **deviceId**: クライアント生成のUUID（localStorageに保存）

## データモデル

```typescript
interface DeviceDocument {
  // 識別子
  deviceId: string;                  // ドキュメントIDと同一
  
  // 基本情報
  deviceName?: string;               // ユーザーが設定可能なデバイス名（例: "iPhone 13"）
  
  // 状態フラグ
  status: 'active' | 'revoked';      // デバイスの論理状態
                                     // - active: 有効
                                     // - revoked: アクセス権剥奪（紛失・盗難時など）

  // タイムスタンプ
  lastSyncTime: Timestamp | null;    // 最終同期成功日時
  lastHighResSync: Timestamp | null; // 高解像度画像の最終同期日時
  lastSyncAttempt?: Timestamp | null;// 失敗含む試行記録。状態判定・cleanup条件には使用しない
  
  revokedAt?: Timestamp | null;      // 無効化された日時（status='revoked'の場合）

  // システム制御（Phase 2: Reliability）
  clientSeq?: number;                // クライアントシーケンス番号 (Single Source of Truth)
                                     // ※注意: 将来的にマルチキュー構成にする場合は、ここではなくQueueドキュメント等へ移動が必要。
  needsSeqRepair?: boolean;          // シーケンス番号の不整合検知フラグ
  safeMode?: boolean;                // セーフモード（Read-Only）フラグ
  queueHardLimit?: number;           // キュー肥大化防止のための上限設定
}
```

## フィールド詳細

| フィールド名 | 型 | 説明 |
|------------|----|------|
| `deviceId` | string | ドキュメントIDと同一。クライアント側で生成されるUUID。 |
| `status` | string | `'active'` または `'revoked'`。`revoked` の場合、APIアクセスはブロックされる。 |
| `revokedAt` | Timestamp | デバイスがブロックされた日時。監査用。 |
| `lastSyncTime` | Timestamp | 最終同期日時。**UI上の「アクティブ/非アクティブ」判定は、この値と現在時刻の差分（例: 30日以内）から動的に算出する。**永続化フラグとしては持たない。 |

## クリーンアップポリシー

- **自動クリーンアップ**: 以下条件を満たすデバイスは「古いセッション」とみなされ、物理削除される。
    - `lastSyncTime` が **90日以上前** (短期の非アクティブで削除してはならない)
    - かつ、`status` が `'revoked'` **ではない**
- **監査保存**: `status: 'revoked'` のデバイスは、明示的な解除操作がない限り物理削除されず、セキュリティ監査のために残される（あるいは別途アーカイブポリシーに従う）。

## セキュリティルール

```javascript
match /sync_metadata/{userId}/devices/{deviceId} {
  // 本人のみが読み書き可能
  allow read, write: if request.auth.uid == userId;
  
  // 検証ロジック（推奨）
  // allow write: if request.auth.uid == userId 
  //   && request.resource.data.userId == userId;
}
```

## インデックス

現状、複合インデックスは不要（コレクションサイズが小さいため）。
標準の単一フィールドインデックスのみ使用。

## 関連サービス

- **SyncServiceV2**:
    - `checkDeviceStatus()`: 同期開始時に `status` をチェックし、`revoked` なら例外をスロー。
    - `cleanupInactiveDevices()`: 古いデバイスの削除を実行。
    - `updateDeviceName()`: デバイス名の更新。
