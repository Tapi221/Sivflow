# 一問一答モード 機能仕様書 (Rev.5)

**概要:** 連続作成に最適化した一問一答モードの仕様。`CardEditor`を再利用し、保存済みカードを残すスタック型UIを採用します。

**重要ポイント:**
- 連続作成で新しいカードを下に追加、自動スクロールで入力をスムーズに継続可能。
- 空カードの保存を禁止し、部分入力は下書き扱いで保存します。

## 1. 概要
ユーザーフィードバックに基づき、一問一答モードのUI/UXを刷新しました。
従来のリスト形式ではなく、**既存のカード編集画面（CardEditor）をベースとした単票形式**を採用し、連続作成フローを最適化しています。

## 2. 実装範囲
- **対象**: 新規カード作成（一問一答・連続作成モード）
- **UIベース**: `CardEditor` コンポーネント (共通コンポーネントの再利用)

## 3. 画面仕様

### 3.1 アクセス経路
- フォルダ詳細画面 (`FolderView`) ＞ 「新規カード」 ＞ 「連続作成モード」 ＞ 「一問一答」

### 3.2 画面構成 (UI)
- 基本的に `CardEditor` と同一のUI。
- **特徴的な差異 (スタック表示)**:
    1. **初期状態**: 1枚目のカード入力欄（初期テキストブロック付き）が表示される。
    2. **連続作成時**: 「続けて作成」を押すと、**作成済みのカードはその場に残り（保存済み状態）**、その下に**新しいカード入力欄が追加**される。
    3. **スクロール**: 新しいカードが追加されると、自動的にその位置までスムーズスクロールする。
    4. **編集可能性**: 保存済みの過去カードも画面上に残るため、必要に応じて再編集が可能（ただしフォーカスは最新のカードに移る）。

## 4. 操作プロセス (インタラクション)

### 4.1 カード作成フロー
1. 画面を開くと、Q1のカードエディタが表示される。
2. 問題・解答を入力し、**「続けて作成」**ボタンを押下。
    - Q1の内容が保存され、「保存済み」マーク等が表示される（編集自体は引き続き可能）。
    - 画面下部にQ2のカードエディタが新規追加される。
    - 画面がスクロールし、Q2の入力欄が表示される。
3. これを繰り返し、Q3, Q4... と縦に積み上げていく。
4. 終了時は最下部のカードで**「保存」**（またはキャンセル）を押して終了。
    - 途中のカードを再編集した場合、そのカードの「保存」ボタンを押すことで再保存が可能。

## 5. データ保存とバリデーション

- `useCards` フックの `createCard` を使用。
- Firestore および LocalDB (Dexie) へ保存。
- **空カードの保存制限**: 
    - 完全に内容が空（タイトル、タグ、各ブロックのすべてがなし）のカードは保存できません。
    - この制限により、一問一答モードで誤って空のエディタを追加したり保存したりすることを防止します。
- **下書き自動設定**: 問題または解答のいずれかが空の場合、自動的に「下書き」フラグがONの状態で保存されます。

## 6. 技術仕様
- **コンポーネント**: `Pages/OneQAMode.jsx`
    - `CardEditor` のラッパーとして機能。
    - 状態管理（エディタのリスト配列、保存状態の管理）を担当。
    - `useSearchParams` を使用して `?folderId=xxx` を取得。
- **Props**:
    - `showContinueButton={!isSaved}`: 保存済みカードには「続けて作成」を出さない（最新カードのみに出す）。
    - `defaultToTextBlock={true}`: 初期ロード時にテキストブロックを強制配置するフラグ。

## 7. 変更履歴
- **Rev.1**: リスト形式のUIとして設計。
- **Rev.2**: CardEditorベースの単票・連続作成UI（リセット式）へ変更。
- **Rev.3**: スタック形式（履歴を残しつつ下に追記）へ変更。
- **Rev.4**: クエリパラメータ (`folderId`) の取得方式を修正。
- **Rev.5**: 空カードの保存防止バリデーションを導入。
