# 数式ブロック（KaTeX対応）設計仕様書

**概要:** カードにKaTeX対応の数式ブロックを導入するための仕様。編集・レンダリング・バリデーション方針を定義します。

**重要ポイント:**
- 数式は専用ブロックとして管理し、LaTeX入力＋リアルタイムプレビューを提供します。
- 保存はLaTeXエラーがあっても許容し下書き扱いとすることでUXを優先します。
- KaTeXは遅延ロードしバンドル影響を低減します。

**作成日**: 2026-02-04  
**対象アプリ**: FlashCard Master  
**目的**: 学習用カードに数式ブロックを追加し、数学・理系科目の学習を強力にサポート

---

## 1. 数式ブロックの役割定義

### 1.1 基本方針

数式ブロックは**独立したブロック種別**として扱います。テキストブロックの拡張ではありません。

**理由**:
- 数式は「説明」ではなく**学習・暗記の主対象**
- 数式特有の編集体験（LaTeX入力、リアルタイムプレビュー、エラーハンドリング）が必要
- 既存の6ブロック（テキスト、コード、画像、音声、メモ、リンク）と同等の重要性

### 1.2 想定ユースケース

| ユースケース | 例 |
|------------|-----|
| 数学の公式暗記 | 問題: 二次方程式の解の公式は？<br>答え: `x = \frac{-b \pm \sqrt{b^2-4ac}}{2a}` |
| 物理の法則 | 問題: 運動エネルギーの式<br>答え: `E_k = \frac{1}{2}mv^2` |
| 化学式 | 問題: 硫酸の化学式<br>答え: `\ce{H2SO4}` (mhchem拡張) |
| 統計学 | 問題: 正規分布の確率密度関数<br>答え: `f(x) = \frac{1}{\sqrt{2\pi\sigma^2}}e^{-\frac{(x-\mu)^2}{2\sigma^2}}` |

### 1.3 既存ブロックとの住み分け

| ブロック種別 | 用途 | 数式ブロックとの違い |
|------------|------|---------------------|
| **テキスト** | 自然言語の説明・文章 | 数式は含まない |
| **コード** | プログラムコード | 構文強調が目的、数式レンダリングは不要 |
| **数式** | 数学的表現 | KaTeXによる美しいレンダリング、専用エディタ |
| **メモ** | 補足説明 | 学習対象外、数式は含まない |

> **重要**: Markdown数式（`$...$`, `$$...$$`）への依存はしません。数式は必ず専用ブロックで管理します。

## 2. データ構造

### 2.1 型定義

#### `types/index.ts` への追加

```typescript
/**
 * 数式ブロックのデータ構造
 */
export type MathBlockData = {
  latex: string;           // KaTeX準拠のLaTeX文字列
  displayMode: 'block' | 'inline'; // 表示モード
  note?: string;           // 数式の意味や補足（省略可）
};

/**
 * CardBlock型の拡張
 */
export type CardBlock = {
  id: string;
  type: 'text' | 'code' | 'image' | 'audio' | 'memo' | 'reference' | 'math'; // 'math' を追加
  content?: string;
  images?: UploadedImage[];
  audios?: Array<{ url: string; filename: string; order: number }>;
  code?: CodeBlockData;
  references?: ReferenceBlockData[];
  math?: MathBlockData; // 新規追加
  orderIndex: number;
};
```

### 2.2 データ設計の意図

| フィールド | 型 | 必須 | 説明 |
|-----------|-----|------|------|
| `latex` | `string` | ✅ | KaTeX準拠のLaTeX文字列。空文字列も許容（編集中） |
| `displayMode` | `'block' \| 'inline'` | ✅ | デフォルトは `'block'`。将来的にインライン数式も対応可能 |
| `note` | `string?` | ❌ | 数式の意味や補足。長文は推奨しない（メモブロックを使用） |

### 2.3 バリデーション方針

- **保存時**: LaTeXエラーがあっても保存可能（下書き状態として扱う）
- **表示時**: エラーがある数式は描画せず、エラーメッセージを表示
- **理由**: 編集途中の数式を保存できないとUXが悪化する

## 4. KaTeX利用時の注意点

### 4.1 パフォーマンス

| 項目 | 対策 |
|------|------|
| **レンダリング速度** | KaTeXは高速（MathJaxより10倍以上速い） |
| **バンドルサイズ** | 約300KB（gzip後 ~100KB）。必要なフォントのみ読み込み |
| **メモリ使用量** | 1カードあたり数式が10個以下なら問題なし |

**最適化方針**:
- 遅延ロード: 数式ブロックが存在する場合のみKaTeXを読み込む
- CDN利用: 初期バンドルサイズを削減

### 4.2 セキュリティ

| リスク | 対策 |
|--------|------|
| **XSS攻撃** | KaTeXは安全（HTMLエスケープ済み） |
| **無限ループ** | KaTeXは安全（タイムアウト機構あり） |
| **DoS攻撃** | 極端に長いLaTeX文字列を制限（最大10,000文字） |

**実装**:
```typescript
const MAX_LATEX_LENGTH = 10000;

const validateLatex = (latex: string): boolean => {
  if (latex.length > MAX_LATEX_LENGTH) {
    throw new Error('LaTeX文字列が長すぎます');
  }
  return true;
};
```

### 4.3 対応構文

**基本方針**: KaTeX公式のサポート範囲のみ
- 公式ドキュメント: https://katex.org/docs/supported.html
- 非対応の構文はエラーメッセージで明示

**よく使う構文**:
- 分数: `\frac{a}{b}`
- 平方根: `\sqrt{x}`, `\sqrt[n]{x}`
- 上付き・下付き: `x^2`, `x_i`
- ギリシャ文字: `\alpha`, `\beta`, `\gamma`
- 積分: `\int`, `\sum`, `\prod`
- 行列: `\begin{matrix}...\end{matrix}`

**拡張機能（オプション）**:
- 化学式: `mhchem` 拡張（`\ce{H2O}`）
- 色付け: `\textcolor{red}{x}`

## 7. 将来拡張の余地

### 7.1 インライン数式

現在は `displayMode: 'block'` のみ対応。将来的に以下を追加可能:

```typescript
displayMode: 'inline' // 文中に埋め込む数式
```

**ユースケース**: 「円周率は $\pi \approx 3.14$ である」のような表現

### 7.2 数式テンプレート

よく使う数式をテンプレートとして提供:

- 二次方程式の解の公式
- 三角関数の定義
- 微分・積分の基本公式
- 行列の基本形

### 7.3 数式検索

LaTeX文字列での検索機能:

```typescript
// 例: "\frac" を含む数式を検索
searchCards({ mathContains: "\\frac" })
```

### 7.4 WebView不要設計

現在の設計はすでにWebView不要（React + KaTeX）。将来的にネイティブアプリ化しても問題なし。
