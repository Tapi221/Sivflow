**概要:** 現行カード（Card）構造のフィールド一覧、同期メタデータ、SRS統計、移行ルールを整理した調査レポートです。

**重要ポイント:**

# 現行のカード構造調査レポート

**調査日**: 2026-02-02

**対象バージョン**: Phase 2 (Sync Engine V2 導入済み)

## 1. 概要
本アプリケーションにおける「カード（Flashcard）」は、学習コンテンツの本体に加え、同期用メタデータ、SRS（間隔反復）用統計情報を含む多機能なデータ構造を持っています。

## 2. データ定義 (`Card` 型)
`src/types/index.ts` に定義されている `Card` 型は、`BaseEntity` を継承し、以下のカテゴリのフィールドで構成されます。

### 2.1 同期・基本メタデータ (`BaseEntity` 由来)

| フィールド | 型 | 説明 |
| :--- | :--- | :--- |
| `id` | `string` | カードの一意な識別子。 |
| `userId` | `string` | 所有ユーザーのID。 |
| `createdAt` | `Date \| Timestamp` | 作成日時。 |
| `updatedAt` | `Date \| Timestamp` | 最終更新日時（差分同期に使用）。 |
| `deviceId` | `string` | 最後に編集した端末のID。 |
| `isDeleted` | `boolean` | 論理削除フラグ。 |

### 2.2 組織・表示関連

| フィールド | 型 | 説明 |
| :--- | :--- | :--- |
| `folderId` | `string` | 所属するフォルダのID。 |
| `orderIndex` | `number` | フォルダ内での並び順。 |
| `questionNumber` | `string` | 問題番号（手動入力用）。 |
| `title` | `string` | カードのタイトル（任意）。 |
| `isDraft` | `boolean` | 下書きフラグ。学習対象から除外される。 |
| `isBookmarked` | `boolean` | ブックマーク（お気に入り）フラグ。 |
| `isSilent` | `boolean` | 消音モード。フォルダ設定とは別にカード単位で指定可能。 |

### 2.3 コンテンツ（問題と解答）

問題（Question）と解答（Answer）で対照的なフィールドセットを持ちます。

| カテゴリ | 問題用フィールド | 解答用フィールド | 型 |
| :--- | :--- | :--- | :--- |
| **テキスト** | `questionText` | `answerText` | `string` |
| **画像** | `questionImages` | `answerImages` | `UploadedImage[]` |
| **音声** | `questionAudios` | `answerAudios` | `AudioData[]` |
| **コード** | `questionCode` | `answerCode` | `CodeBlockData` |
| **メモ** | `questionMemo` | `answerMemo` | `string` |
| **マーク** | `questionMarked` | `answerMarked` | `string` (内部処理用) |

### 2.4 SRS (復習アルゴリズム) 統計

| フィールド | 型 | 説明 |
| :--- | :--- | :--- |
| `memoryStability` | `number` | 定着度 (0.01 ～ 1.0)。間隔計算の基礎値。 |
| `currentLevel` | `number` | 習熟段階 (0 ～ 5 等)。 |
| `nextReviewDate` | `Date \| Timestamp` | 次回復習予定日。 |
| `lastReviewAt` | `Date \| Timestamp` | 最後に復習した日時。 |
| `reviewCount` | `number` | 通算の復習回数。 |
| `state` | `CardState` | `'PRE-LEARN'`, `'STABLE'`, `'DECAYING'` 等の状態。 |
| `lastSubjectiveScore` | `number` | 最後に選択した自己評価 (0:忘れた ～ 3:余裕)。 |

## 3. 正規化と互換性 (`normalizeCard`)
`src/utils/index.ts` の `normalizeCard` 関数により、以下の旧形式データや外部データが自動的に現行構造へ変換されます。

- **フィールド名エイリアス**:

    - `front`, `question`, `q` → `questionText`
    - `back`, `answer`, `a` → `answerText`

- **定着度の正規化**:

    - 旧形式（5～100日の日数指定）から新形式 (0～1) への自動計算。
    - 旧レベル (0～5) からの推定変換。

## 4. 特記事項
### データ復旧用フィールド (`_rescueRaw`)
大規模なデータ修復や同期エラーからの復旧時に、元の生データを一時的に保持するためのフィールドです。UIには表示されませんが、不整合が発生した際の「最後の砦」として機能します。

### 画像の取り扱い
`UploadedImage` 型として保持され、`localUrl` (Blob) と `remoteUrl` (Firebase Storage) を使い分けます。DB保存時にはセキュリティと容量の観点から `localUrl` は除去され、永続可能な `remoteUrl` のみが保持されるよう制限されています。
