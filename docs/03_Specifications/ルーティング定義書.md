# ルーティング定義書 (Routing Definitions)

## 1. 概要

本ドキュメントは、Flashcard Master アプリケーションにおけるURLルーティング、画面遷移、およびナビゲーションパラメータの仕様を定義します。

### 技術スタック
- **Library**: `react-router-dom` (v6)
- **Strategy**: Client-Side Routing (SPA)
- **Optimization**: Route-based Code Splitting (React.lazy + Suspense)

---

## 2. ルート定義一覧

| パス | コンポーネント | 遅延ロード | 説明 |
| :--- | :--- | :--- | :--- |

| `/FolderView` | `FolderView` | Yes | **フォルダ詳細**。特定のフォルダ内のカード一覧やサブフォルダを表示。<br>※ `id` パラメータが必須。 |

| `/CardView` | `CardView` | Yes | **カード詳細**。カード内容の閲覧専用モード。 |
| `/study` | `StudyMode` | Yes | **学習モード**。フラッシュカードの実践画面。 |

## 3. 詳細パラメータ仕様

主要な画面におけるクエリパラメータ（Query Parameters）の定義です。

### 3.1 フォルダ詳細 (`/FolderView`)

| パラメータ | 必須 | 型 | 説明 |
| :--- | :--- | :--- | :--- |
| `id` | **Yes** | UUID | 表示対象のフォルダID。 |

### 3.2 カード編集 (`/CardEdit`)

| パラメータ | 必須 | 型 | 説明 |
| :--- | :--- | :--- | :--- |
| `id` | No | UUID | 既存カードの編集時に指定。未指定の場合は**新規作成**モードとなる。 |
| `folderId` | No | UUID | 新規作成時、親となるフォルダを指定。 |

### 3.3 カード詳細 (`/CardView`)

| パラメータ | 必須 | 型 | 説明 |
| :--- | :--- | :--- | :--- |
| `id` | **Yes** | UUID | 表示対象のカードID。 |

### 3.4 学習モード (`/study`)

| パラメータ | 必須 | 型 | 説明 |
| :--- | :--- | :--- | :--- |
| `folderId` | No | UUID | 指定時、そのフォルダ配下のカードのみを学習対象とする。<br>未指定時は「今日の復習（全フォルダ対象）」となる。 |

## 4. 認証とガード (Authentication Guard)

### 4.1 認証フロー
本アプリは明示的な `/login` ルートを持ちません（URLとしては存在しない）。
アプリケーションルート (`AppContent`) にて認証状態を監視し、以下の挙動をとります。

1. **未認証状態**:
   - URLに関わらず `<LoginPage />` コンポーネントをレンダリングします。
   - 認証成功後、ユーザーは直前の画面（あるいはDashboard）に留まります。

2. **認証済み状態**:
   - 要求されたルートのコンポーネントを表示します。
   - 初期ロード時に `ProtectedRoute` がチェックを行い、データの読み込み待ち(`LoadingFallback`)を表示する場合があります。

### 4.2 開発用バイパス (Development Bypass)
開発およびテスト環境 (`import.meta.env.DEV === true`) に限り、URLパラメータによる認証バイパスが可能です。

- **使用法**: `?test_bypass=true` を付与
- **効果**: `currentUser` が null 判定であっても、強制的に保護されたルートへアクセスさせます。
- **用途**: E2Eテスト、認証基盤がオフライン時のUI確認。

## 5. レイアウト構造

全てのルート（ログイン画面を除く）は `Layout` コンポーネント配下でレンダリングされます。

```
<Layout>
  <Sidebar /> (PC only)
  <Header />
  <MainContent>
    <Outlet /> (各ページコンポーネント)
  </MainContent>
  <MobileNavigation /> (Mobile only)
</Layout>
```

- **SP/PC分岐**: CSS メディアクエリおよび条件付きレンダリングにより、デバイスサイズに応じたナビゲーションを提供します。
- **WorldMap**: サイドバーや特定のナビゲーションからのみアクセス可能で、モバイル版のボトムナビゲーションには含まれません。
