# ブロック型コンテンツ実装計画

カードの編集・表示を柔軟にするため、従来の固定フィールド形式から「ブロック型（CardBlock）」への移行準備として、データ構造の基盤を実装します。

## 概要
従来の `questionText`, `questionImages` などの固定フィールドを保持しつつ、新しく `questionBlocks`, `answerBlocks` を導入します。これにより、問題や回答の中でテキスト、画像、コードなどを自由な順序で複数回配置することが可能になります。

## ユーザーレビューが必要な項目
> [!IMPORTANT]
> - 本フェーズではデータ構造の定義と正規化ロジックの追加のみを行い、UIの実装は含まれません。
> - `Card` 型に `questionBlocks` と `answerBlocks` の2つの配列を追加します。
