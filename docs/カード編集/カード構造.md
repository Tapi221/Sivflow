# カード構造の仕様書

このドキュメントでは、FlashcardMasterにおける「カード（Card）」のデータ構造について記述します。

## 1. 概要
カードはシステムの最小単位であり、問題（Question）と回答（Answer）のペア、および学習状態を保持します。すべてのカードは `BaseEntity` を継承しており、同期と削除管理のためのメタデータを持ちます。

## 2. 共通メタデータ (BaseEntity)

すべてのエンティティに共通するフィールドです。

| フィールド名 | 型 | 説明 |
| :--- | :--- | :--- |
| `id` | `string` | 一意の識別子（UUID） |
| `userId` | `string` | 所有者のユーザーID |
| `createdAt` | `Date \| Timestamp` | 作成日時 |
| `updatedAt` | `Date \| Timestamp` | 最終更新日時 |
| `stats` | `UserStats` | 統計情報（オプション） |
| `isArchived` | `boolean` | アーカイブフラグ |
| `reviewCount` | `number` | 復習回数 |
| `lastReviewAt` | `Date \| Timestamp` | 最終復習日時 |
| `deviceId` | `string` | 最終編集端末の識別子 |
| `isDeleted` | `boolean` | ソフトデリートフラグ |
| `hasSyncConflict` | `boolean` | 競合発生フラグ（オプション） |

## 3. カード固有フィールド (Card)

### 3.1. 基本情報
| フィールド名 | 型 | 説明 |
| :--- | :--- | :--- |
| `folderId` | `string` | 所属するフォルダのID |
| `orderIndex` | `number` | フォルダ内での並び順 |
| `title` | `string` | 任意のタイトル（オプション） |
| `tags` | `string[]` | タグの配列（オプション） |
| `isDraft` | `boolean` | 下書きフラグ |
| `isBookmarked` | `boolean` | ブックマークフラグ（オプション） |

### 3.2. コンテンツ (問題・回答)

問題（Question）と回答（Answer）のそれぞれに対して、以下のフィールドが存在します。

#### レガシー構造（互換保持）

既存カードおよび外部データ互換のため、以下のフィールドは引き続き保持されます。

| カテゴリ | フィールド名 (問題側) | フィールド名 (回答側) | 説明 |
| :--- | :--- | :--- | :--- |
| テキスト | `questionText` | `answerText` | メインテキスト（LaTeX対応） |
| 画像 | `questionImages` | `answerImages` | `UploadedImage[]` 型の画像配列 |
| 音声 | `questionAudios` | `answerAudios` | 音声データのメタデータ配列 |
| コード | `questionCode` | `answerCode` | コードブロックデータ（オプション） |
| 備考 | `questionMemo` | `answerMemo` | ユーザーメモ |

#### ブロック型構造（主構造）

新しい編集方式では、以下のフィールドにコンテンツが保持されます。

| フィールド名 | 型 | 説明 |
| :--- | :--- | :--- |
| `questionBlocks` | `CardBlock[]` | 問題側のブロック配列 |
| `answerBlocks` | `CardBlock[]` | 回答側のブロック配列 |

**CardBlock 定義:**

```ts
type CardBlock = {
  id: string;
  type: 'text' | 'code' | 'image' | 'audio' | 'memo';
  content?: string;
  images?: UploadedImage[];
  audios?: Array<{ url: string; filename: string; order: number }>;
  code?: CodeBlockData;
  orderIndex: number;
};
```

## 4. ユーザー設定
ユーザーごとに編集モードを管理します。

| フィールド名 | 型 | 説明 |
| :--- | :--- | :--- |
| `cardEditorMode` | `'tab' | 'block'` | エディタの表示・編集モード |

## 5. 特殊フィールド
- `_rescueRaw`: データ復旧時に元の生データを一時的に保持するために使用されます。

## 6. 関連データ
- **CardRelation**: カード間の関係（前提、派生、関連）を管理します。
- **MapNodePosition**: マップ機能におけるカードの座標情報を保持します。
