# 下書き機能（作成中トグル）仕様書

## 概要
カード作成・編集画面に存在する「下書き保存（作成中）」トグルの動作仕様、およびシステム全体への影響範囲について定義します。
この機能は、コンテンツが未完成のカードや、一時的に学習対象から外したいカードを管理するために使用されます。

## データ構造
各カードドキュメント（Firestore/LocalDB）は `isDraft` (boolean) フィールドを持ちます。

- `true`: 下書き状態（作成中）
- `false`: 完成状態（学習可能）
- `undefined/null`: 完成状態とみなす（後方互換性のため）

## 動作仕様

### 1. カード編集画面 (`CardEditor.tsx`)
- **手動切り替え**:
    - ヘッダー部分のトグルスイッチで `isDraft` の有効/無効をユーザーが手動で切り替え可能です。
    - ONの場合、画面上に「作成中」というバッジが表示されます。
- **自動判定ロジック**:
    - 保存（「カードを保存する」ボタン押下）時に、以下の条件のいずれかに該当する場合、**強制的に `isDraft = true`** として保存されます。
        - 問題テキスト（Question Text）が空、かつ画像・音声・コードがない。
        - 解答テキスト（Answer Text）が空、かつ画像・音声・コードがない。
    - つまり、問題または解答のコンテンツが欠落しているカードは、ユーザーが「下書き」をOFFにしていても、自動的に下書き状態になります。
    - **設定による無効化**: 設定画面の「下書き自動判定」をOFFにすることで、この自動判定を無効にできます。その場合、コンテンツが空でもユーザーの選択通りの状態で保存されます。

### 2. 学習モード (`StudyMode.jsx`)
- **フィルタリング**:
    - 学習セッション開始時、`isDraft` が `true` のカードは、**学習対象から除外** されます。
    - これにより、作りかけのカードが学習に出題されることを防ぎます。

### 3. ダッシュボード・統計 (`Dashboard.jsx`, `Statistics.jsx`, `Calendar.jsx`)
- **集計からの除外**:
    - 「本日の復習」カウント、統計グラフ、カレンダーのヒートマップ等の集計対象から、下書きカードは除外されます。
    - ユーザーの学習進捗には影響を与えません。

### 4. フォルダ詳細画面 (`FolderView.jsx`)
- **表示**:
    - カードリスト上で、下書きカードは点線のボーダーで表示され、視覚的に区別されます。
    - 「作成中」バッジが表示されます。
- **一括操作**:
    - 複数の下書きカードを選択し、一括で「下書き解除（公開）」する機能が提供されています。（未実装の場合は将来実装予定機能）
    - 逆に、選択したカードを一括で「下書き」にする機能もあります。

## 目的別ユースケース
| 目的 | 操作 | 結果 |
|---|---|---|
| コンテンツを入力中に一時保存したい | トグルをONにして保存 | 学習に出題されず、いつでも編集再開可能。 |
| 問題文だけ入れて解答は後で書きたい | （入力して保存） | コンテンツ不足のため自動で「下書き」として保存される。 |
| 特定のカードを一時的に学習から外したい | 編集画面でトグルをONにする | 削除せずに学習ループから除外できる。 |
