# タグ選択UIの改善計画

## 概要
タグ入力を「入力メイン」から「選択メイン」のUIに改善します。既存のタグを簡単に選択できるようにしつつ、新しいタグの作成もスムーズに行えるデザインを目指します。

## ユーザーレビューが必要な項目
> [!IMPORTANT]
> - タグの表示領域：選択済みのタグと、選択可能なタグを明確に分けます。
> - デザイン：現在のプレミアムなカードエディタのスタイル（丸み、色使い）に合わせます。

## 変更内容

### [Component] タグ入力コンポーネントの改良
#### [MODIFY] [tag-input.tsx](file:///c:/FlashcardMaster/src/Components/ui/tag-input.tsx)
- **レイアウトの刷新**:
  - 選択済みのタグを表示するエリアを整理。
  - ポップオーバー内で「既存のタグから選択」セクションを強調表示。
  - タグがない場合のプレースホルダーや空の状態の文言を改善。
- **インタラクションの改善**:
  - クリックで即座に既存タグのリストを表示。
  - 新規作成が必要な場合のみ、入力フィールドを使用する流れを明確化。
- **デザインの調整**:
  - `lucide-react` のアイコンを活用し、視覚的に分かりやすく。
  - カラーパレットをアプリ全体（`primary-600` 等）と同期。

### [Component] カードエディタの調整
#### [MODIFY] [CardEditor.tsx](file:///c:/FlashcardMaster/src/Components/card/CardEditor.tsx)
- タグ入力セクションのラベルや余白を微調整し、新しい `TagInput` が自然に馴染むようにします。

## 検証計画

### 自動テスト（開発者による確認）
- タグの選択・解除が正常に動作すること。
- 新規タグの作成（カラー選択含む）が正常に動作し、DBに保存されること。
- ドラッグ＆ドロップによる並び替えが維持されていること。

### 手動検証
- モバイルおよびデスクトップでの操作感を確認。
- 既存タグが多い場合に、選択リストが使いやすいか確認。
