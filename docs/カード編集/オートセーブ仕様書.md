# オートセーブ（下書き保持）仕様書

## 1. 概要

カード編集画面におけるデータの紛失を防止するため、編集中の内容をブラウザの `localStorage` に自動的に保存し、再開時に復元する機能です。

## 2. 動作仕様

### 2.1. 自動保存 (Auto-Save)

- **トリガー**: `formData`（タイトル、問題、解答、タグ、各種設定）の内容が変更されるたびに実行されます。
- **デバウンス**: 変更から1秒経過した後に保存処理が行われます。
- **保存先**: ブラウザの `localStorage`。
- **保存キー**:
  - 新規作成時: `card-editor-draft-new`
  - 既存編集時: `card-editor-draft-{cardId}`

### 2.2. データ復元 (Restoration)

- **トリガー**: カード編集画面（新規・編集）が開かれた直後に実行されます。
- **動作**: 該当する保存キーの内容を確認し、存在すれば現在の入力データに反映します。
- **ユーザー通知**: 復元が成功した場合、画面上部に「編集中のデータを復元しました」というメッセージが約5秒間表示されます。

### 2.3. クリーンアップ (Clean-up)

- 下記の操作が行われた際、該当する `localStorage` のデータは即座に削除されます。
  - **保存成功**: 「カードを保存する」または「続けて作成する」ボタンで保存が完了した時。
  - **キャンセル**: 「キャンセルして戻る」ボタンで意図的に編集を終了した時。

## 3. 注意事項

- 本機能はブラウザ単位の保存です。異なる端末やブラウザ間では同期されません。
- あくまで不意のリロードやブラウザ終了に対する保護を目的としています。
