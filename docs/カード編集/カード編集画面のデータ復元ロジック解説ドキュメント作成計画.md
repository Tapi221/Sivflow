# カード編集画面のデータ復元ロジック解説ドキュメント作成計画

カード編集画面で「編集中のデータを復元しました」と表示される仕組みについて、技術的な詳細をまとめたドキュメントを作成します。

## 作成目的
ユーザーがカード編集画面において、どのようにデータが保護・復元されるのか、またそのUIがどのような条件で表示されるのかを明文化し、今後のメンテナンス性を向上させます。

## 構成
1. **概要**: オートセーブと復元機能の目的。
2. **保存ロジック**: `localStorage` への保存タイミングとキーの管理。
3. **復元ロジック**: 画面遷移時のチェックとデータのマージ。
4. **UI表示制御**: `isRestored` ステートによる通知バーの表示と自動消去。
5. **クリーンアップ**: データが削除されるタイミング。
