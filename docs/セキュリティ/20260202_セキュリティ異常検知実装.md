# 実装計画: セキュリティ異常検知ルールの実装

ユーザーが作成した `異常検知ルール仕様書.md` および `セキュリティイベントログ仕様書.md` に基づき、クライアントサイドでの不正アクセス検知・記録システムを実装します。

## 概要

SyncServiceV2 の動作を監視し、異常なパターン（Revoked端末からのアクセス、ロック奪取ループ、過剰な競合など）を検知してログを記録、およびリスクスコアに応じた対応（SafeMode等）を行うための基盤を作成します。

## User Review Required

> [!IMPORTANT]
> **クライアントサイド検知の限界**
> 仕様書には「Serverが100%発火源」とありますが、本実装はクライアントアプリ (`src`) 内での実装となります。
> クライアント自身が「自分は怪しい挙動をしている」と申告する形になるため、真に悪意のある改変済みクライアントに対しては無力です（ログを送らないため）。
> ただし、「バグによる暴走」や「古いキャッシュによる意図しないDoS攻撃」、「正規ユーザーの誤操作」を検知し、サーバーを守るための防壁としては有効です。
> ログの `source` フィールドを `client` に設定し、サーバーサイドログと区別します。

### 2. セキュリティ監視サービスの実装
**[NEW] [`src/services/logic/SecurityMonitor.ts`](file:///c:/FlashcardMaster/src/services/logic/SecurityMonitor.ts)**
- **役割**: セキュリティイベントの集約、異常検知ルールの評価、リスクスコア管理
- **機能**:
    - `logEvent(type, metadata)`: イベント記録
    - `evaluateRules(type)`: ルール判定（時間窓を用いた頻度スロットリング）
    - `calculateRiskScore()`: リスクスコア算出と減衰処理
    - `getSecurityStatus()`: 現在の `safeMode` 状態などを返却

### 3. SyncServiceV2 へのフック追加
**[`src/services/SyncServiceV2.ts`](file:///c:/FlashcardMaster/src/services/SyncServiceV2.ts)**
- 各所でのイベント発火を追加

    - ロック奪取時 -> `LOCK_CONTENTION_EXCESS` 判定用のイベント
    - `forceFullResync` (競合多発) -> `SYNC_CONFLICT_EXCESS` 判定用のイベント

### 4. Firestore連携（ログ送信）
- `SecurityMonitor` から `users/{userId}/securityLogs` への書き込み処理を実装
- ユーザーに通知すべき高リスク状態の検知
