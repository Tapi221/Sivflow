## 概要

クライアントから送信された `securityLogs` をトリガーに、サーバーサイドで異常検知・リスクスコア計算・自動対処（通知、2FA要求、強制ログアウト）を行うパイプラインを構築します。
※ 本プロジェクト環境 (`src` 配下) はクライアントサイドメインですが、`functions` ディレクトリが存在すると仮定、あるいはクライアントサイドでのシミュレーション実装が必要な場合を考慮しつつ、まずは設計書通りの Cloud Functions コードを準備します。

## Proposed Changes

### 1. Cloud Functions トリガー実装
**[NEW] [`functions/src/security/index.ts`](file:///c:/FlashcardMaster/functions/src/security/index.ts)**
- `onSecurityLogCreated`: Firestoreトリガー
- `detectAbnormalPatterns`: 過去ログ集計とルール判定
- `calculateRiskScore`: スコア計算と減衰処理
- `handleRiskActions`: スコアに基づくアクション実行 (UI通知, 2FA, Revoke)

### 2. クライアントサイド UI連携
**[`src/services/logic/SecurityMonitor.ts`](file:///c:/FlashcardMaster/src/services/logic/SecurityMonitor.ts)**
- Cloud Functions による対処結果（`deviceStatus` コレクションの更新など）を購読し、UIに反映するロジックを追加できないか検討（本計画ではまずサーバーロジック優先）

## 注意点
- 現在のワークスペースに `functions` ディレクトリが無い場合、新たに作成します。
- `SendGrid` や `FCM` などの外部サービス連携はモック実装とします（APIキー等がないため）。
