アラート通知と自動対処フロー設計

## 目的
* 高リスク状態のユーザーや端末に対して即座に対応
* ユーザーと管理者双方に適切な通知
* 被害拡大を防止（ログアウト、操作制限、メール通知など）

## 1. 通知対象とトリガー（実装完了）

| RiskScore | レベル | 対象 | 通知・対処内容 | 備考 |
| :--- | :--- | :--- | :--- | :--- |
| **0 - 20** | Info | なし | 通常動作 | - |
| **21 - 50** | Warning | ユーザー | **SecurityAlertBanner** 表示 (UI) | 「不審なアクティビティが検知されました」等の警告 |
| **51 - 80** | High | ユーザー | 次回ログイン時 2FA 要求 | `requires2FA = true` を設定 |
| **81 - 100** | Critical | ユーザー | **AccountLockedScreen** 表示 | 全画面ロック、操作不能化 |
| **81 - 100** | Critical | システム | 全デバイス Revoke | `sync_metadata` のステータスを `revoked` に変更 |

## 2. 実装コンポーネント

### クライアントサイド
*   **SecurityMonitor (`src/services/logic/SecurityMonitor.ts`)**: Firestore の `users/{userId}` をリアルタイム監視 (`onSnapshot`) し、`isAccountLocked` や `requires2FA` の変更を検知。
*   **AuthContext (`src/contexts/AuthContext.tsx`)**: アプリケーション全体のセキュリティ状態 (`securityState`) を管理。
*   **SecurityAlertBanner (`src/Components/security/SecurityAlertBanner.tsx`)**: Warning レベルのアラートを画面上部に表示。
*   **AccountLockedScreen (`src/Components/security/AccountLockedScreen.tsx`)**: アカウントロック時 (`isAccountLocked: true`) に最前面に表示され、一切の操作をブロック。

### バックエンド (Cloud Functions)
*   **Security Logic (`functions/src/security/index.ts`)**:
    *   `detectAbnormalPatterns`: セキュリティログから異常パターンを検知。
    *   `calculateRiskScore`: リスクスコアの計算と減衰処理。
    *   `handleRiskActions`: スコアに応じた自動対処 (通知作成、2FAフラグ設定、Revoke/Lock実行)。

## 3. 自動対処フロー

### 3-1. アカウントロック (Critical)
1.  **検知**: RiskScore が 81 以上に到達 (例: 管理者によるロック、複数回のクリティカルな違反)。
2.  **Server Action**:
    *   `users/{userId}` の `isAccountLocked` を `true` に設定。
    *   全アクティブデバイスのステータスを `revoked` に更新。
3.  **Client Reaction**:
    *   `SecurityMonitor` が `isAccountLocked: true` を検知。
    *   `AuthContext` の `securityState.isLocked` が `true` に更新。
    *   **AccountLockedScreen** が表示され、アプリ利用が停止。

### 3-2. 2FA要求 (High)
1.  **検知**: RiskScore が 51 以上。
2.  **Server Action**: `users/{userId}` の `requires2FA` を `true` に設定。
3.  **Client Reaction**: 次回ログイン時または重要な操作時に 2要素認証を要求 (今後の実装予定)。

### 3-3. 警告表示 (Warning)
1.  **検知**: RiskScore が 21 以上。
2.  **Server Action**: `users/{userId}/notifications` に警告タイプのアラートを追加。
3.  **Client Reaction**:
    *   `SecurityMonitor` (または通知リスナー) がアラートを検知。
    *   `AuthContext` の `securityState.alerts` に追加。
    *   **SecurityAlertBanner** が画面上部に表示。

## 4. 運用・検証
*   検証は `scripts/verify_security_signal.ts` (または `functions/src/tools`) を使用して、擬似的なセキュリティイベントを発生させ、ロック挙動を確認する。
*   手動での解除は Firestore コンソールから `users/{userId}` の `isAccountLocked` を `false` に戻すことで可能（リスクスコアのリセットも推奨）。
