# 数式ブロック（KaTeX）仕様書

## 概要

数式ブロックは、KaTeX ライブラリを使用して LaTeX 形式の数学記号や方程式を高品質にレンダリングするためのコンポーネントです。
パフォーマンスを重視した `renderToString` 方式と `useMemo` によるキャッシュ戦略を採用しています。

## 技術スタック

- **ライブラリ**: `katex` (npm package)
- **CSS**: `katex.min.css` (main.tsx にてグローバル読み込み)
- **レンダリング方式**: `renderToString` (Safe dangerouslySetInnerHTML)

## 主要コンポーネント: `MathRenderer`

### Props

| Prop | 型 | デフォルト | 説明 |
| :--- | :--- | :--- | :--- |
| `latex` | `string` | (必須) | レンダリングする LaTeX 文字列 |
| `displayMode` | `'block' \| 'inline'` | `'block'` | `'block'` は中央揃えの独立行、`'inline'` は文中表示 |
| `className` | `string` | `''` | カスタムスタイルクラス |

### 特徴

1. **パフォーマンス最適化**:
   - `React.memo` と `useMemo` により、LaTeX 文字列に変更がない限りの再計算・再レンダリングを完全に防止しています。
   - 大規模なカードセットや仮想リスト内での使用を想定した軽量設計です。

2. **堅牢なエラーハンドリング**:
   - `throwOnError: false` 設定により、不正な構文が入力されてもアプリがクラッシュしません。
   - エラー時は、入力された LaTeX 文字列をソースコード風（背景：薄赤、フォント：monospace）に表示し、編集を継続しやすくします。

3. **セキュリティ**:
   - `trust: false` に設定し、`\htmlClass` などの危険な LaTeX マクロを無効化しています。

4. **表示制限 (安全弁)**:
   - 5,000文字を超える極端に長い文字列は、パフォーマンス保護のためレンダリングを制限します。

## データの安全性と正規化

1.  **防御的アクセス**:
    - 全てのブロック描画および保存前のバリデーションにおいて、`latex` 属性へのアクセスはオプショナルチェイニング (`math?.latex`) を使用し、未定義エラーを防止する。
2.  **空ブロックの自動除外**:
    - `normalizeCard` ユーティリティにおいて、`latex.trim()` が空の数式ブロックは自動的にフィルタリングされ、データベースおよびUIから除外される。これにより、不要な余白やゴミデータの蓄積を防止する。

## 設計ルールと保守指針

1.  **責務の明確化**:
    -   数式の `display` / `inline` 判定は、YAML メタデータやブロックの種類といった**「構文レベル」**で決定し、Props として渡す。
    -   `MathRenderer` は描画に徹し、ロジカルな判定を混入させない。

2.  **CSS コンプライアンス**:
    -   **内部要素への介入禁止**: `.katex-html`, `.base`, `.strut` 等の KaTeX 内部要素に対して直接スタイルを当ててはならない。
    -   **配置制御**: 中央揃えや余白調整は、必ず `MathRenderer` またはその親のラッパー（`.katex-display-wrapper` 等）で行う。
    -   **オーバーフロー対策**: 数式が画面幅を超える場合は、折り返しを強制せず、`overflow-x: auto` による横スクロールで対応する。

## 開発者向けルール

- **KaTeX を直接呼ばない**: 全てのレンダリングは `MathRenderer` を通して行ってください。
- **HTML を保存しない**: データベースには LaTeX の生の文字列のみを保存してください。
