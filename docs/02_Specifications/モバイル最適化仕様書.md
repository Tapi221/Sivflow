# モバイル最適化仕様書

最新更新日時: 2026-02-05 05:25

本アプリをモバイルブラウザでネイティブアプリのように快適に利用するための最適化仕様について記述します。

## 1. レイアウトとスクロール

### 横揺れ防止

- 全てのページで `max-width: 100%` および `overflow-x: hidden` を適用し、意図しない横スクロール（横揺れ）を完全に防止しています。
- 長文テキストや複雑な数式も、画面幅に合わせて自動的に折り返されるか、数式ブロック内でのみ横スクロールが可能になるよう設定されています。

### タッチスクロールの滑らかさ

- `-webkit-overflow-scrolling: touch` を適用し、iOS/Android のブラウザでネイティブのような慣性スクロールを実現しています。


## 2. タッチ操作の最適化

### スワイプジェスチャ（学習画面）

- `react-swipeable` を導入し、カードの学習をスワイプ操作で行えるようにしました。
  - **左スワイプ**: 「覚えた」としてマーク
  - **右スワイプ**: 「忘れた」としてマーク
- スワイプ中にはカードが半透明になり、移動方向に合わせて傾く視覚的フィードフィードバックを提供します。

### タップターゲットの改善

- ボタンやリンクの最小サイズを `44px x 44px` 以上に確保し、指での操作ミスを軽減しています。
- タップ時の不快なハイライト（グレーの四角など）を `-webkit-tap-highlight-color: transparent` で無効化しています。


### 入力補助

- `inputmode` の最適化により、数値入力時には自動的に数字キーボードが表示されます。
- フォーカス時に画面が意図しない位置にジャンプするのを防ぐため、`scroll-margin-top` を調整しています。


## 3. 画面遷移とアニメーション

### SPA 遷移の滑らかさ

- `framer-motion` を使用し、ページ間の移動時に滑らかなスライド＆フェードアニメーションを挿入しています。
- フルリロードを伴わない SPA 遷移により、ネイティブアプリのような連続性を維持します。

### カードめくり

- カードの裏返し（Flip）アニメーションを 3D トランジションで実装し、現実のカードをめくるような体験を提供します。


## 4. PWA (Progressive Web App) 対応

### ホーム画面追加

- ホーム画面に追加することで、ブラウザのツールバーが表示されない「フルスクリーンモード（standalone）」で起動します。
- 専用のアイコンとテーマカラーを設定済みです。

### オフライン対応

- Service Worker をバックグラウンドで実行し、一度読み込んだ静的資産やアプリケーション本体をキャッシュします。
- データの保存には IndexedDB (Dexie) を使用しており、オフライン状態でもカードの閲覧や学習が可能です。


## 5. 高速化

### 遅延ロード (Lazy Loading)

- 各ページコンポーネントを `React.lazy` で分割し、必要な時にのみ読み込むことで初期表示速度を向上させています。
