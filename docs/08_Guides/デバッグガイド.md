## ブラウザDevTools

### 基本操作

#### DevToolsを開く
```
Windows/Linux: F12 または Ctrl+Shift+I
Mac: Cmd+Option+I
```

### Consoleタブ

#### エラーメッセージの確認
```javascript
// エラーが発生した場所を確認
// スタックトレースをクリックしてソースコードへジャンプ
```

#### ログの出力
```javascript
// デバッグ用のログ
console.log('[DEBUG] データ:', data);
console.error('[ERROR] エラー:', error);
console.warn('[WARN] 警告:', warning);
```

#### ログのフィルタリング
```
Console設定 → Log levels
- Verbose: すべて表示
- Info: 情報以上
- Warnings: 警告以上
- Errors: エラーのみ
```


**操作**:

## デバッグ手法

### 1. console.logデバッグ

#### 基本的な使い方
```typescript
function calculateStability(card: Card): number {
  console.log('[calculateStability] 入力:', card);
  
  const stability = card.reviewCount * 10;
  console.log('[calculateStability] 計算結果:', stability);
  
  return stability;
}
```

#### オブジェクトの詳細表示
```typescript
// ✅ 良い例
console.log('[DEBUG] カード:', JSON.stringify(card, null, 2));

// ❌ 悪い例
console.log('[DEBUG] カード:', card); // [object Object] と表示される
```

### 2. ブレークポイントデバッグ

#### VSCodeでのデバッグ
```json
// .vscode/launch.json
{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "chrome",
      "request": "launch",
      "name": "Launch Chrome",
      "url": "http://localhost:5173",
      "webRoot": "${workspaceFolder}/src"
    }
  ]
}
```

**使い方**:
1. VSCodeで行番号をクリック (ブレークポイント設定)
2. F5でデバッグ開始
3. ブレークポイントで停止
4. 変数の値を確認

### 3. エラーバウンダリ

#### エラーをキャッチ
```tsx
class ErrorBoundary extends React.Component {
  state = { hasError: false };
  
  static getDerivedStateFromError(error) {
    return { hasError: true };
  }
  
  componentDidCatch(error, errorInfo) {
    console.error('[ErrorBoundary]', error, errorInfo);
  }
  
  render() {
    if (this.state.hasError) {
      return <div>エラーが発生しました</div>;
    }
    return this.props.children;
  }
}

// 使用
<ErrorBoundary>
  <App />
</ErrorBoundary>
```

## よくあるバグパターン

### 1. データが表示されない

#### デバッグ手順
```typescript
// 1. データの取得を確認
useEffect(() => {
  console.log('[useEffect] データ取得開始');
  fetchData().then(data => {
    console.log('[useEffect] 取得データ:', data);
    setData(data);
  });
}, []);

// 2. 状態の更新を確認
console.log('[render] 現在のデータ:', data);

// 3. 条件分岐を確認
if (data.length === 0) {
  console.log('[render] データが空です');
  return <div>データがありません</div>;
}
```

### 2. 無限ループ

#### 原因
```typescript
// ❌ 悪い例
useEffect(() => {
  setCount(count + 1); // 無限ループ
}, [count]); // count が変わるたびに実行

// ✅ 良い例
useEffect(() => {
  setCount(c => c + 1); // 1回だけ実行
}, []); // 依存配列が空
```

#### デバッグ
```typescript
useEffect(() => {
  console.log('[useEffect] 実行回数:', ++executionCount);
  // 実行回数が異常に多い場合は無限ループ
}, [dependency]);
```

### 3. 画像が表示されない

#### デバッグ手順
```typescript
// 1. URLを確認
console.log('[Image] URL:', imageUrl);

// 2. 読み込みエラーを確認
<img 
  src={imageUrl}
  onLoad={() => console.log('[Image] 読み込み成功')}
  onError={(e) => console.error('[Image] 読み込み失敗:', e)}
/>

// 3. Networkタブで確認
// DevTools → Network → Img
// ステータスコードを確認 (200: 成功, 404: 見つからない, 403: 権限なし)
```

### 4. 同期が失敗する

#### デバッグ手順
```typescript
// 1. ネットワーク状態を確認
console.log('[Sync] オンライン:', navigator.onLine);

// 2. 同期処理を確認
try {
  await syncService.sync();
  console.log('[Sync] 同期成功');
} catch (error) {
  console.error('[Sync] 同期失敗:', error);
  console.error('[Sync] エラー詳細:', error.message);
}

// 3. Firestoreを確認
// Firebase Console → Firestore Database
// データが正しく保存されているか確認
```

## 🔍 デバッグのコツ

### 1. 問題を再現する
```
1. 問題が発生する手順を記録
2. 同じ手順で再現できるか確認
3. 再現できない場合は条件を変えて試す
```

### 2. 問題を分割する
```
1. 大きな問題を小さく分割
2. 1つずつ確認
3. 原因を特定
```

### 3. 仮説を立てる
```
1. 「〇〇が原因ではないか」と仮説を立てる
2. 仮説を検証
3. 正しければ修正、間違っていれば別の仮説
```

### 4. ログを活用する
```typescript
// 処理の開始と終了をログ
console.log('[Function] 開始');
// 処理
console.log('[Function] 終了');

// 条件分岐をログ
if (condition) {
  console.log('[Function] 条件A');
} else {
  console.log('[Function] 条件B');
}
```
