## 手動テスト

### 基本フロー

#### 1. ログイン
```
1. アプリを開く
2. Googleアカウントでログイン
3. ダッシュボードが表示されることを確認
```

**確認項目**:
- [ ] ログインボタンが表示される
- [ ] Googleログインが成功する
- [ ] ユーザー名が表示される
- [ ] プロフィール画像が表示される

#### 2. カード作成
```
1. 「新規カード」ボタンをクリック
2. 質問と答えを入力
3. 「保存」ボタンをクリック
4. カードが作成されることを確認
```

**確認項目**:
- [ ] 入力フォームが表示される
- [ ] 質問・答えが入力できる
- [ ] 保存が成功する
- [ ] カードリストに表示される

#### 3. 学習モード
```
1. ダッシュボードで「学習開始」をクリック
2. カードが表示されることを確認
3. 「答えを見る」をクリック
4. 自己評価を選択
5. 次のカードが表示されることを確認
```

**確認項目**:
- [ ] カードが表示される
- [ ] 答えが表示される
- [ ] 自己評価ボタンが動作する
- [ ] 次のカードに進む

#### 4. 統計確認
```
1. 統計画面を開く
2. グラフが表示されることを確認
3. データが正しいことを確認
```

**確認項目**:
- [ ] グラフが表示される
- [ ] データが正確
- [ ] フェーズ分布が表示される
- [ ] 平均定着度が表示される

### モバイルテスト

#### Chrome DevToolsでテスト
```
1. DevTools を開く (F12)
2. Toggle device toolbar (Ctrl+Shift+M)
3. デバイスを選択
4. 操作を確認
```

**テストデバイス**:
- [ ] iPhone SE (375x667)
- [ ] iPhone 12 Pro (390x844)
- [ ] iPad (768x1024)
- [ ] Galaxy S20 (360x800)

#### 確認項目
```
- [ ] レイアウトが崩れていない
- [ ] ボタンがタップできる
- [ ] テキストが読める
- [ ] 画像が表示される
- [ ] スクロールが正常
```

#### 実機でテスト
```
1. スマートフォンでアプリを開く
2. 基本フローを実行
3. 問題がないか確認
```

**確認項目**:
- [ ] タッチ操作が正常
- [ ] 表示速度が適切
- [ ] オフライン動作
- [ ] 同期が正常

## 自動テスト

### ユニットテスト

#### テストの実行
```bash
# すべてのテストを実行
npm run test

# 特定のファイルをテスト
npm run test -- src/utils/reviewUtils.test.ts

# ウォッチモード
npm run test -- --watch
```

#### テストの書き方
```typescript
// src/utils/reviewUtils.test.ts
import { calculateStability } from './reviewUtils';

describe('calculateStability', () => {
  it('復習回数0の場合、定着度は0', () => {
    const card = { reviewCount: 0 } as Card;
    expect(calculateStability(card)).toBe(0);
  });
  
  it('復習回数10の場合、定着度は100', () => {
    const card = { reviewCount: 10 } as Card;
    expect(calculateStability(card)).toBe(100);
  });
  
  it('復習回数5の場合、定着度は50', () => {
    const card = { reviewCount: 5 } as Card;
    expect(calculateStability(card)).toBe(50);
  });
});
```

## テストのベストプラクティス

### 1. テストは独立させる
```typescript
// ✅ 良い例
describe('calculateStability', () => {
  it('テスト1', () => {
    const card = { reviewCount: 0 };
    expect(calculateStability(card)).toBe(0);
  });
  
  it('テスト2', () => {
    const card = { reviewCount: 10 };
    expect(calculateStability(card)).toBe(100);
  });
});

// ❌ 悪い例 (テストが依存)
describe('calculateStability', () => {
  let card;
  
  it('テスト1', () => {
    card = { reviewCount: 0 };
    expect(calculateStability(card)).toBe(0);
  });
  
  it('テスト2', () => {
    card.reviewCount = 10; // テスト1に依存
    expect(calculateStability(card)).toBe(100);
  });
});
```

### 2. テストケースを明確に
```typescript
// ✅ 良い例
it('復習回数が0の場合、定着度は0を返す', () => {
  // ...
});

// ❌ 悪い例
it('テスト1', () => {
  // ...
});
```

### 3. エッジケースをテスト
```typescript
describe('calculateStability', () => {

    // ...
  });

    // ...
  });

    // ...
  });
  
  it('異常値: 復習回数が負', () => {
    // ...
  });

    // ...
  });
});
```
