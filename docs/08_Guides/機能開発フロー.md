# 機能開発フロー

**最終更新**: 2026-01-30

このガイドでは、新機能の開発から本番環境へのデプロイまでの標準的な流れを説明します。

---

## 📋 開発フロー概要

```
1. 仕様確認 (30分)
   ↓
2. 実装計画作成 (1時間)
   ↓
3. 実装 (可変)
   ↓
4. テスト (1時間)
   ↓
5. デプロイ (30分)
   ↓
6. ドキュメント更新 (30分)
```

## 3. 実装 (可変)

### 3.1 ブランチを作成

```bash
# mainブランチから最新を取得
git checkout main
git pull origin main

# 新しいブランチを作成
git checkout -b feature/your-feature-name
```

**ブランチ命名規則**:
- `feature/` - 新機能
- `fix/` - バグ修正
- `refactor/` - リファクタリング
- `docs/` - ドキュメント更新

### 3.2 コードを実装

#### 型定義から始める
```typescript
// src/types/index.ts
export interface NewFeature {
  id: string;
  name: string;
  createdAt: number;
}
```

#### サービス層を実装
```typescript
// src/services/NewFeatureService.ts
export class NewFeatureService {
  // ビジネスロジックを実装
}
```

#### UIコンポーネントを実装
```tsx
// src/Components/NewFeature.tsx
export const NewFeature: React.FC = () => {
  // UIを実装
};
```

### 3.3 コメントを日本語で記述

```typescript
// ✅ 良い例
/**
 * カードの定着度を計算する
 * @param card - 対象のカード
 * @returns 定着度 (0-100)
 */
function calculateStability(card: Card): number {
  // 復習回数に基づいて定着度を計算
  const baseStability = card.reviewCount * 10;
  return Math.min(100, baseStability);
}

// ❌ 悪い例
function calc(c) {
  return c.r * 10;
}
```

### 3.4 コミットを小さく保つ

```bash
# 機能ごとにコミット
git add src/types/index.ts
git commit -m "feat: add NewFeature type definition"

git add src/services/NewFeatureService.ts
git commit -m "feat: implement NewFeatureService"

git add src/Components/NewFeature.tsx
git commit -m "feat: create NewFeature component"
```

**コミットメッセージ規則**:
- `feat:` - 新機能
- `fix:` - バグ修正
- `refactor:` - リファクタリング
- `docs:` - ドキュメント更新
- `style:` - コードスタイルの変更
- `test:` - テストの追加・修正

## 🎯 ベストプラクティス

### コードの品質

1. **型安全性を保つ**
   ```typescript
   // ✅ 良い例
   function processCard(card: Card): void {
     // ...
   }
   
   // ❌ 悪い例
   function processCard(card: any): void {
     // ...
   }
   ```

2. **関数を小さく保つ**
   - 1つの関数は1つの責務
   - 50行以内を目安

3. **エラーハンドリングを忘れない**
   ```typescript
   try {
     await saveCard(card);
   } catch (error) {
     console.error('カードの保存に失敗:', error);
     notificationService.error('保存に失敗しました');
   }
   ```

### パフォーマンス

1. **不要な再レンダリングを避ける**
   ```tsx
   // useMemo, useCallback を活用
   const memoizedValue = useMemo(() => computeExpensiveValue(a, b), [a, b]);
   ```

2. **大きなリストは仮想化**
   ```tsx
   // react-window などを使用
   import { FixedSizeList } from 'react-window';
   ```

### セキュリティ

1. **ユーザー入力をサニタイズ**
   ```typescript
   const sanitizedInput = input.trim().slice(0, 100);
   ```

2. **認証状態を確認**
   ```typescript
   if (!user) {
     navigate('/login');
     return;
   }
   ```
