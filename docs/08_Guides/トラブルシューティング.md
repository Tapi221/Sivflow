## ビルドエラー

### TypeScriptエラー

#### 症状
```
Error: Type 'xxx' is not assignable to type 'yyy'
```

#### 原因
- 型定義の不一致
- 型ファイルの読み込み失敗

#### 解決方法
```bash
# 1. node_modulesを再インストール
npm install

# 2. VSCodeを再起動
# Ctrl+Shift+P → "Reload Window"

# 3. TypeScriptサーバーを再起動
# Ctrl+Shift+P → "TypeScript: Restart TS Server"
```

### Viteビルドエラー

#### 症状
```
Error: Failed to build
```

#### 原因
- キャッシュの問題
- 依存関係の不整合

#### 解決方法
```bash
# 1. キャッシュをクリア
rm -rf node_modules .vite dist
npm install

# 2. ビルドを再実行
npm run build
```

### 依存関係エラー

#### 症状
```
Error: Cannot find module 'xxx'
```

#### 原因
- パッケージがインストールされていない
- package.jsonとpackage-lock.jsonの不整合

#### 解決方法
```bash
# 1. package-lock.jsonを削除して再インストール
rm package-lock.json
npm install

# 2. 特定のパッケージを再インストール
npm install xxx --save
```

## 実行時エラー

### IndexedDBエラー

#### 症状
```
QuotaExceededError: The quota has been exceeded
```

#### 原因
- ブラウザのストレージ容量不足

#### 解決方法
1. **ブラウザのキャッシュをクリア**
   - Chrome: 設定 → プライバシーとセキュリティ → Cookie とサイトデータ
   - 不要なサイトのデータを削除

2. **IndexedDBを手動でクリア**
   - DevTools (F12) → Application → IndexedDB
   - flashcardDB を右クリック → Delete database

3. **ページをリロード**
   - Ctrl+Shift+R (ハードリロード)

#### 症状
```
Error: Database version cannot be lower than current version
```

#### 原因
- IndexedDBのバージョン不整合

#### 解決方法
```javascript
// DevToolsのConsoleで実行
indexedDB.deleteDatabase('flashcardDB');
// ページをリロード
```

### Firebase認証エラー

#### 症状
```
Error: auth/invalid-api-key
```

#### 原因
- `.env`ファイルの設定が間違っている
- Firebase設定が古い

#### 解決方法
1. **`.env`ファイルを確認**
   ```
   VITE_FIREBASE_API_KEY=your_api_key
   VITE_FIREBASE_AUTH_DOMAIN=your_auth_domain
   VITE_FIREBASE_PROJECT_ID=your_project_id
   ```

2. **Firebase Consoleで設定を確認**
   - https://console.firebase.google.com/
   - プロジェクト設定 → 全般 → マイアプリ

3. **環境変数を再読み込み**
   ```bash
   # 開発サーバーを再起動
   npm run dev
   ```

#### 症状
```
Error: auth/network-request-failed
```

#### 原因
- ネットワーク接続の問題
- Firebaseサービスのダウン

#### 解決方法
1. **ネットワーク接続を確認**
2. **Firebase Status を確認**
   - https://status.firebase.google.com/
3. **しばらく待ってから再試行**

### 画像表示エラー

#### 症状
- 画像が表示されない
- 壊れた画像アイコンが表示される

#### 原因
- Firebase Storageの権限設定
- CORS設定の問題
- URLの形式が間違っている

#### 解決方法
1. **Firebase Storageの権限を確認**
   ```
   Firebase Console → Storage → Rules
   
   service firebase.storage {
     match /b/{bucket}/o {
       match /{allPaths=**} {
         allow read: if request.auth != null;
         allow write: if request.auth != null;
       }
     }
   }
   ```

2. **CORS設定を確認**
   ```bash
   # cors.json を作成
   [
     {
       "origin": ["*"],
       "method": ["GET"],
       "maxAgeSeconds": 3600
     }
   ]
   
   # CORSを設定
   gsutil cors set cors.json gs://your-bucket-name
   ```

3. **URLの形式を確認**
   - Blob URLではなく、Firebase Storage URLを使用
   - 例: `https://firebasestorage.googleapis.com/...`

## デプロイエラー

### Firebase Hostingエラー

#### 症状
```
Error: HTTP Error: 403, Permission denied
```

#### 原因
- Firebase CLIの認証が切れている
- プロジェクトIDが間違っている

#### 解決方法
```bash
# 1. Firebaseに再ログイン
firebase logout
firebase login

# 2. プロジェクトIDを確認
firebase projects:list

# 3. プロジェクトを選択
firebase use your-project-id

# 4. デプロイを再実行
firebase deploy --only hosting
```

#### 症状
```
Error: Failed to list Firebase projects
```

#### 原因
- ネットワークの問題
- Firebase CLIのバージョンが古い

#### 解決方法
```bash
# 1. Firebase CLIを更新
npm install -g firebase-tools

# 2. 再ログイン
firebase login

# 3. デプロイを再実行
firebase deploy --only hosting
```

## データ関連の問題

### データが表示されない

#### 症状
- カードが表示されない
- フォルダが表示されない
- 統計が表示されない

#### 原因
- IndexedDBにデータがない
- Firestoreとの同期が失敗
- データの形式が間違っている

#### 解決方法
1. **IndexedDBを確認**
   ```
   DevTools (F12) → Application → IndexedDB → flashcardDB
   - cards テーブルにデータがあるか確認
   - folders テーブルにデータがあるか確認
   ```

2. **Firestoreを確認**
   ```
   Firebase Console → Firestore Database
   - users/{userId}/cards にデータがあるか確認
   ```

3. **同期を手動で実行**
   ```
   設定画面 → 同期設定 → 手動同期
   ```

### データが同期されない

#### 症状
- デバイス間でデータが同期されない
- オフラインで作成したデータが同期されない

#### 原因
- ネットワーク接続の問題
- 同期サービスのエラー
- 競合の発生

#### 解決方法
1. **ネットワーク接続を確認**
   - StatusBarで「オンライン」になっているか確認

2. **同期ログを確認**
   ```
   DevTools (F12) → Console
   - 同期関連のエラーメッセージを確認
   ```

3. **手動同期を実行**
   ```
   設定画面 → 同期設定 → 手動同期
   ```

4. **最終手段: データを再同期**
   ```
   設定画面 → 同期設定 → クラウドから復元
   ```

## UI表示の問題

### レイアウトが崩れる

#### 症状
- モバイル表示で要素が重なる
- ボタンが見切れる
- テキストがはみ出る

#### 原因
- Tailwind CSSのクラスが間違っている
- レスポンシブ対応が不足

#### 解決方法
1. **DevToolsでモバイル表示を確認**
   ```
   DevTools (F12) → Toggle device toolbar (Ctrl+Shift+M)
   ```

2. **Tailwind CSSのレスポンシブクラスを確認**
   ```jsx
   // 悪い例
   <div className="w-full">
   
   // 良い例
   <div className="w-full md:w-1/2">
   ```

3. **paddingとmarginを調整**
   ```jsx
   // モバイル: pb-24, デスクトップ: pb-10
   <main className="pb-24 md:pb-10">
   ```

### ダークモードが正しく動作しない

#### 症状
- ダークモードで文字が見えない
- 背景色が変わらない

#### 原因
- Tailwind CSSのdarkクラスが不足
- カスタムCSSでdarkモードが考慮されていない

#### 解決方法
```jsx
// 悪い例
<div className="bg-white text-black">

// 良い例
<div className="bg-white dark:bg-slate-900 text-black dark:text-white">
```
