# 本番環境へのデプロイ手順書

このドキュメントでは、`FlashcardMaster` アプリケーションを本番環境（Firebase Hosting）にデプロイする手順について説明します。

## 前提条件

- [Node.js](https://nodejs.org/) (バージョン 18 以上を推奨) がインストールされていること。
- [Firebase CLI](https://firebase.google.com/docs/cli) がインストールされ、Googleアカウントでログイン済みであること (`firebase login`)。
- プロジェクトの書き込み権限を持つ Firebase プロジェクトにアクセスできること。

## デプロイ手順

デプロイ作業は、ローカルの開発環境から行います。

### 1. 最新のソースコードを取得する

デプロイする前に、バージョン管理システム（Gitなど）から最新のソースコードを取得してください。

```bash
git pull origin main
```
*(ブランチ名はプロジェクトに合わせて変更してください)*

### 2. 依存パッケージをインストールする

プロジェクトのルートディレクトリで、必要な npm パッケージをインストールします。

```bash
npm install
```

### 3. アプリケーションをビルドする

Vite を使用して、本番用の静的ファイルを生成します。このコマンドは、プロジェクトのルートディレクトリに `dist` フォルダを作成します。

```bash
npm run build
```

`package.json` の `scripts` セクションで `"build": "vite build"` として定義されています。
`firebase.json` の `hosting.public` 設定により、この `dist` ディレクトリがホスティングの対象となります。

### 4. Firebase Hosting にデプロイする

生成された静的ファイルを Firebase Hosting にアップロードします。

```bash
firebase deploy --only hosting
```

`--only hosting` フラグは、Cloud Functions や Firestore Rules など、他のFirebaseサービスをデプロイ対象から除外し、Hosting のみデプロイすることを保証します。

### 5. デプロイの確認

デプロイが完了すると、CLI にホスティング先のURLが表示されます。そのURLにアクセスし、アプリケーションが正しく表示・動作することを確認してください。

## 注意事項

- デプロイを行う前には、必ずローカル環境でビルドとプレビュー (`npm run preview`) が成功することを確認してください。
- 重大な変更をデプロイする場合は、事前にステージング環境など、本番に近い環境でテストを行うことを推奨します。
