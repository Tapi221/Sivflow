# 一問一答モード 改修計画 (スタック表示対応)

## 概要
「続けて作成」ボタン押下時に、入力欄をリセットするのではなく、新しいカード入力欄を下に追加する形式に変更します。 これにより、作成済みのカードを確認しながら連続で作業できるようになります。

## 実装内容
### 1. OneQAMode.jsx の再実装 (リスト化)
- **状態管理:**
  - `editors` ステート: 各カードエディタの状態を管理する配列。
  - 構造: `{ id: number, initialData: object | null, isSaved: boolean }`
- **レンダリング:**
  - `editors.map` で `CardEditor` をリスト表示。
- **Props制御:**
  - `key`: エディタID。
  - `questionNumber`: `index + 1`。
  - `isReadOnly` (または類似の制御): 保存済みカードを編集不可にするか、あるいは編集可能なままにするか検討。
    → 要望は「元のカードも表示されたまま」なので、編集可能なまま残すが、フォーカスは新しいカードに移るようにする。
  - `showContinueButton`: 最後のカードのみ `true` にする。
- **イベントハンドリング (onSave):**
  - 保存処理成功時:
    - 対象エディタの `isSaved` フラグを `true` に更新。
  - `continueCreating` が `true` の場合:
    - `editors` 配列に新規エディタオブジェクトを追加。
    - 画面最下部へスクロール。

### 2. スクロール制御
- 新規カード追加時 (`useEffect` on `editors.length`) に、`window.scrollTo` または `scrollIntoView` でスムーズに末尾へ移動。

## 検証計画
### 手動テスト
- **初期表示:** 1枚目のカード入力欄が表示される。
- **連続作成:**
  - 1枚目を入力し「続けて作成」を押下。
  - 1枚目がそのまま表示され続け、その下に2枚目の入力欄が出現すること。
  - 自動的にスクロールされ、2枚目の入力欄が見えること。
  - ヘッダーのカード番号が「Q2」になっていること。
- **保存:**
  - 最後のカードで「保存」を押すと、フォルダ画面へ戻ること。
