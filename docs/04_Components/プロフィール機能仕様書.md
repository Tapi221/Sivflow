# プロフィール機能詳細仕様書

## 1. 概要
ユーザーのアイデンティティを管理・表示するための機能群。
ユーザー名のカスタマイズ、プロフィール画像のアップロード・管理、およびそれらのデータの整合性を維持するための自動修復メカニズムを含む。

## 2. データ構造

### 2.1 UserSettings (IndexedDB & Firestore)
ユーザー設定の一部として以下のフィールドで管理される。

| フィールド | 型 | 説明 | 制約・備考 |
|---|---|---|---|
| `displayName` | `string` | 表示用ユーザー名 | 1〜20文字、許可文字制限あり |
| `profileImage` | `UploadedImage \| null` | プロフィール画像オブジェクト | 画像未設定時は `null` |

### 2.2 UploadedImage 構造
`src/types/index.ts` で定義される画像管理の標準構造。

| プロパティ | 型 | 説明 |
|---|---|---|
| `id` | `string` | 画像ID (UUID) |
| `localUrl` | `string?` | クライアントローカルの一時表示用 URL (`blob:...`) |
| `remoteUrl` | `string?` | Firebase Storage の永続化 URL (`https:...`) |
| `status` | `'uploading' \| 'ready' \| 'failed'` | アップロード状態 |
| `storagePath` | `string?` | Storage上のパス (`profile-images/{uid}/{filename}`) |
| `source` | `'cloud' \| 'local_fallback'` | データソース（オフライン時は `local_fallback`） |

## 3. 機能詳細

### 3.1 ユーザー名編集
**仕様**:
- **インライン編集**: 設定ダイアログ内で直接編集可能。
- **バリデーション (`src/utils/userValidation.ts`)**:
    - **文字数**: 1文字以上、20文字以内（Unicode文字数カウント）
    - **禁止文字**: 制御文字等は不可。
    - **許可文字**: 日本語（ひらがな・カタカナ・漢字）、英数字、スペース、ハイフン、アンダースコア。
- **表示**: サイドバー等では長すぎる場合「...」で省略表示される。

### 3.2 プロフィール画像アップロード
**仕様**:
- **対応フォーマット**: JPG, PNG, HEIC (iPhone写真)
- **ファイルサイズ制限**: 10MB (クライアント側でチェック推奨)
- **処理フロー (`src/utils/imageUtils.ts`)**:
    1. **選択**: ユーザーがファイルを選択。
    2. **HEIC変換**: `image/heic` の場合、`heic2any` ライブラリを用いて JPEG に変換。
    3. **プレビュー**: `URL.createObjectURL` で即時に `localUrl` を生成し表示 (UIブロックなし)。
    4. **アップロード**: バックグラウンドで Firebase Storage へアップロード。
    5. **同期**: 完了後、`remoteUrl` を Firestore / IndexedDB に保存。

**UI/UX**:
- 画像ホバー時にカメラ型アイコンをオーバーレイ表示。
- アップロード中はスピナーアニメーションを表示。
- 失敗時は「！」アイコンとともにエラー状態を表示し、ローカルURLでの表示を維持（フォールバック）。

### 3.3 自動修復 (Self-Healing)
**仕様 (`src/hooks/useProfileImageMonitor.ts`)**:
- **目的**: リンク切れ（404/403）画像の自動検出と復旧。
- **トリガー**: アプリ起動時、または定期的に（1時間に1回）チェック。
- **ロジック**:
    1. `remoteUrl` に HEAD リクエストを送信。
    2. ステータスが OK なら健全とみなす。
    3. 404/403 エラーの場合、`localUrl` (Blob) が存在すれば再アップロードを試行。
    4. 再アップロード成功時、新しい URL で設定を更新。
    5. ローカルバックアップもない場合は `status: 'failed'` に変更し、デフォルトアバターに戻す。

### 3.4 アバター表示
- **カスタム画像**: `profileImage.status !== 'failed'` の場合に表示。
- **デフォルト（イニシャル）**: 画像未設定またはロード失敗時。
    - **色生成**: ユーザー名から一意のハッシュで背景色を決定 (`getAvatarColors`)。
    - **文字**: ユーザー名の先頭文字を表示。

## 4. UIコンポーネント
- **`SettingsDialog.jsx`**: 編集のメインインターフェース。
- **`Layout.tsx`**: サイドバー下部へのアイコン表示（Tooltip付き）。

## 5. エラーハンドリング
- **ネットワークエラー**: アップロード失敗時は `status: 'failed'` とし、再試行ボタンを提供（または次回の自動修復に委ねる）。
- **非対応フォーマット**: ファイル選択時にフィルタリングし、UIで警告。
