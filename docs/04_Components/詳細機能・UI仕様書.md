# 詳細機能・UIコンポーネント仕様書

## 概要
メイン画面の背後で動作する重要な補助機能や、特定のアクションによって呼び出されるモーダルUIの詳細仕様です。

## 対象機能
1. **同期コンフリクト解決 (Sync Conflict Resolution)**
2. **高度なタグ管理 (Tag Manager)**
3. **数式・コードレンダリング (Rich Text Rendering)**

---

## 1. 同期コンフリクト解決 (`ConflictResolutionDialog`)

複数デバイス間でデータに矛盾が生じた際、ユーザーに解決を求めるダイアログです。

### 表示フロー
- **起動条件**: 同期プロセス (`SyncService`) が競合 (`conflict`) を検知した後に自動的に表示されます。
- **表示内容**:
  - **エンティティ情報**: カード/フォルダのID。
  - **自動マージ済み**: 競合していないフィールドは自動的に統合され、その結果が表示されます（緑色）。
  - **競合フィールド**: 差異があるフィールドのみを左右（Local vs Remote）対比で表示。

### 解決アクション
ユーザーは以下の3つの方法で解決できます。
1. **全てローカル (Resolve All Local)**: 全フィールドで自分の端末のデータを採用。
2. **全てクラウド (Resolve All Remote)**: 全フィールドでサーバー上のデータを採用。
3. **選択内容で解決 (Field-by-Field)**: フィールドごとに「Local」または「Remote」をラジオボタンで選択してマージ。

## 2. 高度なタグ管理 (`TagManagerDialog`)

設定画面の簡易リストとは別に、全タグを俯瞰・編集するための専用ダイアログです。

### 構成
- **フォルダ別グルーピング**: カードの使用状況に基づき、タグをフォルダ単位でグループ化して表示します（未分類は `UNKNOWN`）。
- **フィルタ**: 左上のドロップダウンで特定のフォルダのタグのみに絞り込み可能。

### 編集機能
- **カラー変更**: 全8色のパレットからタグの色を変更可能。この変更はルートフォルダ単位で保存され、配下のすべてのカードに反映されます。
- **UI**: モダンなカード型レイアウトを採用しており、ホバー時にカラーピッカーが展開されます。

## 3. レンダリングエンジン (`MathRender` / `CodeRenderer`)

カードのテキストエリアに入力された特殊記法を視覚的に変換する仕組みです。

### 数式レンダリング (`MathRender.tsx`)
- **エンジン**: `KaTeX` (CDNロード)。
- **対応デリミタ**:
  - インライン: `$ ... $` または `\( ... \)`
  - ブロック: `$$ ... $$` または `\[ ... \]`
- **挙動**: DOMレンダリング時に動的に変換を実行。エラー時は生のテキストを表示（`throwOnError: false`）。

### コードハイライト (`CodeBlockEditor` / `CodeRenderer`)
- **エンジン**: `prism-react-renderer`.
- **機能**:
  - シンタックスハイライト（言語指定可）。
  - 編集モードではタブキーによるインデント入力などをサポート。

### 3. カードリスト空状態 (`CardList` Empty State)
フォルダ内にカードが存在しない場合、以下の構成で「Ghost Grid」を表示します。
- **構成**:
  1. **新規作成カード**: 破線ボーダー (`border-dashed`), 中央に「＋」アイコン。クリックでエディタ起動。
  2. **Ghost Cards**: スケルトン表示されたカードのシルエット（不透明度 0.3〜0.5）。
- **目的**: 初回利用者が「ここにカードが並ぶ」ことを直感的に理解できるようにする。

## 📱 モバイル UX 仕様

### 1. 全画面ダイアログ
設定画面 (`SettingsDialog`) 等の主要ダイアログは、モバイル (`md`未満) では以下のスタイルを適用します。
- `h-[100dvh]`, `w-[100vw]`, `rounded-none`, `max-h-none`
- ネイティブアプリのような全画面体験を提供し、誤入力を防ぐためスクロールをダイアログ内に閉じ込める。

### 2. モバイルヘッダー・ナビゲーション
- **Streak位置**: 画面右上の設定アイコン（Settings）の直下にフローティング表示。
- **ボトムナビ**: 主要5メニュー（学習、フォルダ、マップ、予定表、統計）をタブ形式で配置。

### 2. タッチデバイス最適化
- **Tooltip**: ホバー不可能なモバイル環境では、`FolderTree` 等のカスタムツールチップは原則 `hidden`。
- **ドラッグハンドル**: フォルダツリーでは、誤操作防止のため「編集モード」時のみ `Grip` アイコンを表示。
- **スクロール**: 横スクロールトレイ（`HandTray`等）は `scrollbar-hide` を適用。
