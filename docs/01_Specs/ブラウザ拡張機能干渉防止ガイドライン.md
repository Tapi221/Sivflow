# 外部拡張機能・ツールへの干渉防止ガイドライン

本アプリケーションでは、ユーザーの入力体験を妨げないよう、ブラウザ拡張機能（パスワードマネージャー、AIアシスタント等）の自動介入を制御しています。

## 背景
パスワードマネージャーや入力補助ツールは、特定のキーワード（「名前」「ログイン」「電子メール」など）を検知すると、入力欄の横にアイコンを表示したり、自動補完を試みたりします。
これが通常の入力フォーム（リンクの表示名など）で発生すると、UIが崩れたり、誤入力の原因となるため、明示的に無視する設定を推奨します。

## 実装方法

入力欄（`input` 要素など）に以下の属性を付与することで、主要なツールの自動介入を回避できます。

### 基本的なHTML属性
- `autoComplete="off"`: ブラウザ標準の自動補完を停止。
- `spellCheck={false}`: OSやブラウザによる赤い波線などのスペルチェックを無効化。

### 特定ツール用の独自属性
| 属性 | 対象ツール |
| :--- | :--- |
| `data-lpignore="true"` | LastPass |
| `data-1p-ignore` | 1Password |
| `data-form-type="other"` | Dashlane 等の一般的なフォーム解析 |

### 実装例 (React/Tailwind)

```tsx
<Input 
  placeholder="表示名"
  autoComplete="off"
  spellCheck={false}
  data-lpignore="true"
  data-1p-ignore
  data-form-type="other"
/>
```

## 適用箇所
- カード作成画面のリンクブロック（URL、表示名）
- その他、パスワードや機密情報ではないが拡張機能が反応しやすいテキスト入力欄
