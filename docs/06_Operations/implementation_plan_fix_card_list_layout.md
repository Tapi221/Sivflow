# カード一覧のレイアウト修正

ユーザーからの報告（「境界いっぱいに内容が表示されるようにしてください」という要望と添付画像）に基づき、カード一覧画面におけるカード（`CardItem`）の表示崩れを修正します。
現状、カードの幅が意図せず縮小されている、またはパディングが大きすぎて内容が圧迫されている可能性があります。

## 変更内容

### 1. `CardItem` のスタイル調整
- **対象ファイル**: `src/Components/card/CardList.tsx`
- **変更点**:
    - `CardItem` のルート要素に `w-full` を追加し、Gridセル内で最大幅を確保するようにします。
    - `text-left` を明示的に追加し、予期せぬ継承によるセンタリングを防止します。
    - パディングを `p-6` (24px) から `p-5` (20px) にわずかに縮小し、可視領域を広げます。
    - `flex-col` に加えて `items-stretch` (デフォルトですが明示) を確認します。

### 2. ローディングスケルトン（Ghost Card）のスタイル同期
- **対象ファイル**: `src/Components/card/CardList.tsx`
- **変更点**:
    - `CardItem` の変更に合わせて、プレースホルダー用のゴーストカードのパディングも `p-6` から `p-5` に変更します。

## 検証計画

### 手動検証
1. **フォルダ画面（カード一覧）** を開きます。
2. カードの表示を確認します。
   - 「0%」などのステータス表示が左端、アイコンが右端に配置されていること（`justify-between` が正しく効いていること）。
   - タイトルと本文がカードの幅いっぱいに（パディングを除いて）表示されていること。
   - テキストが左寄せになっていること。
3. ドラッグ＆ドロップ機能（並び替え）が正常に動作し、ドラッグ中も表示が崩れないことを確認します。

## 実行手順
1. `src/Components/card/CardList.tsx` を修正。
