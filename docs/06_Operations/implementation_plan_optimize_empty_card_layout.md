# 空のカード編集時のレイアウト最適化計画

## 概要
現在、ブロックを追加していない状態（またはブロック数が少ない状態）のカード編集画面において、以下の問題が発生しています。
1. カードの高さが左右でズレる（コンテンツ量に応じた高さになっていない、または強制的な最小高さが影響している）。
2. ブロックがない状態のプレースホルダーが大きく、無駄なスペースを取っている。

これらを解決し、ブロックが少ない・ない状態でもコンパクトに（縮小して）表示できるように修正します。

## 変更内容

### 1. CardShell の最小高さ設定の変更 (`src/index.css`)
`.card-shell--resizable` クラスにおいて、高さ指定がない場合（`--card-resize-height` 未定義時）のデフォルト `min-height` が `calc(var(--card-w) * 3 / 4)`（4:3比率、約480px）になっています。
これを撤廃し、より小さい固定値（例: `120px`）または `auto` に変更することで、コンテンツが少ない場合にカードが縮小できるようにします。

**変更前:**
```css
min-height: var(--card-resize-height, calc(var(--card-w) * 3 / 4));
```

**変更後:**
```css
/* デフォルトのアスペクト比強制を削除し、コンテンツに合わせて縮小可能にする */
min-height: var(--card-resize-height, 144px); 
```

### 2. BlockEditor プレースホルダーの縮小 (`src/Components/card/BlockEditor.tsx`)
「ブロックを追加してください」というプレースホルダーの上下パディング (`py-12`) が大きく、これが最小高さを押し上げる原因の一つになっています。
これを縮小 (`py-6`) し、見た目をコンパクトにします。

**変更前:**
```tsx
<div className="text-center py-12 ...">
```

**変更後:**
```tsx
<div className="text-center py-6 ...">
```

## 検証計画

### 動作確認
1. **新規カード作成**:
    - 新規カード作成画面を開く（初期状態でブロックなし、またはテキストブロック1つ）。
    - 左右のカード高さが、巨大な空白を持たずにコンパクトに表示されていることを確認する。
    - 「ブロックを追加してください」のプレースホルダーが以前より薄く（背が低く）なっていることを確認する。

2. **ブロック追加・削除**:
    - ブロックを追加していき、高さが自然に伸びることを確認する。
    - ブロックを削除して0個にしたとき、再度コンパクトなプレースホルダーに戻ることを確認する。

3. **リサイズ機能**:
    - カード下部のリサイズハンドルをドラッグして、手動での高さ変更が引き続き機能することを確認する。

4. **モバイル表示**:
    - モバイル幅でもレイアウトが崩れていないか確認する。
