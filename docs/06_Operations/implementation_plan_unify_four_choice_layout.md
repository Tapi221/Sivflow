# 4択モードのレイアウト統一 実装計画

4択モード（`FourChoiceMode.jsx`）のデザインと操作性を、一問一答モード（`OneQAMode.jsx`）と統一します。主な変更点は、単一のエディタ表示から、複数のカードを並べて作成できるリスト形式への変更です。

## 概要
ユーザーからの要望に基づき、4択モードにおいても一問一答モードと同じ基本レイアウト（カードが縦に並び、ドラッグ＆ドロップで入れ替え可能なリスト形式）を採用します。初期化時に生成されるブロックのみが4択専用のものとなります。

## 変更内容

### [FourChoiceMode.jsx](file:///c:/FlashcardMaster/src/Pages/FourChoiceMode.jsx) [MODIFY]
- `OneQAMode.jsx` のロジックをベースにリファクタリングを行います。
- `editors` ステートを導入し、複数の `CardEditor` インスタンスを管理できるようにします。
- `@hello-pangea/dnd` を使用して、カードの並び替え機能を実装します。
- 保存、キャンセル、削除、移動のハンドラを一問一答モードと同様に実装します。
- `CardEditor` に渡す `mode="four_choice"` を維持し、4択専用のブロックが初期状態で表示されるようにします。

### [OneQAMode.jsx](file:///c:/FlashcardMaster/src/Pages/OneQAMode.jsx) [REFERENCE]
- `OneQAMode.jsx` の構造（コンテナのスタイリング、ヘッダー、ドラッグ＆ドロップの実装）を参考にします。

## 検証計画

### 自動テスト
- 現状、UIコンポーネントの自動テストは含まれていないため、手動検証を中心に行います。

### 手動検証
1. **新規作成画面の遷移**: フォルダ画面から「4択問題」を選択し、新しいレイアウトが表示されることを確認します。
2. **複数カードの追加**: 「続けて作成」ボタンを押し、新しいカードエディタがリストの最後に追加されることを確認します。
3. **並び替え**: ドラッグハンドルまたは上下ボタンを使用して、カードの順序が入れ替わることを確認します。
4. **削除**: 削除ボタンを押し、カードがリストから取り除かれることを確認します。
5. **保存**: すべてのカードが正しく保存され、フォルダ画面に戻ることを確認します。
6. **同期**: 4択モード特有の「正解ブロックの自動同期」が正しく動作し、リスト形式でも問題ないか確認します。
