# 検証報告：カード一覧レイアウト修正とボタンサイズ調整

## 1. カード一覧のレイアウト崩れ修正

### 修正内容
`CardList.tsx` において、ドラッグ機能有効時（FolderViewなど）に `Draggable` コンポーネントが生成するラッパー `div` と、カード自体の `div` が分かれていたため、Gridレイアウト内でのスタイル適用に不整合が生じていました。
これを修正するため、`Draggable` の `ref` と `draggableProps` をカードのメイン `div` に直接適用するようにリファクタリングしました。

### 確認項目
- [x] **フォルダ画面**: カードの枠線や影が、Gridセル内に正しく収まっていること。
- [x] **ドラッグ操作**: カードを掴んで移動させた際、表示が崩れず、ドロップ後も正しい位置に収まること。
- [x] **選択状態**: チェックボックスで選択した際もレイアウトが維持されること。

## 2. ブロック編集ボタンのサイズ調整

### 修正内容
`BlockWrapper.tsx` において、ブロック右側（モバイルでは上部）に表示される操作ボタン（移動、複製、削除）のサイズを縮小しました。

### 変更点
- **パディング**: `p-2` → `p-1.5` (デスクトップ)
- **アイコンサイズ**: `w-4 h-4` → `w-3.5 h-3.5` (デスクトップ)

### 確認項目
- [x] **ブロックホバー時**: 表示されるボタン群が以前よりコンパクトになっていること。
- [x] **操作性**: ボタンが小さくなりすぎず、クリック可能であることを確認（コード上の数値は十分なサイズを確保）。
