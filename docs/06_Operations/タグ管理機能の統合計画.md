# タグ管理機能の設定画面統合計画

フォルダ管理画面やフォルダ詳細画面にある「タグ管理」アイコンをクリックした際、既存の `TagManagerDialog` を開くのではなく、設定画面（`SettingsDialog`）の「タグ管理」タブを直接開くように変更します。

## 変更内容

### 1. [SettingsDialog.jsx](file:///c:/FlashcardMaster/src/Components/settings/SettingsDialog.jsx) の拡張
- `initialTab` プロップを追加します。
- ダイアログが開いた際、または `initialTab` が変更された際に `activeTab` を同期する `useEffect` を追加します。

### 2. [Layout.tsx](file:///c:/FlashcardMaster/src/Layout.tsx) の調整
- URL のクエリパラメータから `tab` を取得し、`SettingsDialog` に `initialTab` として渡すようにします。

### 3. [Folders.jsx](file:///c:/FlashcardMaster/src/Pages/Folders.jsx) のリファクタリング
- `TagManagerDialog` のインポートと使用を削除します。
- `useSearchParams` を導入し、タグアイコンクリック時に `settings=true` かつ `tab=tags` を URL にセットするように変更します。

### 4. [FolderView.jsx](file:///c:/FlashcardMaster/src/Pages/FolderView.jsx) の改善
- メニュー内の「タグ管理」を選択した際、同様に設定画面のタグ管理タブを開くようにします（現在は無効化されています）。

## 検証計画

### 手動確認
1.  フォルダ一覧画面（Folders）でタグアイコンをクリックし、設定ダイアログが「タグ管理」タブが開いた状態で表示されることを確認する。
2.  フォルダ詳細画面（FolderView）のメニューから「タグ管理」を選択し、同様に設定ダイアログが開くことを確認する。
3.  設定ダイアログを閉じ、URL からパラメータが正しく消えることを確認する。
4.  直接の URL（例: `/folders?settings=true&tab=tags`）でアクセスした際、最初からダイアログが開くか確認する。
