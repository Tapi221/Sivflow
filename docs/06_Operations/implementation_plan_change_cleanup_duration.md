# 端末自動削除期間の変更 (24時間 -> 60日) 実装計画

## 概要
ユーザーの要望により、非アクティブな端末（古いセッション）の自動削除判定期間を「24時間」から「60日」に延長します。
これにより、頻繁に利用しない端末が意図せず削除されるのを防ぎます。

## 変更内容

### 1. ロジック変更 (`src/services/SyncServiceV2.ts`)
*   `cleanupInactiveDevices` メソッド内の判定閾値を変更します。
*   現状: `24 * 60 * 60 * 1000` (24時間)
*   変更後: `60 * 24 * 60 * 60 * 1000` (60日)

### 2. UI/メッセージ変更 (`src/Components/settings/DeviceSyncSettings.tsx`)
*   ユーザー向けの説明テキスト、確認ダイアログのメッセージを「24時間」から「60日」に変更します。
    *   確認ダイアログ (`window.confirm`)
    *   スイッチのサブテキスト (`<p>`タグ)
    *   削除ボタンのツールチップ (`title`属性)

## 検証計画
### 自動テスト
*   現状のユニットテストに `cleanupInactiveDevices` のテストが含まれているか確認し、あれば閾値変更に合わせて更新します（今回の変更範囲にはテスト修正は明示されていませんが、念のため）。

### 手動検証
1.  **UI確認**: 設定画面「同期・ストレージ管理」を開き、以下の表示が「60日」になっていることを確認。
    *   「非アクティブ端末の自動整理」の説明文
    *   「古いセッションを掃除」ボタンのツールチップ
    *   ボタン押下時の確認ダイアログ
2.  **ロジック動作確認（擬似的）**:
    *   コード内の定数を一時的に「1分」などに短縮し、1分以上放置した端末が削除されるか確認（あるいはコードレビューにて `60 * 24...` の計算式が正しいことを確認）。
    *   本番適用時は「60日」であることを確認。
