# コードブロック表示改善および型エラー修正の検証

## 概要
ユーザーの要望に基づき、コードブロックの表示スタイル（ヘッダー高さ縮小、インデントガイド追加）を改善し、TypeScriptの型エラーを修正しました。

## 変更点

### 1. コードブロックのスタイル改善
- **対象コンポーネント**: `src/Components/card/CodeRenderer.tsx`
- **変更内容**:
    - ヘッダー (`Toolbar`) の高さを `h-6` (24px) に縮小し、パディングを `py-0.5` に変更。
    - コード表示領域 (`pre`) にインデントガイド（縦線）を表示するCSSを追加。
        - `backgroundImage`: `linear-gradient`を使用し、2文字（`2ch`）ごとに薄い灰色の縦線を表示。
        - `backgroundAttachment`: `local` に設定し、スクロールに追従。

### 2. TypeScript型エラーの修正
- **対象ファイル**: `src/types/index.ts`
- **変更内容**:
    - `UserSettings` インターフェースに `blockButtonShowLabel?: boolean;` を追加。
    - これにより `BlockEditor.tsx` で発生していた「プロパティが存在しない」エラーが解消。

## 検証手順

### ビジュアル確認
1. アプリケーションを起動。
2. 設定画面またはカード編集画面で「コード」ブロックを追加。
3. 言語を選択し、インデントを含むコードを入力。
    - **期待値**:
        - ヘッダー部分が以前よりコンパクトに表示されること。
        - コードのインデント部分に薄い縦線が表示され、インデントレベルが視認しやすくなっていること。

### ビルド確認
1. ターミナルで `npm run build` (または `tsc`) を実行。
    - **期待値**: `src/Components/card/BlockEditor.tsx` に関連する型エラーが発生せず、ビルドが成功すること。

## 補足
- インデントガイドはブラウザのフォントレンダリングに依存するため、`ch`単位でのグリッドが完全に一致しない場合がありますが、等幅フォント（`Fira Code`など）を指定しているため概ね整合します。
