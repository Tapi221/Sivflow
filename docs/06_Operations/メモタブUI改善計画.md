# メモタブUIデザイン改善計画

フォルダ詳細画面の「メモ」タブのUIを、カード編集画面などの他の画面と統一感のある、よりモダンで洗練されたデザインに改善します。

## 改善のポイント

1.  **ヘッダーの刷新**:
    - `StickyNote` アイコンに背景円を追加し、アクセントカラーを適用します。
    - ラベルを他の画面で使われている `uppercase tracking-widest` なスタイルに変更します。
2.  **入力エリアのモダン化**:
    - `AutoResizeTextarea` の外枠を廃止し、`bg-slate-50` の背景と大きめの角丸（`rounded-xl`）を持つコンテナに収めます。
    - フォーカス時のリングなどを他の入力要素と統一します。
3.  **画像グリッドとアクションの調整**:
    - 画像のプレビュー枠や削除ボタンのスタイルを調整します。
    - 「画像追加」ボタンを、ダッシュボードやカード一覧で使われている丸みのあるボタンに変更します。
4.  **全体のレイアウト**:
    - 余白（padding）やフォントサイズを見直し、窮屈さを解消します。

## 検証計画

### 自動テスト
- 現状、UIコンポーネントのユニットテストは存在しないため、ビルドエラーが発生しないことを確認します。
- `npm run build` を実行して、型定義や構文に問題がないことを確認します。

### 手動確認
1.  ブラウザでフォルダ詳細画面を開き、「メモ」タブを選択する。
2.  新しくデザインされた入力エリアが表示されていることを確認する。
3.  テキストの入力、画像の追加・削除が正常に行えることを確認する。
4.  他のタブ（CARDS, NOTES等）と行間やフォント、色の使い方が統一されているか目視で確認する。
5.  モバイル表示でレイアウトが崩れていないか確認する。
