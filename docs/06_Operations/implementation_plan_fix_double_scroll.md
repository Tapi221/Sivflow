# 実装計画: サイドバーの二重スクロール解消と詳細修正

## 問題点
1. **二重スクロール**: `TreeViewLayout` と `FolderTreeWithCards` の両方で `overflow-y-auto` が指定されており、サイドバー内にスクロールバーが2つ発生している（またはスクロール挙動が干渉している）。
2. **青い枠線**: スクロール要素にフォーカスが当たった際のブラウザデフォルトのリング（`outline`）が表示されている。
3. **高さの不整合**: レイアウトの計算値が厳密でないため、微妙なスクロールが発生する可能性がある。

## 解決策

### 1. [Component] FolderTreeWithCards.tsx
- コンテナの `h-full overflow-y-auto` を削除します。
- スクロール管理は親コンポーネント（`TreeViewLayout`）に委譲します。
- `py-2` も削除し、親側でパディングを管理します。

### 2. [Layout] TreeViewLayout.tsx
- **左ペイン（サイドバー）**:
  - `overflow-y-auto` を維持し、ここでスクロールを一元管理します。
  - `py-2` を追加して余白を復元します。
  - `outline-none` を維持し、フォーカスリングを防止します。
- **高さ調整**:
  - モバイル(`pt-14 + pb-24 = 152px`) と デスクトップ(`pt-0 + pb-10 = 40px`) の差を考慮し、レスポンシブな高さ計算を適用します。
  - `h-[calc(100dvh-152px)] md:h-[calc(100dvh-40px)]` に変更。

### 3. [Component] CardEditorPane.tsx
- `outline-none` を追加し、フォーカスリングを防止します。
