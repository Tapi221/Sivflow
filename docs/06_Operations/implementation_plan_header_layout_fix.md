# カード編集画面ヘッダーのレイアウト最適化（Implementation Plan）

カード編集画面のヘッダーを、PC表示（lg以上）において「3カラム × 2行」のGridレイアウトに再構成し、各要素の高さ、角丸、縦位置を完全に揃えます。

## 変更の概要

現状のFlexboxベースのレイアウトから、指定されたGrid構成へ移行します。

### Grid構成 (lg以上)
- **Column 1 (180px)**: INDEX(Q#) と 耐性スコア。2行をまたいで表示（row-span-2）。
- **Column 2 (1fr)**: タイトル（Row 1）、タグ（Row 2）。`min-w-0` を付与し縮小に対応。
- **Column 3 (auto)**: アクションバー（Row 1）、プレビュー（Row 2）。`shrink-0` を付与。

### 統一スタイル
- 共通の高さ（`h-14` 程度）と角丸（`rounded-2xl` 程度）を採用。
- 入力フィールドと右側の操作ブロックの高さを完全に一致させます。

## 検証プラン

### 自動テスト / コード確認
- `npm run build` でビルドエラーが発生しないことを確認。
- Tailwind CSS の Grid クラスが正しく適用されているか目視確認。

### 手動検証
- PC幅（lg以上）で、タイトル入力とアクションバー、タグ入力とプレビューの高さと縦位置が揃っていることを確認。
- 画面幅を縮めていった際、Col2（タイトル/タグ）が適切に縮小され、右側が押し出されないことを確認。
- モバイル表示時（lg未満）に、要素が適切にフォールバック（例：縦並び）されていることを確認。
- 入力、トグル、ブックマーク等の既存機能が正常に動作することを確認。
