# コードブロックの縦幅コンパクト化計画

## 概要
コードブロック（編集時および表示時）の縦方向の余白（パディング）を削減し、よりコンパクトな表示にします。特に閲覧用（Renderer）のパディングが著しく大きいため、これを調整し、編集・閲覧時のデザインの統一感も向上させます。

## 現状の課題
1.  **CodeRenderer (閲覧時)**:
    *   コードエリアのパディングが `p-6` (24px) と非常に大きい。
    *   ツールバーのパディングが `py-2` (8px) で、編集時より広い。
2.  **CodeBlockEditor (編集時)**:
    *   内部パディングが `8px`。これは比較的小さいが、さらなるコンパクト化が可能であれば調整する。

## 変更内容

### 1. 閲覧用コンポーネントの調整 (`src/Components/card/CodeRenderer.tsx`)
*   **コードエリア**: `p-6` (24px) を `p-4` (16px) に変更。
*   **ツールバー**: `px-4 py-2` を `px-3 py-1.5` に変更し、ヘッダー部分の高さを抑える。

### 2. 編集用コンポーネントの調整 (`src/Components/card/CodeBlockEditor.tsx`)
*   **エディタエリア**: `padding={8}` を `padding={6}` に微調整し、より引き締まった印象にする。

## 検証計画
### 手動検証
1.  **表示確認**: コードブロックを含むカードを表示し、コード部分の余白が以前より狭くなっていることを確認する。
2.  **編集確認**: コードブロックを編集モードにし、違和感なく入力できること、ツールバーのサイズ感が適切であることを確認する。
