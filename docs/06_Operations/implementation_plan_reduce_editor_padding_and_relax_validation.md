# カードエディタの余白削減と4択モードのバリデーション緩和 実装計画

カードエディタ内の視覚的な密度を高めるための余白削減と、4択モードにおける入力制約の緩和を行います。

## 概要
1. **余白削減**: ブロック間の `space-y-4` を縮小し、よりコンパクトな表示にします。
2. **4択モードのバリデーション緩和**: 4択モードでは「問題文」ブロック（インデックス0）が空であっても、選択肢（インデックス1以降）や解答欄に入力があれば保存できるようにします。現在は「完全に空」なカードの保存を防止していますが、4択モードの特定の構成を許容するように調整します。

## 変更内容

### 1. 余白の削減

#### [MODIFY] [BlockEditor.tsx](file:///c:/FlashcardMaster/src/Components/card/BlockEditor.tsx)
- ブロックを囲む `div` の `space-y-4` を `space-y-1.5` 程度に縮小します。
- セクション全体の `space-y-4` も適宜調整します。

#### [MODIFY] [TextBlock.tsx](file:///c:/FlashcardMaster/src/Components/card/blocks/TextBlock.tsx)
- `AutoResizeTextarea` の `py-1` を見直し、上下の詰まり具合を調整します。

### 2. 4択モードのバリデーション緩和

#### [MODIFY] [useCards.ts](file:///c:/FlashcardMaster/src/hooks/useCards.ts)
- `isCardCompletelyEmpty`（または `hasBlocksContent`）のロジックを微調整します。
- 4択モード（`mode === 'four_choice'`）の場合、問題文ブロックが空でも他のブロックに内容があれば「空ではない」と判定するようにします。
- ※現状の `hasBlocksContent` は「いずれかのブロックに内容があれば true」を返すため、実は4択モードでも「正解」や「不正解」に内容があれば保存可能なはずです。ユーザーが「保存できない」と感じている原因（例えば `CardEditor.tsx` 側での独自バリデーションなど）を確認し、整合性を取ります。

#### [MODIFY] [CardEditor.tsx](file:///c:/FlashcardMaster/src/Components/card/CardEditor.tsx)
- `isCardEmpty` 関数を確認し、4択モードにおける「内容あり」の判定を正確に行うように修正します（問題文が空でも、選択肢があればOKとする）。

## 検証計画

### 手動検証
1. **余白の確認**: カード編集画面を開き、各ブロック（テキスト、数式など）の間の余白が以前より詰まっていることを視覚的に確認します。
2. **4択モードでの空問題保存**: 
   - 4択モードで「問題文」を空にしたまま、「正解」と「不正解」だけを入力して保存できることを確認します。
   - すべてのブロックが空の場合は、引き続き保存がブロックされることを確認します。
3. **一問一答モードへの影響**: 通常モードでの保存バリデーションに意図しない緩和が起きていないか確認します。
