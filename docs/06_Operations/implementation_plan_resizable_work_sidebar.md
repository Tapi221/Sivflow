# 実装計画: 作業ビューのフォルダペインのリサイズ対応

## ゴール
1. **作業ビュー（Work View）**のフォルダツリー部分をリサイズ可能にする。
2. フォルダツリーとカード編集エリアの間の隙間（gap）をなくし、「一本の境界線」で区切られた一体型レイアウトに変更する。
3. 境界線をドラッグして幅を調整できるようにし、設定を保存する。

## Proposed Changes

### レイアウト変更 (`src/Components/folder/TreeViewLayout.tsx`)
1. **コンテナ変更**:
   - 現在の「独立した2つのカードが並ぶレイアウト（`gap-4`）」から、「1つの大きな枠の中に2つのペインがあるレイアウト」に変更します。
   - `flex h-[calc(100vh-140px)] gap-4` → `flex h-[calc(100vh-140px)] border border-slate-200 rounded-xl shadow-sm overflow-hidden bg-white`

2. **左ペイン（フォルダツリー）**:
   - 個別のボーダーと角丸を削除。
   - 幅を `sidebarWidth` ステートで管理（初期値 320px, 範囲 200px ~ 600px）。
   - 右側に `border-r border-slate-200` を追加し、これを「境界線」とします。

3. **右ペイン（カード編集）**:
   - 個別のボーダーと角丸を削除。
   - `flex-1` で残りの領域を埋める。

4. **リサイズ機能**:
   - `mousedown` でドラッグ開始、`mousemove` で幅更新、`mouseup` で終了・保存。
   - 境界線上に透明な（またはホバーで色付く）ドラッグハンドル `w-1` を配置し、操作しやすくする。

### ステート管理
- `sidebarWidth`: LocalStorage (`workViewSidebarWidth`) に保存。

## Verification Plan

### Manual Verification
1. **外観確認**:
   - 作業ビューを開き、以前のような「隙間」がなくなり、一本の線で区切られていることを確認。
   - 全体が1つのパネルのように見えること。
2. **リサイズ操作**:
   - 境界線をドラッグして、フォルダツリーの幅が滑らかに変わることを確認。
   - 最小幅(200px)・最大幅(600px)の制限が効いていることを確認。
3. **永続化**:
   - ページをリロードしても幅が維持されることを確認。
