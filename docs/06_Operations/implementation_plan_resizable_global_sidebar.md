# グローバルサイドバーのリサイズ機能実装計画

グローバルナビゲーションサイドバー（`Layout.tsx`）にドラッグ可能なハンドルを追加し、ユーザーが幅を自由に調整できるようにします。また、幅と開閉状態を `ui.*` プレフィックスのLocalStorageキーで永続化します。

## 変更内容

### [Layout.tsx](file:///c:/FlashcardMaster/src/Layout.tsx)

1.  **状態管理の追加**:
    *   `sidebarWidth`: サイドバーの現在の幅を管理（初期値: 72px）。LocalStorage `ui.sidebarWidth` から読み込み。
    *   `isSidebarOpen`: 既存の `isSidebarOpen` のLocalStorageキーを `ui.sidebarOpen` に変更。
    *   `isResizing`: ドラッグ中かどうかを判定するフラグ。

2.  **ドラッグロジックの実装**:
    *   `onMouseDown`: ハンドルをクリックしたときに `isResizing` を `true` に設定。
    *   `onMouseMove`: マウスの動きに合わせて `sidebarWidth` を更新（最小幅 72px、最大幅 300px 程度の制約を設ける）。
    *   `onMouseUp`: ドラッグ終了時に `isResizing` を `false` にし、幅をLocalStorageに保存。

3.  **UIの調整**:
    *   `aside` 要素の幅を `sidebarWidth` に基づいて動的に設定。
    *   右端に透明なドラッグハンドル（`div`）を追加。ホバー時に強調表示。
    *   メインコンテンツ（`main`）の `marginLeft` を `sidebarWidth` に合わせて調整。
    *   ドラッグ中、`document.body` またはコンテナに `user-select: none` を適用。

4.  **アクセント対応**:
    *   サイドバーが広がった際、ナビゲーションアイテムがアイコンだけでなくテキスト（ラベル）も横に並ぶようにスタイルを調整する。

## 検証プラン

### 自動テスト
*   現状、UIのドラッグ動作を検証する既存の自動テストはないため、ブラウザツールを用いた確認を優先します。

### 手動確認
1.  **リサイズ動作**: サイドバーの右端をドラッグして幅が変わることを確認。
2.  **境界制限**: 最小幅（72px）以下や過度な最大幅にならないことを確認。
3.  **永続化**: 幅を変更してリロードした後、変更後の幅が維持されていることを確認。
4.  **開閉連動**: `Ctrl+B` で閉じ、再度開いた際に元の幅に戻ることを確認。
5.  **テキスト選択防止**: ドラッグ中に周囲のテキストが選択されないことを確認。
