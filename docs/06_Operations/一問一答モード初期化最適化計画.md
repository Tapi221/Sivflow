# 一問一答モードの初期状態およびフォーカス最適化

一問一答モードにおいて、ユーザーがスムーズにカード作成を開始できるよう、初期状態で2枚のカードを表示し、適切な入力項目に自動的にフォーカスを当てるように改善します。

## Proposed Changes

### 一問一答モード (OneQAMode)
#### [MODIFY] [OneQAMode.jsx](file:///c:/FlashcardMaster/src/Pages/OneQAMode.jsx)
- `editors` ステートの初期値を変更し、初期状態で 2 つのエディタ（カード）が表示されるようにします。
- 1枚目のカードにのみ `autoFocus: true` を設定します。

### カードエディタ (CardEditor)
#### [MODIFY] [CardEditor.tsx](file:///c:/FlashcardMaster/src/Components/card/CardEditor.tsx)
- `autoFocus` プロップの処理を拡張します。
- `hideTitle` が `true` の場合、タイトルではなく最初の問題ブロック（BlockEditor）に `autoFocus` フラグを伝播させます。

### ブロックエディタ (BlockEditor)
#### [MODIFY] [BlockEditor.tsx](file:///c:/FlashcardMaster/src/Components/card/BlockEditor.tsx)
- `autoFocus` プロップを追加します。
- `autoFocus` が `true` かつブロックが存在する場合、最初のブロック（特にテキスブロックを想定）に `autoFocus` フラグを渡します。

### テキストブロック (TextBlock)
#### [MODIFY] [TextBlock.tsx](file:///c:/FlashcardMaster/src/Components/card/blocks/TextBlock.tsx)
- `autoFocus` プロップを追加します。
- `AutoResizeTextarea` に `autoFocus` を適用し、レンダリング時に自動的にフォーカスが当たるようにします。

## Verification Plan

### 自動テスト/手動確認
- 一問一答モードに遷移した際、カードが2枚表示されていることを確認。
- タイトルあり設定の場合：1枚目のタイトルの入力欄にフォーカスが当たっていることを確認。
- タイトルなし設定の場合：1枚目の「問題」セクションの最初のテキスト入力欄にフォーカスが当たっていることを確認。
- カード保存後に「続けて作成」を押した際、新しいカードが追加され、そこにフォーカスが当たる既存機能が壊れていないことを確認。
