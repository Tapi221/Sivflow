# 作業完了報告：画像ブロックの枚数制限と自動ブロック生成

画像ブロックの画像枚数を1枚に制限し、あふれた画像を自動的に新規ブロックへ割り振る機能を実装しました。

## 実施内容

### 1. `src/Components/card/blocks/MediaBlock.tsx`
- **枚数制限の適用**:
    - `MediaUploader` に渡す `maxFiles` を常に `1` に設定しました（画像・音声共通）。
    - これにより、ブロック内には常に1つのファイルしか保持されなくなります。

### 2. `src/Components/card/BlockEditor.tsx`
- **Pending Uploads 管理**:
    - `pendingUploads` ステートを追加し、新規作成されたブロックに渡すべき初期ファイルを管理します。
- **Overflow ハンドリング**:
    - `handleBlockOverflow` を実装し、あふれたファイルを受け取ると直下に新しい画像ブロックを作成し、ファイルを割り当てます。

### 3. `src/Components/card/MediaUploader.tsx`
- **Overflow 検知**:
    - アップロード時および貼り付け時に、`maxFiles` を超えるファイルを検知します。
    - 超過分を `onFilesExcess` コールバックを通じて親コンポーネント（BlockEditor）に通知します。
- **初期ファイル処理**:
    - `initialFile` プロパティを受け取り、マウント時に自動的にアップロードを開始する処理を追加しました。
    - これにより、自動生成されたブロックで即座に画像のアップロードが始まります。

## 動作確認方法

以下の手順で動作を確認してください。

1. **基本制限の確認**
    - 画像ブロックを作成し、画像を1枚追加する。
    - 「追加」ボタンが表示されなくなることを確認する。

2. **複数ファイルの同時ドロップ**
    - 空の画像ブロックに、**3枚の画像** を同時にドラッグ＆ドロップする。
    - 1枚目が現在のブロックに入り、その下に**2つの新しい画像ブロック** が自動生成され、残りの画像が入ることを確認する。

3. **満杯ブロックへの追加**
    - 既に画像が入っているブロックに、別の画像をドロップ（または Ctrl+V）する。
    - 現在のブロックは変更されず、**直下に新しいブロック** が作成され、画像が入ることを確認する。

4. **音声ブロックの確認**
    - 音声ブロックも同様に1つ制限であり、複数ファイル追加時は分割されることを確認する（音声は仕様上元々1つ制限でしたが、自動生成ロジックが共通化されました）。
