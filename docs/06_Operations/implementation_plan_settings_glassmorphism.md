# 設定画面 Glassmorphism デザイン改修計画

## 概要
設定画面 (`SettingsDialog.jsx`) を、提供された画像のような Glassmorphism（すりガラス風）デザインに刷新します。
背景の透過、ぼかし効果、および全体的な「プレミアム感」のあるUIを目指します。

## ユーザーレビューが必要な事項
- デザインの大幅な変更となりますが、既存の機能配置は可能な限り維持します。
- モバイル表示時のレイアウトについても、Glassmorphism を適用しつつ使いやすさを維持します。

## 変更内容

### [src/index.css](file:///c:/FlashcardMaster/src/index.css)
Glassmorphism 用のユーティリティクラスを追加します。（必要に応じて Tailwind の設定も検討しますが、今回は CSS 変数とクラスで対応予定）
- `.glass-panel`: 背景色 rgba、backdrop-filter: blur を適用
- `.glass-sidebar`: サイドバー用の少し濃いガラス効果

### [src/Components/settings/SettingsDialog.jsx](file:///c:/FlashcardMaster/src/Components/settings/SettingsDialog.jsx)
- `DialogContent` のスタイルをオーバーライドし、デフォルトの白背景 (`bg-background`) を削除、透明 + Blur に変更します。
- 内部構造を「サイドバー」と「メインコンテンツ」に明確に分け、それぞれにガラス効果を適用します。
- アイコン、テキストの色を背景に合わせて調整します（白文字ベース、またはコントラスト確保）。
- アクティブなタブの表示スタイルを、画像のような「光る」または「目立つ」スタイルに変更します。

## 検証計画

### 自動テスト
- 現状の自動テストはないため、ビルドが通ること (`npm run build`) を確認します。

### 手動検証
1. アプリケーションを起動し、設定アイコンをクリックして設定画面を開く。
2. 背景が透けていること、ぼかし（Blur）が効いていることを確認。
3. サイドバーの項目を切り替え、スムーズに表示されることを確認。
4. ダークモード・ライトモード切り替え時の表示崩れがないか確認（今回は画像に合わせてダーク/Glass 基調にする可能性が高いが、視認性を確認）。
5. モバイルサイズにウィンドウを縮小し、レスポンシブ表示が崩れないか確認。
