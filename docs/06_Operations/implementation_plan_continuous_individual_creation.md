# 個別作成モードの連続作成対応 実装計画

個別作成モード（`CardEdit.jsx`）において、一問一答モードや4択モードと同様に、複数のカードを垂直に並べて連続作成できるようにリファクタリングします。

## 現状の課題
- 個別作成モード（`CardEdit.jsx`）は一度に1枚のカードしか表示・作成できない。
- 「続けて作成」をクリックすると `window.location.reload()` が走り、ユーザー体験が分断される。
- 他の連続作成モード（一問一答、4択）との操作感に乖離がある。

## 変更内容

### 1. `src/Pages/CardEdit.jsx` のリファクタリング
- **状態管理の変更**: `card` (単一オブジェクト) ではなく、`editors` (エディタのリスト) を管理するように変更します。
- **編集モードの維持**: `cardId` が指定されている（既存カードの編集）場合は、リストの要素を1つだけに固定し、従来の挙動を維持します。
- **新規作成モードの拡張**: `cardId` がない（新規作成）場合、`OneQAMode.jsx` と同様のリストベースの UI を提供します。
    - エディタの追加 (`+` ボタンまたは保存時の「続けて作成」)
    - エディタの削除
    - エディタの並び替え（Drag & Drop）
- **保存ロジック**: 各エディタごとに保存状態を管理し、個別に作成・更新を行えるようにします。

### 2. UI の統合
- `OneQAMode.jsx` で採用されている「垂直コネクタ線」や「フローティング操作ボタン」を `CardEdit.jsx` にも適用し、デザインの統一を図ります。
- `DragDropContext` を導入し、作成中のカード順序を直感的に変更できるようにします。

## 検証計画

### 自動テスト
- `npx tsc --noEmit` による型チェック。

### 手動確認
1. **新規作成**: フォルダから「個別作成」を選択し、複数のエディタが表示されること、および「続けて作成」でエディタが増えることを確認。
2. **保存**: 各エディタで保存ボタンを押し、正しくデータベースに登録されるか確認。
3. **並び替え**: ドラッグ＆ドロップでエディタの順序を入れ替えられるか確認。
4. **既存編集**: 既存のカードを編集する際、単一のエディタとして正しく動作し、新しいエディタが追加されないことを確認。
