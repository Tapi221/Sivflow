# ブラウザストレージ - ユーザー向け定型文

## 目的

技術的な問題をユーザーに説明するための定型文集。

---

## 共通ルール

> ※ 表示に「リロードしてください」と書かれていない場合、リロードしても改善しない可能性があります。

---

## 起動・初期化関連

### 🔄 再構築中（正常）

**状況**: `[AppInit] Rebuilding IndexedDB...` が出ている

**ユーザー向け説明**:
```
データを初期化しています。
この処理によってデータが失われることはありません。

⏱️ 通常は1〜2分で完了します。
5分以上続く場合は、異常の可能性があるためページをリロードしてください。

この画面を閉じないでください。
```

**補足**:
- 初回ログイン時
- データ破損検知時
- スキーマバージョン不一致時

---

### ⏳ 同期中（正常）

**状況**: `[Sync] Synchronizing...` が出ている

**ユーザー向け説明**:
```
クラウドと同期中です。

この端末のデータは保存されています。

⏱️ 通常は30秒〜1分で完了します。
3分以上続く場合は、ページをリロードしてください。

オフラインでも編集は可能ですが、
同期完了まで他のデバイスには反映されません。
```

**補足**:
- 初回ログイン時
- 長時間オフライン後
- 大量のデータ変更後

---

### 🚫 ストレージ容量不足

**状況**: `[Storage] Entered READ_ONLY mode: quota_exceeded` が出ている

**ユーザー向け説明**:
```
ブラウザのストレージ容量が不足しています。

既存のデータは保存されています。
新しいデータの保存ができません。

【対処法】
1. ブラウザの設定を開く
2. 「プライバシーとセキュリティ」→「Cookie とサイトデータ」
3. 不要なサイトのデータを削除
4. このページをリロード

容量を空けるまで、新しいデータは保存できません。
```

**補足**:
- READ_ONLY モード中
- 書き込みが拒否される
- 容量を空けるまで解決しない

---

### ⚠️ 無限再構築ループ

**状況**: `[AppInit] Health check failed: rebuild_loop` が出ている

**ユーザー向け説明**:
```
申し訳ございません。

通常は自動的に復旧しますが、
今回は自動復旧の上限を超えたため、起動できない状態です。

この問題はユーザー操作が原因ではありません。
システム側の調査が必要です。

サポートにお問い合わせください。

【エラーコード】: rebuild_loop
【ユーザーID】: <userId>
【タイムスタンプ】: <timestamp>
```

**補足**:
- 再構築が3回以上失敗
- ユーザー操作では解決不可
- 開発者の介入が必要

---

## データ関連

### 📦 データ復元完了

**状況**: 再構築 + 同期が完了

**ユーザー向け説明**:
```
データの復元が完了しました。

通常通りご利用いただけます。
```

**補足**:
- 再構築後
- 同期完了後

---

### ❌ データ復元失敗

**状況**: Firestore にデータがない、または破損

**ユーザー向け説明**:
```
データを復元できませんでした。

以下の可能性があります：
- 初回ログインのため、データがまだありません
- システム側のデータに問題があり、調査が必要です

【次のステップ】
- 初回ログインの場合: 新規にデータを作成してください
- データがあるはずの場合: サポートにお問い合わせください

この操作によって、既存のクラウドデータが消えることはありません。
```

**補足**:
- Firestore が空
- Snapshot が破損
- 同期が失敗

---

## ネットワーク関連

### 📡 オフライン

**状況**: `navigator.onLine === false`

**ユーザー向け説明**:
```
現在、オフラインです。

この端末のデータは保存されています。

編集は可能ですが、
オンラインに戻るまで他のデバイスには反映されません。
```

**補足**:
- オフライン編集可能
- 同期は自動

---

### ⚡ 同期エラー

**状況**: `[Sync] Failed:` が出ている

**ユーザー向け説明**:
```
クラウドとの同期に失敗しました。

この端末のデータは保存されています。

【確認事項】
- インターネット接続を確認してください

⏱️ 自動的に再試行します。
問題が解決しない場合は、サポートにお問い合わせください。
```

**補足**:
- ネットワークエラー
- Firebase ダウン
- 権限エラー

---

## エラー関連

### 💥 予期しないエラー

**状況**: 想定外のエラー

**ユーザー向け説明**:
```
申し訳ございません。
予期しないエラーが発生しました。

この問題はユーザー操作が原因ではありません。

【次のステップ】
1. ページをリロードしてください
2. 問題が解決しない場合は、サポートにお問い合わせください

データは保存されています。

【エラー情報】
エラーコード: <error_code>
ユーザーID: <userId>
タイムスタンプ: <timestamp>
```

**補足**:
- エラーコードを明示
- サポート連絡先を提示
- ユーザーの責任ではないことを明示

---

## 圧縮・メンテナンス関連

### 🗜️ 履歴圧縮中

**状況**: `[Compression] Compressing...` が出ている

**ユーザー向け説明**:
```
古い学習履歴を整理中です。

バックグラウンドで実行されます。
通常の操作に影響はありません。
```

**補足**:
- バックグラウンド処理
- ユーザー操作に影響なし

## まとめ

このドキュメントは「ユーザーに何を言うか」を明確にするものです。

**重要な原則**:
1. **技術用語を使わない**
2. **次の行動を明示する**
3. **諦めるラインを伝える**
4. **サポート連絡先を提示する**
