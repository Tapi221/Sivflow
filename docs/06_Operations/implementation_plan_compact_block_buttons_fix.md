# 実装計画：ブロックボタンのモバイル表示およびレイアウト最適化

## 背景
個別カード作成（`CardEditor.tsx`）や一問一答モード（`OneQAMode.jsx`）において、ブロック追加ボタン（テキスト、コード、画像、音声、リンク、数式）のラベルが表示されることで、画面幅によってはボタンが2列に折り返されてしまい、垂直方向のスペースを占有してしまう問題があります。
特にPCでの2カラムレイアウト時に顕著です。

## 変更内容
### 1. `BlockEditor.tsx` のレイアウト調整

- **ツールバーの折り返し防止**:
    - ボタンコンテナに `flex-nowrap` を指定し、不意な折り返しを防止します。
    - コンテナの最大幅とオーバーフロー制御を適切に行います。
- **レスポンシブなラベル表示タイミングの変更**:
    - 現状 `md` (768px) 以上でラベルを表示していますが、2カラムレイアウト（`lg` 1024px 以上）では1カラムあたりの幅が狭くなるため、ラベルを表示すると1列に収まりません。
    - ラベルの表示タイミングを `xl` (1280px) 以上に変更します。
    - また、ボタンのパディングとギャップを微調整し、より多くの画面サイズで1列に収まるようにします。

## 修正詳細

### [MODIFY] [BlockEditor.tsx](file:///c:/FlashcardMaster/src/Components/card/BlockEditor.tsx)

- ボタンコンテナの `flex-wrap` を `flex-nowrap` に変更（または `overflow-x-auto` を追加）。
- ボタン内のラベル `<span>` のクラスを `hidden md:inline` から `hidden xl:inline` に変更。
- ボタン自体の幅の指定 `w-8 md:w-auto` を `w-8 xl:w-auto` に変更。
- `md` 以上のパディング `md:px-4` を `xl:px-4` に変更し、中間サイズでは `px-0` (アイコンのみ) を維持。
