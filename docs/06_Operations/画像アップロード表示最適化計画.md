# 画像アップロード表示の最適化計画

画像アップロード後に表示される巨大なアップロードエリアが視覚的なノイズとなっている問題を解決するため、UIを動的に変更します。

## 変更内容

### [MediaUploader.tsx](file:///c:/FlashcardMaster/src/Components/card/MediaUploader.tsx)

1.  **表示ロジックの変更**:
    *   データ（`urls`）が空の場合: 現状どおりの大きなドロップゾーンを表示します。
    *   データ（`urls`）が存在する場合:
        *   タイプが `image` の場合: 画像グリッドの最後に「＋」または「画像を追加」というラベルの付いた小さなタイルを表示します。
        *   タイプが `audio` の場合: 音声リストの最後（または最初）に「音声を追加」という小さなボタンを表示します。

2.  **既存のドロップゾーンの条件付きレンダリング**:
    *   `urls.length === 0` の時のみ現在の `border-dashed` なエリアを表示するようにします。

## 検証計画

### 動作確認
1.  カード編集画面で画像ブロックを追加する。
2.  「大きなアップロードエリア」が表示されることを確認する。
3.  画像を1つアップロードする。
4.  アップロード完了後、「大きなアップロードエリア」が消え、画像タイルの横に「追加」ボタンが表示されることを確認する。
5.  追加ボタンをクリックして、さらに画像が追加できることを確認する。
6.  画像をすべて削除した際、再び「大きなアップロードエリア」が表示されることを確認する。
7.  音声ブロックについても同様の挙動を確認する。
