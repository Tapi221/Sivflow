# カード編集画面：コンテキストメニュー表示防止の対応報告

## 概要
カード編集画面の「続けて作成」ボタンを高速でクリックした際に、ブラウザのコンテキストメニュー（右クリックメニュー）が表示されてしまう問題を修正しました。

## 修正内容

### [Component] [CardEditor.tsx](file:///C:/FlashcardMaster/src/Components/card/CardEditor.tsx)

以下の2つのボタンに対して修正を適用しました：
1. **続けて作成** ボタン
2. **保存** ボタン

#### 具体的な変更点
- `onContextMenu={(e) => e.preventDefault()}` を追加
  - 右クリックおよび長押しによるコンテキストメニューの表示をプログラムレベルで無効化しました。
- `select-none` (CSS: `user-select: none`) クラスを追加
  - 高速クリック時にボタン内のテキストが誤って選択され、青く反転したりブラウザの特殊な挙動を誘発したりするのを防止しました。

## 動作確認結果
- [x] 「続けて作成」ボタンを右クリックしてもメニューが表示されない
- [x] 「続けて作成」ボタンを高速で連続クリックしてもメニューが表示されない
- [x] 「保存」ボタンについても同様の挙動を確認

これにより、カードの連続作成時にブラウザのメニューが邪魔をすることなく、スムーズに作業を継続できるようになりました。
