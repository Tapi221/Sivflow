# フォルダカラムビューのショートカットキー実装計画

## 概要

フォルダカラムビュー(`ColumnNavigator.tsx`)に包括的なキーボードショートカットシステムを実装し、エクスプローラーライクな操作性を実現します。また、設定画面のショートカットキー説明タブに新しいショートカットキーの説明を追加します。

## 目的

- フォルダナビゲーションをキーボードのみで完結できるようにする
- エクスプローラー(Finder)と同様の直感的な操作性を提供する
- 学習フローへのスムーズな移行を実現する
- 設定画面でショートカットキーを確認できるようにする

## 実装するショートカットキー

### 1. ナビゲーション系(移動)

| キー | アクション | 説明 |
|------|-----------|------|
| `↑` | 前のフォルダを選択 | 現在のカラム内で上のフォルダに移動 |
| `↓` | 次のフォルダを選択 | 現在のカラム内で下のフォルダに移動 |
| `→` | フォルダを開く | 選択中のフォルダを開いて次のカラムを表示 |
| `←` | 親フォルダへ戻る | 一つ前のカラムに戻る |
| `Alt + ←` | 履歴: 前の場所 | ナビゲーション履歴を戻る |
| `Alt + →` | 履歴: 次の場所 | ナビゲーション履歴を進む |

### 2. フォルダ・カード操作系

| キー | アクション | 説明 |
|------|-----------|------|
| `Ctrl/Cmd + N` | 新規フォルダ | 現在のカラムに新しいフォルダを作成 |
| `Ctrl/Cmd + Shift + N` | 新規カード | 選択中のフォルダに新しいカードを作成 |
| `Enter` | 名前変更 | 選択中のフォルダの名前を変更 |
| `F2` | 名前変更(Windows) | 選択中のフォルダの名前を変更 |
| `Delete` / `Backspace` | 削除 | 選択中のフォルダを削除(確認あり) |
| `Shift + Delete` | 完全削除 | 選択中のフォルダを完全削除(確認あり) |

### 3. 学習直結ショートカット

| キー | アクション | 説明 |
|------|-----------|------|
| `Space` | 即学習開始 | 選択中のフォルダ/カードで学習を開始 |
| `1` / `2` / `3` / `4` | 評価(学習中) | 学習中の評価を入力 |
| `Esc` | 学習終了 | 学習モードを終了 |

### 4. 構造操作(エクスプローラー型)

| キー | アクション | 説明 |
|------|-----------|------|
| `Ctrl/Cmd + X` | カット | 選択中のフォルダをカット |
| `Ctrl/Cmd + V` | ペースト | カットしたフォルダを貼り付け |
| `Tab` | インデント | 選択中のフォルダをサブフォルダ化 |
| `Shift + Tab` | アウトデント | 選択中のフォルダを親階層に移動 |

### 5. 生成・入力系

| キー | アクション | 説明 |
|------|-----------|------|
| `Ctrl/Cmd + Shift + V` | プレーンテキスト貼り付け | テキストから自動生成 |
| `Ctrl/Cmd + R` | 再生成 | 選択フォルダを再生成(確認あり) |

### 6. 検索・絞り込み

| キー | アクション | 説明 |
|------|-----------|------|
| `Ctrl/Cmd + F` | 現在フォルダ内検索 | 現在のフォルダ内を検索 |
| `Ctrl/Cmd + Shift + F` | 全体検索 | すべてのフォルダを検索 |

### 7. マルチペイン前提

| キー | アクション | 説明 |
|------|-----------|------|
| `Ctrl/Cmd + 1` | 左ペインフォーカス | 左ペインにフォーカスを移動 |
| `Ctrl/Cmd + 2` | 中央ペイン | 中央ペインにフォーカスを移動 |
| `Ctrl/Cmd + 3` | 右ペイン | 右ペインにフォーカスを移動 |

## 実装の詳細

### 変更するファイル

#### 1. `src/Components/folder/ColumnNavigator.tsx`

**変更内容:**
- フォーカス管理用の状態を追加(`selectedColumnIndex`, `selectedFolderIndex`)
- ナビゲーション履歴管理用の状態を追加(`navigationHistory`, `historyIndex`)
- `useKeyboardShortcuts`フックを使用してショートカットキーを登録
- 各ショートカットキーに対応するハンドラー関数を実装
  - `handleArrowUp/Down`: カラム内のフォルダ選択を移動
  - `handleArrowRight`: フォルダを開く
  - `handleArrowLeft`: 親フォルダへ戻る
  - `handleAltArrowLeft/Right`: 履歴移動
  - `handleCreateFolder`: 新規フォルダ作成
  - `handleCreateCard`: 新規カード作成
  - `handleRename`: 名前変更
  - `handleDelete`: 削除
  - `handleCut/Paste`: カット&ペースト
  - `handleIndent/Outdent`: インデント/アウトデント
  - `handleSearch`: 検索
  - `handlePaneSwitch`: ペイン切り替え

**実装のポイント:**
- フォーカス状態を視覚的に表示(選択中のフォルダをハイライト)
- 履歴管理はブラウザの履歴と同様の仕組みを実装
- カット&ペーストは既存のドラッグ&ドロップロジックを再利用
- インデント/アウトデントは階層制限を考慮

#### 2. `src/Components/folder/FolderColumn.tsx`

**変更内容:**
- フォーカス状態を受け取るプロップスを追加(`focusedIndex`)
- フォーカス中のフォルダに視覚的なハイライトを追加
- キーボード操作時のスクロール調整を実装

#### 3. `src/Components/settings/SettingsDialog.jsx`

**変更内容:**
- ショートカットキータブに「フォルダカラムビュー」セクションを追加
- 7つのカテゴリに分類してショートカットキーを表示
- 既存のテーブルフォーマットに合わせて実装

#### 4. `docs/04_Reference/shortcut_keys_specification.md`

**変更内容:**
- 「5. フォルダカラムビュー (Folder Column View)」セクションを追加
- 実装したすべてのショートカットキーを記載

## 検証計画

### 1. 手動テスト

#### ナビゲーション系のテスト
1. フォルダカラムビューを開く
2. `↑`/`↓`キーでフォルダ選択が移動することを確認
3. `→`キーでフォルダが開くことを確認
4. `←`キーで親フォルダに戻ることを確認
5. 複数のフォルダを開いた後、`Alt + ←`/`Alt + →`で履歴移動できることを確認

#### フォルダ・カード操作系のテスト
1. `Ctrl + N`で新規フォルダ作成ダイアログが開くことを確認
2. `Ctrl + Shift + N`で新規カード作成ダイアログが開くことを確認
3. フォルダを選択して`Enter`または`F2`で名前変更モードになることを確認
4. `Delete`キーで削除確認ダイアログが表示されることを確認

#### 構造操作のテスト
1. フォルダを選択して`Ctrl + X`でカット状態になることを確認
2. 別のフォルダを選択して`Ctrl + V`でペーストされることを確認
3. `Tab`キーでサブフォルダ化されることを確認(階層制限を考慮)
4. `Shift + Tab`で親階層に移動することを確認

#### 設定画面のテスト
1. 設定画面を開く
2. ショートカットキータブを選択
3. 「フォルダカラムビュー」セクションが表示されることを確認
4. すべてのショートカットキーが正しく記載されていることを確認

### 2. ビルド確認

```bash
npm run build
```

ビルドが成功することを確認します。

### 3. 本番環境での動作確認

デプロイ後、本番環境で以下を確認:
- すべてのショートカットキーが正常に動作すること
- 既存の機能に影響がないこと
- モバイルでの表示に問題がないこと(ショートカットキーは無効化される)

## 注意事項

> [!IMPORTANT]
> - 入力フィールドにフォーカスがある場合、ショートカットキーは無効化されます
> - 学習モード中のショートカットキー(1-4, Space, Esc)は既存の実装と競合しないように注意が必要です
> - モバイル環境ではキーボードショートカットは使用できないため、タッチ操作は引き続きサポートします

> [!WARNING]
> - 階層制限(最大深度)を超える操作は禁止されます
> - ルートフォルダを他のフォルダの子にする操作は禁止されます
> - 循環参照を防ぐため、自分自身や子孫フォルダへの移動は禁止されます

## 実装の優先順位

1. **Phase 1: ナビゲーション系** (最優先)
   - ↑/↓/→/← の基本移動
   - Alt + ←/→ の履歴移動

2. **Phase 2: フォルダ・カード操作系**
   - Ctrl + N, Ctrl + Shift + N
   - Enter, F2, Delete

3. **Phase 3: 構造操作**
   - Ctrl + X, Ctrl + V
   - Tab, Shift + Tab

4. **Phase 4: その他**
   - 学習直結ショートカット
   - 検索・絞り込み
   - マルチペイン前提

5. **Phase 5: ドキュメント更新**
   - 設定画面への説明追加
   - 仕様書の更新
