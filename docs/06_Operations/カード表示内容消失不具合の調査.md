# カード一覧表示不具合の調査 (2026-02-03)

## 現象
- カード一覧（FolderView）において、カードのタイトルが「無題のカード」、質問テキストが「質問テキストがありません」と表示される。
- 画像データなどが反映されていない可能性がある。
- 全てのカードが「作成中（Draft）」として表示される場合がある。

## 調査結果
1. **原因の特定**:
    - 新しい「ブロックエディタ」で保存した際、データは `questionBlocks` に保存されるが、旧来の `questionText` や `title` フィールドが空のまま保存されている。
    - `normalizeCard` (`utils/index.ts`) は「Text → Blocks」の変換は持っているが、「Blocks → Text」の変換を持っていない。
    - カード一覧（`CardList.tsx`）は依然として旧来の `questionText` を優先表示しようとするため、ブロックの中身が表示されない。
2. **Draft 状態の原因**:
    - `CardEditor.tsx` の `handleSubmit` において、`questionText` が空だと自動的に `isDraft: true` になるロジックが働いている可能性がある（またはブロック内のコンテンツ判定が不十分）。

## 修正方針
1. **`src/utils/index.ts` の `normalizeCard`**:
    - `questionBlocks` から最初のテキストブロックを抽出し、`questionText` が空の場合に補完するロジックを追加。
    - 同様に `answerBlocks` から `answerText` を補完。
    - `title` が空の場合、`questionText` の冒頭をタイトルとして使用するフォールバックを追加。
2. **`src/services/localDB.ts` の `denormalizeCardForStorage`**:
    - DB保存時にも、ブロックからテキストフィールドを同期するようにし、データの整合性を高める。
3. **`src/Components/card/CardEditor.tsx`**:
    - 保存前のバリデーションと Draft 判定ロジックがブロックの内容を正しく反映するように修正。
