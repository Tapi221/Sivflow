# 開発環境における安全なカード運用手順チェックリスト

開発環境 (`npm run dev`) で作成するカード・画像・音声データを消失リスクなしで日常利用するための設定・運用手順書です。

## 概要
*   **目的**: 開発中のアプリで作成したデータを消さずに日常利用する
*   **方法**: 不揮発なエミュレータ利用をやめ、本番環境 (Google Cloud) に直接保存する
*   **安全性**: Google のサーバーに保存されるため、PC再起動やキャッシュクリアでも消失しない

---

## 1. 設定変更手順

### [ ] Step 1: エミュレータの停止
現在実行中の `firebase emulators:start` はデータ消失の主な要因です。これを利用停止します。
*   [ ] ターミナルで `firebase emulators:start` が動いているウィンドウを開く
*   [ ] `Ctrl + C` を押してプロセスを完全に終了する
*   [ ] 今後、日常利用のためにエミュレータを起動する必要はありません

### [ ] Step 2: 環境変数の変更
本番環境へ接続するための設定を変更します。
*   [ ] ファイル `c:\FlashcardMaster\.env.local` を開く
*   [ ] 以下の行を見つける
    ```properties
    VITE_USE_FIREBASE_EMULATOR=true
    ```
*   [ ] 値を `false` に書き換えて保存する
    ```properties
    VITE_USE_FIREBASE_EMULATOR=false
    ```

### [ ] Step 3: 開発サーバーの再起動
設定変更をアプリに反映させます。
*   [ ] ターミナルで `npm run dev` が動いているウィンドウを開く
*   [ ] `Ctrl + C` を押してサーバーを停止する
*   [ ] 再度コマンドを実行する
    ```bash
    npm run dev
    ```

---

## 2. 動作確認手順

設定が正しく反映され、データがクラウドに保存されているか確認します。
**※注意: エミュレータ利用時の過去データは、この切り替えによって表示されなくなります（消失はしていませんが、参照先が変わるため）。**

### [ ] Step 4: 画像・音声の保存確認
1.  アプリを開き、適当なカードを新規作成する
2.  カードの「質問画像」または「解答画像」に画像をアップロードする
3.  画像が正常に表示されることを確認し、保存ボタンを押す
4.  **ブラウザの更新ボタン (F5) を押してページをリロードする**
5.  先ほどのカードを開き、**画像がまだ表示されていれば成功**
    *   *失敗例*: 画像が「×」アイコンになったり、読み込めない場合はエミュレータ設定が残っている可能性があります。Step 2, 3 を再確認してください。

---

## 3. 運用上の注意点・落とし穴

### クラウドへの反映タイミング
*   **編集直後**: 編集内容は即座にクラウドへ送信されますが、ネットワーク状況によっては数秒の遅延があります。
*   **注意**: 編集して保存ボタンを押した直後に（0.1秒以内に）ブラウザを強制終了すると、送信が間に合わない場合があります。保存完了のトースト表示を確認してから閉じれば確実です。

### 無料枠 (Sparkプラン) の上限
日常利用では余裕がありますが、以下の制限があります。
*   **Storage (画像・音声)**: 5GB まで
    *   目安: スマホ写真 (2MB) 約2,500枚、学習用圧縮画像 (200KB) 約25,000枚
*   **Firestore (文字データ)**: 1GB まで
    *   目安: 数百万枚のカード
*   **万が一超過した場合**: 追加の書き込みができなくなります（勝手に課金されることはありません）。

### メリット
*   PCが故障してもデータは Google サーバーに残ります。
*   将来、スマートフォンから同じアカウントでログインすれば、データをそのまま共有できます。
