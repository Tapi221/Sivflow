押す時、。# 設定画面のブロックエディタ並び替え修正報告

## 修正内容

設定画面の「ブロックエディタ設定」における並び替え挙動を安定化させるため、以下の修正を実施しました。

### 1. React StrictMode への対応

React 18 の `StrictMode` 環境下では、コンポーネントが2回マウントされる影響で `@hello-pangea/dnd` が正常に初期化されない場合があります。
これを防ぐため、マウント後に 1 フレーム待機してから `Droppable` を有効化するロジックを追加しました。

### 2. ドラッグ中のスタイル干渉除去

ドラッグ中の要素に `transition-all` クラスが適用されていると、ライブラリによる位置計算とブラウザのアニメーションが競合し、動きが遅れたり座標がずれたりする原因となります。
ドラッグ開始時に `transition-all` を一時的に無効化するように変更しました。

### 3. ドラッグ中の状態管理の安定化

設定が自動保存・同期される際に、ドラッグ中のローカルな並び替え状態が外部からの更新（`useEffect`）によって上書き（リセット）されないよう、ドラッグ状態を判定するガードを追加しました。

### 4. 視覚的な調整

- ドラッグ中の要素が他の要素の下に隠れないよう、`zIndex` を明示的に設定。
- 未定義だった `scale-102` クラスを適切な Tailwind クラス (`scale-[1.02]`) に修正し、ドラッグ時のハイライトを強調。

## 検証おすすめ手順

1. 設定画面 > 学習設定 > ブロックエディタ設定 を開く。
2. ブロック（テキスト、コード等）をドラッグ＆ドロップして並び替える。
3. ドラッグがスムーズに行えること、ドロップ後に順序が正しく保存されることを確認。
4. 一度ダイアログを閉じ、再度開いた際も変更した順序が維持されていることを確認。
