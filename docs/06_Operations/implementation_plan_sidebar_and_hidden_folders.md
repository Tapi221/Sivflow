# 実装計画: サイドバー最適化と非表示フォルダの対応

## ゴール
1. サイドバーの上部余白を極限まで削減し、画面領域を有効活用する。
2. フォルダツリー（作業ビュー）において、非表示設定(`isHidden: true`)のフォルダを表示しないようにする。

## User Review Required
> [!IMPORTANT]
> 非表示設定されたフォルダはツリーから完全に除外されます。再表示するには、データベースを直接操作するか、別途「非表示フォルダを表示する」機能が必要になりますが、今回の対応では「表示しない」ことに注力します。

## Proposed Changes

### レイアウト調整
#### [MODIFY] [Layout.tsx](file:///c:/FlashcardMaster/src/Layout.tsx)
- サイドバーのトグルボタン(`Menu`)の位置を `top-4` から `top-1` に変更し、サイズを調整。
- デスクトップ用サイドバー(`aside`)内のスペーサー(`h-20`)を縮小し、ナビゲーションアイテムの開始位置を上げる。
- 同期ステータスインジケーター(`SyncStatusIndicator`)の位置も `top-4` から `top-1` に合わせて調整。

### フォルダツリー修正
#### [MODIFY] [FolderTree.tsx](file:///c:/FlashcardMaster/src/Components/folder/FolderTree.tsx)
- `childFolders` および `rootFolders` のフィルタリングロジックに `!f.isHidden` を追加し、非表示フォルダがレンダリングされないようにする。
- これにより、「非表示のフォルダのボタン（フォルダ項目そのもの）」が表示されなくなる。

## Verification Plan

### Manual Verification
1. **サイドバーレイアウト確認**
   - ブラウザでアクセスし、サイドバー上部の余白がなくなっていることを確認。
   - トグルボタンがヘッダー領域（または最上部）に収まり、デザインが崩れていないか確認。
2. **非表示フォルダ確認**
   - 任意のフォルダを「非表示にする」操作を行う（既存機能）。
   - そのフォルダが即座にツリーから消えることを確認。
   - ページをリロードしても消えたままであることを確認。
