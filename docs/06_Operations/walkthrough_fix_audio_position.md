# 検証報告：音声ブロックの配置固定・並び替え制限

音声ブロックの配置を「リンクブロックの上（なければ最下部）」に固定し、ユーザーによる並び替えを制限しました。

## 実装内容の確認

### 1. 挿入位置の自動制御 (`BlockEditor.tsx`)
- [x] **音声ブロック追加時**:
    - リンクブロックが存在する場合 → その直前に挿入されること。
    - リンクブロックが存在しない場合 → リストの末尾に挿入されること。
- [x] **通常ブロック（テキスト等）追加時**:
    - 音声ブロックが存在する場合 → その直前に挿入されること。
    - リンクブロックしか存在しない場合 → その直前に挿入されること。
- [x] **リンクブロック追加時**:
    - 常にリストの末尾に挿入されること。

### 2. 並び替えの制限 (`BlockEditor.tsx`)
- [x] **ドラッグ無効化**:
    - 音声ブロック自体をドラッグしようとしても移動できないこと（`isDragDisabled={true}`）。
- [x] **ハンドル非表示**:
    - 音声ブロックの左側にドラッグ用のグリップアイコンが表示されないこと（`dragHandleProps={undefined}`）。

### 3. ドラッグ操作後の強制整列 (`CardEditor.tsx`)
- [x] **順序の強制**:
    - 他のブロックをドラッグして移動させた後、ドロップした瞬間に以下の順序が適用されること。
    - `[ ...その他のブロック, 音声ブロック(あれば), リンクブロック(あれば) ]`
    - 例：テキストブロックを音声ブロックの下にドロップしても、自動的に音声ブロックの上に移動する。

## 確認手順（検証用）

1. **新規作成**: 新しいカードを作成し、ボタンを「音声」→「リンク」→「テキスト」の順に押す。
    - 期待される順序: テキスト → 音声 → リンク
2. **ドラッグ試行**: 音声ブロックをドラッグしようとする。
    - 期待される挙動: ドラッグできない（反応しない）。
3. **割り込み**: テキストブロックを一番下にドラッグしてドロップする。
    - 期待される挙動: ドロップ後、即座に音声・リンクの上に戻る。
