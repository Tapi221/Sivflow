# 全画面でのテキスト選択防止 実装計画（更新版）

## 概要
ユーザー体験を向上させ、ブラウザ特有の意図しないテキスト選択（ハイライト）を防ぐため、アプリケーション全体に `select-none` を適用します。ただし、カード編集画面の入力内容や、カード閲覧画面のカードコンテンツなど、テキスト選択が必要な箇所については `select-text` で個別に除外します。

## 変更内容

### 1. グローバル適用 (`src/index.css`)
*   `body` 要素に `select-none` を適用します。
*   これにより、ログイン画面、設定画面、ダッシュボード、各種一覧画面など、全ての画面でデフォルトでテキスト選択が無効になります。

### 2. カード閲覧画面の除外 (`src/Components/card/Flashcard.tsx`)
*   カードのコンテンツエリア（問題・回答・メモなど）を包む `div` に `select-text` を適用します。
*   ユーザーがカード上のテキストをコピーしたり、数式を選択したりすることを可能にします。

### 3. カード編集画面の除外 (`src/Components/card/blocks/BlockWrapper.tsx`)
*   各ブロックのコンテンツエリア（テキストエリア、コードエディタなど）のコンテナに `select-text` を適用します。
*   編集時のカーソル操作やテキスト選択が正常に行えるようにします。

## 検証計画
### 手動検証
1.  **グローバル**: ダッシュボードのタイトルや設定画面のラベルが選択できないことを確認。
2.  **カード閲覧**: 学習中のカード内のテキストが選択できることを確認。背景やボタンは選択できないことを確認。
3.  **カード編集**: 各ブロック内（テキスト、コード等）でテキスト選択やドラッグ操作が正常にできることを確認。ヘッダーやボタンは選択できないことを確認。
