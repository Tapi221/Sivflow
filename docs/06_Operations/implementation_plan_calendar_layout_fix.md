# 実装計画：カレンダー画面のボタンレイアウト修正

カレンダー画面の「Today」ボタンおよび月表示コントロールが、画面右上の「同期状態インジケーター（同期ボタン）」と重なってしまう問題を修正します。

## 概要
現在、カレンダー画面のヘッダー要素は `justify-between` と空の `div` を使用して右側に寄せられていますが、これが原因で固定配置されている同期インジケーターと重なっています。これを左寄せ（`justify-start`）に変更し、重複を回避します。

## ユーザーレビューが必要な事項
- ボタンを左側に移動することについて（ユーザーの依頼は「右側」でしたが、同期ボタンが右側にあるため、左側に移動することで重なりを解消します。もし本当に右側に置いて同期ボタンを避ける必要がある場合は、マージンによる調整を行いますが、まずは標準的な左寄せを提案します。）

## 変更内容

### [Calendar.jsx](file:///c:/FlashcardMaster/src/Pages/Calendar.jsx)

- ヘッダーのコンテナの `justify-between` を `justify-start` に変更します。
- レイアウト調整用の空の `div` を削除します。
- モバイル表示時（`flex-col`）でも適切な余白が維持されるように調整します。

## 検証プラン

### 手動確認
- デスクトップ表示で、同期ボタンとカレンダーのコントロールが重なっていないことを確認。
- モバイル表示で、レイアウトが崩れていないことを確認。
- 「Today」ボタン、月移動ボタンが正常に動作することを確認。
