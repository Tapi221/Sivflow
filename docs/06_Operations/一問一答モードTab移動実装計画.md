# 一問一答モード：Tabキーによるフォーカス移動の最適化計画

一問一答モード（および `CardEditor` を使用する画面）において、Tabキーによる移動を「タイトル」「問題文」「解答」の主要な入力フィールドのみに制限し、素早い入力を可能にします。

## Proposed Changes

### 1. `CardEditor.tsx` の更新
- 全体を囲む `div` に `onKeyDown` ハンドラを追加。
- Tabキー（および Shift+Tab）入力をキャッチし、以下の順序でフォーカスを制御します。
    - **タイトルあり:** タイトル ↔ 問題文（最初のブロック） ↔ 解答（最初のブロック） ↔ タイトル
    - **タイトルなし:** 問題文（最初のブロック） ↔ 解答（最初のブロック） ↔ 問題文
- 各ブロック内（複数のテキストブロックがある場合）の移動も考慮します。

### 2. 要素の識別
- 各要素を確実に取得できるよう、DOM 操作用のクラス名（例: `js-editor-title`, `js-question-editor`, `js-answer-editor`）を付与します。
- タイトルは既存の `id="title-header"` を使用します。

## 3. 実装詳細 (Logic)

`handleKeyDown` 内での動作イメージ:
```javascript
const handleKeyDown = (e) => {
  if (e.key === 'Tab') {
    // 1. 現在の要素が入力フィールド（textarea等）か確認
    // 2. セクション（Title/Question/Answer）を特定
    // 3. 次の要素を DOM から検索
    // 4. 見つかれば e.preventDefault() して .focus()
  }
}
```

## Verification Plan

### Manual Verification
1.  **一問一答モード（タイトルあり）**:
    - タイトル入力中に Tab を押すと問題文に移動すること。
    - 問題文入力中に Tab を押すと解答に移動すること。
    - 解答入力中に Tab を押すとタイトルに移動すること。
    - Shift + Tab で逆順に移動すること。
2.  **一問一答モード（タイトルなし）**:
    - 問題文入力中に Tab を押すと解答に移動すること。
    - 解答入力中に Tab を押すと問題文に移動すること。
3.  **複数ブロック時**:
    - セクション内に複数のブロックがある場合、順次移動してから次のセクションへ移ることを確認。
