# カード編集画面のモバイル表示最適化 実装計画

## 現状の課題
1. **要素の重なり**: `OneQAMode` 画面において、親要素で絶対配置されている「保存済み」バッジや「削除」ボタンが、`CardEditor` 内のタイトル入力欄やタグ入力欄と重なっている。
2. **タグ入力欄の不備**: モバイル環境でタグ入力ボタン（＋）の位置がズレたり、タップしにくい。
3. **ブロック操作ボタンの重なり**: `BlockWrapper` 内の操作ボタン（複製、削除）がモバイルでブロック内容に干渉している。
4. **全体的な詰め込み感**: モバイルの狭い画面幅に対して要素が多方向（左右）に配置されており、整理されていない。

## 修正方針
1. **ヘッダーのレスポンシブ化**: 左右に分かれている要素を、モバイルでは完全に縦にスタック（中央揃えまたは左揃え）させ、絶対配置ボタン用の余白を確保する。
2. **絶対配置要素の調整**: `OneQAMode` のボタン位置を「重なりにくい場所」へ移動するか、`CardEditor` 側に十分なトップパディングを追加する。
3. **ブロックUIの改善**: モバイルでは操作ボタンをブロック内に埋め込むのではなく、適切な位置（例えば右下やブロックタイトル横）に配置し直す。
4. **タグ入力欄の調整**: `TagInput` のレイアウトをより柔軟にし、モバイルでの視認性とタップしやすさを向上させる。

## 完了条件
- モバイル画面でタイトル入力欄と「削除/保存済み」等のボタンが重ならない。
- タグ追加ボタンが適切な位置にあり、タップ可能。
- 各ブロックの入力中に、操作ボタンが邪魔にならない。
- 全体として一貫性のある縦方向のレイアウトになっている。

## 検証計画
### 手動検証
1. ブラウザの開発者ツールで「iPhone SE」などのモバイルサイズに設定。
2. `OneQAMode` ページにアクセスし、新規カード作成を行う。
3. タイトルを入力し、タグを追加する際に重なりが発生しないか確認。
4. 解答・問題ブロックを追加し、操作ボタンが使いやすい位置にあるか確認。
