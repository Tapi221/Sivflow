# 実装計画：ブロックツール表示判定の厳格化

現在のブロックツール（削除・複製・並び替えボタン）の表示判定が緩く、意図しないタイミングで表示される問題（特定のブロックのアクセントカラー枠内にあるときのみ表示したい）を修正します。

## 変更内容

### `src/Components/card/blocks/BlockWrapper.tsx`

`group-hover` パターンから `peer-hover` パターンへ構造を変更し、**枠線のあるメインコンテンツ部分（peer）** にマウスが乗っているとき（およびツール自体を操作しているとき）だけツールが表示されるようにします。

#### 具体的な変更手順

1.  最上位要素を `div.relative.group/block-container` でラップします（位置決め用）。
2.  現在のメインの `div`（枠線や背景色を持つ部分）から `group` クラスを削除し、代わりに `peer/block` クラスを付与します。
    - これにより、この `div` がホバー検知のトリガーとなります（枠内のみ）。
3.  ツール群の `div` を、メイン `div` の**外側（弟要素）**に移動させます。
4.  ツール群の表示制御クラスを以下のように変更します：
    - 変更前: `group-hover:opacity-100 group-hover:visible` etc.
    - 変更後: `peer-hover/block:opacity-100 peer-hover/block:visible` および `hover:opacity-100 hover:visible`
        - `peer-hover/block`: ブロック本体をホバーしているときに表示
        - `hover`: ツール自体にマウスが移動した後に表示を維持するため

## 検証計画

- [ ] **表示トリガーの確認**:
    - ブロックの枠線（Border）の内側にマウスを入れたときのみ、ツールが表示されることを確認。
    - ブロックの外側（ツールが表示される予定の場所など）にいきなりマウスを持っていっても、ツールが表示**されない**ことを確認。
- [ ] **操作性の確認**:
    - ブロックをホバーしてツールを表示させた後、マウスをツール側に移動させてもツールが消えずにクリックできることを確認。
    - ツールからマウスを離すと消えることを確認。
