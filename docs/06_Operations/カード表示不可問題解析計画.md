# カード・フォルダ表示不可問題の解析・修正計画

## 現状の分析
- **症状**: 作成したはずのカード/フォルダが画面に表示されず、IndexedDB (Dexie) にも 0 件。
- **エラー**: `FirebaseError: Expected first argument to collection() to be a CollectionReference...`
- **推論**: 
    1. データベース作成処理 (put/add) の前後に Firestore へのアクセスがあり、そこで `db` (firestoreDb) が未定義のまま `collection()` が呼ばれている。
    2. この例外により、ローカル DB への保存処理がコミットされる前にアボートしている。
    3. リファクタリングで `db` を `firestoreDb` に変えた際、インポートミスや初期化待ちの漏れがある可能性がある。

## 調査・修正タスク

### 1. エラー発生箇所の特定
- `FolderDialog.tsx` および `CardEditor.tsx` (または関連サービス) で `collection()` を呼び出している箇所を `grep` で特定。
- 引数に渡されている `db` (または `firestoreDb`) が正しく有効なインスタンスであるか確認。

### 2. インポートと初期化の整合性確認
- `src/services/firebase.ts` のエクスポートが全ての呼出元で正しく解釈されているか確認。
- `useAuth()` の `currentUser.uid` が確定する前に Firestore にアクセスしていないか確認。

### 3. 保存ロジックの堅牢化
- ローカル保存を優先し、同期処理の失敗がローカル保存をブロッキングしない構造になっているか確認。

## 検証プラン
- 修正後、`dist_test` へのビルドにより型安全性を確認。
- 実際の作成フローをログ出力等で追跡。
