# フォルダ階層デザインの改善

## 概要
フォルダの階層構造が視覚的に「ひとまとまり」として認識できるよう、デザインを刷新します。
現在はインデントのみで表現されていますが、ツリーガイド（縦線）や微細な背景色の調整を行い、親子関係とグループの範囲を明確にします。

## 実装内容

### components/folder/FolderTree.tsx の改修

1.  **ツリーガイド（縦線）の導入**
    - 子フォルダを展開した際、親フォルダから伸びる縦線を表示し、視覚的なつながりを持たせます。
    - `FolderItem` 内の再帰レンダリング部分において、左側にボーダーラインを持つコンテナでラップします。

2.  **インデントロジックの変更**
    - 現在の `marginLeft: level * 24px` による絶対的なインデントから、ネストされたコンテナによる相対的なインデント構造へ変更します。
    - これにより、CSSの親要素のスタイル（ボーダー等）が自然に適用されるようにします。
    - `DragDropContext` との整合性を保つため、ドラッグ中のスタイル崩れに注意します。

3.  **視覚的グループ化**
    - 展開された子フォルダ群に対して、わずかな背景色の変化や、左端のアクセントカラーを追加し「ここからここまでがこのフォルダの中身」と直感的に分かるようにします。

## 検証計画

### 手動テスト
1.  **表示確認**
    - 深い階層（3階層以上）を持つフォルダ構造を作成し、ツリー線が正しくつながって見えるか確認する。
    - フォルダの展開・折りたたみを行い、アニメーションや表示の切り替えがスムーズか確認する。

2.  **操作確認**
    - ドラッグ＆ドロップでフォルダを移動（階層変更）した際、新しい階層構造に合わせてツリー線が再描画されるか確認する。
    - 親フォルダを選択した際、子フォルダが含まれていることが視覚的に違和感なく伝わるか確認する。

3.  **レスポンシブ確認**
    - モバイル画面において、インデントが深くなりすぎて画面外にはみ出さないか、あるいは横スクロール等で対応できているか確認する。
