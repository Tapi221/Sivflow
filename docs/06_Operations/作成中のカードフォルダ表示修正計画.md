# 学習ダッシュボードのフォルダ表示不整合修正計画

学習ダッシュボードの「作成中のカード（下書き）」セクションにおいて、カードの所属フォルダが「無所属」と表示される不具合、および実際のデータとの不整合を解消します。

## 現状の課題と原因
- **原因1**: `Dashboard.jsx` で `draftCards` を抽出する際、`activeCards` とは異なり「所属フォルダが削除されていないか」のチェックが漏れています。そのため、フォルダが削除されても下書きカードが残り、「無所属」と表示されます。
- **原因2**: フォルダ名取得時に `folderId` と `folder_id`、`folderName` と `folder_name` が混在して参照されており、正規化（normalization）の恩恵を十分に受けていません。
- **原因3**: `CardEditor` や `createCard` において、`folderId` が欠落した場合に `''`（空文字列）が設定される仕様があり、これが「無所属」を生む温床となっています。

## 変更内容

### 1. `Dashboard.jsx` の修正
- `draftCards` の定義を `activeCards` ベースに変更し、有効な（削除されていない）フォルダに属するカードのみを表示するようにします。
- フォルダ名取得ロジックを整理し、正規化されたプロパティ (`id`, `folderName`) のみに基づくようにします。
- 「前回の続き」セクションのフォルダ特定ロジックも、正規化されたプロパティを使用するように修正します。

## 修正対象ファイル

## 検証プラン

### 手動検証
1. 任意のフォルダに下書きカード（Draft）を作成する。
2. ダッシュボードでそのカードが正しいフォルダ名で表示されていることを確認する。
3. そのフォルダを削除する。
4. ダッシュボードの「作成中のカード」リストからそのカードが消える（または「削除済みフォルダ」として適切に扱われる）ことを確認する。
5. 複数のフォルダにまたがる下書きカードが、それぞれの正しいフォルダ名を表示しているか確認する。

> [!IMPORTANT]
> 「無所属」という表示は、システム上 `folderId` が空または紐づくフォルダが見つからない（削除済み等）場合に発生します。今回の修正で、通常の利用範囲内では「無所属」が表示されないようにガードを強化します。
