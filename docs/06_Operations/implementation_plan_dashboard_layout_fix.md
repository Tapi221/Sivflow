# 実装計画：ダッシュボードレイアウトの最適化

学習ダッシュボードの「前回の続き（Resume Learning）」と「作成中のカード（Drafts in Progress）」のセクションを、PC表示時に横並びになるように調整します。

## 変更内容

### 1. [Dashboard.jsx](file:///c:/FlashcardMaster/src/Pages/Dashboard.jsx) のレイアウト調整
- 「Resume Learning」セクション（`isLoading ? ... : lastStudiedFolder && ...`）と「Drafts in Progress」セクション（`isLoading ? ... : ( ... )`）を、レスポンシブなグリッドコンテナでラップします。
- `grid grid-cols-1 lg:grid-cols-[1fr_2fr]` のようなクラスを適用し、PC表示（`lg`以上）では「前回の続き」を狭く左側に、「作成中のカード」を広く右側に配置します。
- モバイル表示では引き続き縦に並ぶようにします。
- レイアウト変更に伴い、セクション間のマージンや余白を微調整します。

## 検証プラン
- PCブラウザでダッシュボードを表示し、2つのセクションが横並びに表示されることを確認。
- ブラウザの幅を狭めて、モバイル表示時に縦並びに戻ることを確認。
- カードのドラフトが存在する場合と存在しない場合の両方で、レイアウトが崩れないことを確認。
