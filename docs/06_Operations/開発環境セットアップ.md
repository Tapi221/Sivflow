# 開発環境セットアップガイド

Flash Master の開発環境をセットアップする手順です。

## 前提条件

- Node.js 18.x 以上
- npm または yarn
- Git
- Firebase CLI（Firebase 機能を使用する場合）

---

## 1. プロジェクトのクローン（初回のみ）

```bash
git clone <repository-url>
cd FlashcardMaster
```

---

## 2. 依存関係のインストール

```bash
npm install
# または
yarn install
```

---

## 3. 環境変数の設定

### 3.1 環境変数ファイルの作成

プロジェクトルートに `.env.local` ファイルを作成します。

```bash
cp .env.example .env.local
```

### 3.2 環境変数の設定

`.env.local` に Firebase の設定値を記入します。

```env
VITE_FIREBASE_API_KEY=your_api_key
VITE_FIREBASE_AUTH_DOMAIN=your_project.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=your_project_id
VITE_FIREBASE_STORAGE_BUCKET=your_project.appspot.com
VITE_FIREBASE_MESSAGING_SENDER_ID=your_sender_id
VITE_FIREBASE_APP_ID=your_app_id
VITE_STRIPE_PUBLISHABLE_KEY=your_stripe_key
```

**注意**: `.env.local` は Git にコミットしないでください（`.gitignore` に含まれていることを確認）。

---

## 4. Firebase プロジェクトのセットアップ

### 4.1 Firebase CLI のインストール

```bash
npm install -g firebase-tools
```

### 4.2 Firebase へのログイン

```bash
firebase login
```

### 4.3 Firebase プロジェクトの初期化

```bash
firebase init
```

以下のオプションを選択：
- Firestore: Yes（データベースルールとインデックス設定）
- Functions: Yes（Cloud Functions を使用する場合）
- Hosting: Yes
- Storage: Yes

### 4.4 Firebase エミュレータのセットアップ（オプション）

ローカル開発用に Firebase エミュレータを設定する場合：

```bash
firebase init emulators
```

必要なエミュレータを選択：
- Authentication Emulator
- Firestore Emulator
- Storage Emulator
- Functions Emulator

---

## 5. 開発サーバーの起動

```bash
npm run dev
# または
yarn dev
```

ブラウザで `http://localhost:5173` にアクセスします（Vite のデフォルトポート）。

---

## 6. ビルド

### 6.1 開発ビルド

```bash
npm run build
# または
yarn build
```

### 6.2 本番ビルド

```bash
npm run build -- --mode production
```

ビルド結果は `dist/` ディレクトリに出力されます。

---

## 7. リンター・フォーマッターの設定

### 7.1 ESLint

```bash
npm run lint
# または
yarn lint
```

### 7.2 Prettier

```bash
npm run format
# または
yarn format
```

---

## 8. テストの実行（実装後）

### 8.1 単体テスト

```bash
npm run test
# または
yarn test
```

### 8.2 E2Eテスト

```bash
npm run test:e2e
# または
yarn test:e2e
```

---

## 9. Firebase エミュレータの起動（オプション）

ローカル開発で Firebase エミュレータを使用する場合：

```bash
firebase emulators:start
```

エミュレータの UI は `http://localhost:4000` で確認できます。

---

## 10. デプロイ

### 10.1 Firebase Hosting へのデプロイ

```bash
npm run build
firebase deploy --only hosting
```

### 10.2 Cloud Functions のデプロイ

```bash
firebase deploy --only functions
```

### 10.3 すべてデプロイ

```bash
firebase deploy
```

## トラブルシューティング

### 問題: 環境変数が読み込まれない

**解決策**:
- `.env.local` ファイルがプロジェクトルートにあることを確認
- 環境変数名が `VITE_` で始まっていることを確認
- 開発サーバーを再起動

### 問題: Firebase 認証エラー

**解決策**:
- Firebase プロジェクトの設定を確認
- `.env.local` の値が正しいか確認
- Firebase Console で認証方法が有効になっているか確認

### 問題: ビルドエラー

**解決策**:
- TypeScript の型エラーを確認
- `npm run lint` でコードのエラーを確認
- 依存関係を再インストール: `rm -rf node_modules && npm install`

### 問題: ポートが既に使用されている

**解決策**:
- 別のポートを指定: `npm run dev -- --port 3000`
- または `vite.config.ts` でポートを変更

## 開発ワークフロー

1. 最新のコードを取得: `git pull`
2. 新しいブランチを作成: `git checkout -b feature/your-feature`
3. 開発サーバーを起動: `npm run dev`
4. コードを記述・テスト
5. リンターを実行: `npm run lint`
6. コミット: `git commit -m "your message"`
7. プッシュ: `git push origin feature/your-feature`
8. プルリクエストを作成
