# 編集画面プレビューと閲覧モードの表示統一

## 概要
編集画面の「プレビュー」機能において、実際の閲覧モード（Viewer）や学習時のカード表示と見た目が異なる問題を解消しました。
また、編集画面（2カラム）と閲覧画面（全幅）でテキストやコードブロックの表示幅が異なり、可読性や編集時のイメージとのギャップが生じていた問題を修正しました。

## 変更点

### 1. テキスト・コードブロックの幅固定化
- **対象ファイル**: `src/Components/card/Flashcard.tsx`
- **変更内容**:
    - `MathRenderer`（テキストブロック）と `CodeRenderer`（コードブロック）を `max-w-2xl` (約672px) のコンテナでラップし、中央寄せ (`mx-auto`) に設定しました。
    - これにより、閲覧画面でカード幅が広がっても、文章やコードは読みやすい幅（編集画面のカラム幅相当）に維持されます。

### 2. プレビューダイアログのスタイル修正
- **対象ファイル**: `src/Components/card/CardEditor.tsx`
- **変更内容**:
    - プレビュー表示用の `DialogContent` から、ガラスモーフィズム風の背景や余分なパディング、ボーダーを削除しました。
    - `Flashcard` コンポーネントを `bg-transparent` なダイアログ内に直接配置し、カード本来のデザイン（白背景、シャドウ、角丸）のみが表示されるようにしました。
    - 閉じるボタン（×）の位置を調整し、カードに干渉しないように配置しました。

## 検証結果

### プレビュー表示
- **確認事項**: 編集画面で「プレビュー」ボタンを押下。
- **結果**: フォルダ閲覧画面でカードをクリックした際のポップアップ表示とほぼ同一の見た目（背景透過、カードが浮いている状態）になることを確認。

### ブロック幅
- **確認事項**:
    - デスクトップで閲覧画面を表示。
    - 長文のテキストやコードが含まれるカードを確認。
- **結果**:
    - テキスト・コードが画面端まで広がらず、中央に適切な幅で収まっていることを確認。
    - モバイル表示では画面幅に合わせてリサイズされることを確認。
