# リンクブロックの仕様変更実装計画

リンクブロックにおいて、一つのブロック内に複数のリンクを追加可能にし、かつブロックの並び順に関わらず常に最下部に表示されるように変更します。

## 変更内容

### 1. リンクブロック内での複数リンク対応
`ReferenceBlock.tsx` を修正し、すでにリンクが存在する場合でも新しいリンクを追加できるようにします。

- [MODIFY] [ReferenceBlock.tsx](file:///c:/FlashcardMaster/src/Components/card/blocks/ReferenceBlock.tsx)
    - リンク追加ボタンの表示条件（`references.length === 0`）を削除。
    - 複数のリンクがある場合でも見やすいよう、追加ボタンの配置やスタイルを調整。

### 2. リンクブロックの最下部固定
`BlockEditor.tsx` および `CardEditor.tsx` を修正し、リンクブロックが常にリストの最後に位置するようにします。

- [MODIFY] [BlockEditor.tsx](file:///c:/FlashcardMaster/src/Components/card/BlockEditor.tsx)
    - ブロックのレンダリング時に、リンクブロックをフィルタリングして最後に描画する、あるいはドラッグ＆ドロップのインデックス計算を調整します。
    - リンクブロック自体のドラッグを無効化（常に最下部のため）。
- [MODIFY] [CardEditor.tsx](file:///c:/FlashcardMaster/src/Components/card/CardEditor.tsx)
    - `onDragEnd` のロジックを修正し、他のブロックをリンクブロックより下に移動できないようにします。
    - データの保存時や更新時に、リンクブロックが必ず最後にくるよう `orderIndex` を再計算します。

## 検証計画
1. リンクブロックを追加し、URLと表示名を入力する。
2. 「＋」ボタンが表示され続け、2つ目、3つ目のリンクを追加できることを確認する。
3. 他のブロック（テキスト、画像など）を追加し、それらをドラッグ＆ドロップで移動させても、リンクブロックが常に一番下にあることを確認する。
4. リンクブロックを削除し、再度追加しても一番下に表示されることを確認する。
5. プレビュー画面で複数のリンクが正しく表示されることを確認する。
