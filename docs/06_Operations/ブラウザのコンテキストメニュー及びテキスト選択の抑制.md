# 技術ガイド：ブラウザのコンテキストメニューおよびテキスト選択の抑制方法

## 概要
ボタンやインタラクティブな要素において、高速な連続クリックや長押しが想定される場合、ブラウザ標準のコンテキストメニュー（右クリックメニュー）やテキスト選択機能がユーザー体験（UX）を阻害することがあります。
本ドキュメントでは、これらを抑制するための標準的な実装方法を記載します。

## 1. コンテキストメニュー（右クリックメニュー）の抑制

Reactコンポーネントにおいて、`onContextMenu` イベントを使用して標準動作をキャンセルします。

### 実装方法
```tsx
<button
  onContextMenu={(e) => e.preventDefault()}
  onClick={handleClick}
>
  クリックしてもメニューが出ないボタン
</button>
```

### 用途
- カード作成ボタンなどの「連打」が想定される箇所。
- カスタム右クリックメニューを実装したい箇所。
- モバイル端末での「長押し」によるメニュー表示を防止したい箇所。

## 2. テキスト選択の抑制

ダブルクリックやトリプルクリック時に要素内のテキストが青く反転するのを防止します。

### 実装方法（Tailwind CSS / utility class）
`select-none` クラスを要素に追加します。

```tsx
<div className="select-none">
  選択できないテキスト
</div>
```

### 実装方法（Vanilla CSS）
```css
.prevent-select {
  -webkit-user-select: none; /* Safari */
  -ms-user-select: none;     /* IE 10 and older */
  user-select: none;         /* Standard syntax */
}
```

### 用途
- ボタン、タブ、アイコンなどのUI部品。
- ドラッグ＆ドロップのハンドル部分。

## 3. 実装上の注意点

> [!WARNING]
> インターネットの標準的な挙動（右クリックでリンクを新しいタブで開く、テキストをコピーするなど）を過度に制限すると、アクセシビリティや利便性が低下する恐れがあります。
> 以下の判断基準で適用を検討してください。

- **適用すべきケース**:
  - 純粋なアクションボタン（保存、追加、削除など）。
  - ゲーム性の高い UI 要素。
  - 独自のコンテキストメニューを持つ UI。
- **適用を避けるべきケース**:
  - 記事や説明文などのテキストコンテンツ。
  - 通常のハイパーリンク。

## 4. 適用事例

- `CardEditor.tsx`: 「続けて作成」ボタン、「保存」ボタンにおける連続クリック時のストレス軽減のために導入。
