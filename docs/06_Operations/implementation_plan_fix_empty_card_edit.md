# 不具合修正：カード編集画面の内容が空になる問題の解決

カードの内容が存在するにもかかわらず、編集ボタンを押した際にエディタが空（「ブロックを追加してください」の状態）で表示される不具合を修正します。

## 原因調査の結果

`CardEditorPane.tsx` における `draft`（編集用の一時状態）の初期化ロジックに問題（レースコンディション）があることが判明しました。

1. `selectedCardId` が切り替わった直後に `useEffect` が実行される。
2. このとき、`useCards()` フックがまだデータベースから最新の `cards` 一覧を取得しきれていない場合（非同期のため）、`selectedCard` が `null` になる。
3. `useEffect` は `selectedCardId` のみに依存しているため、その後 `cards` がロードされて `selectedCard` が非 `null` になっても、`draft` の再初期化が行われない。
4. その結果、`draft` は `null` のままとなり、レンダリング時のフォールバック処理 (`initNewDraft()`) によって空のカードとして表示されてしまう。

## 修正内容

### [CardEditorPane.tsx](file:///c:/FlashcardMaster/src/Components/folder/CardEditorPane.tsx)

#### 1. `useEffect` の依存配列とロジックの改善
- `useEffect` の依存配列に `selectedCard` を追加します。
- **データ上書きの防止**: すでに編集中の場合 (`isEditing === true`) は、外部（同期など）からのデータでドラフトを勝手に上書きしないようガードを設けます。
- `selectedCardId` が完全に切り替わった場合（フォルダを跨いだ移動など）は、強制的にリセットします。

#### 2. 編集開始時の安全な初期化
- 閲覧画面から「編集」ボタンが押された際、もし `draft` が最新の `selectedCard` と同期されていない（レースコンディションで `null` だった）場合は、その時点で同期を行うロジックを追加します。

## 検証計画

### 手動確認
1. **カード選択直後の編集**:
   - カードを一覧から選び、表示された直後（または表示されるのとほぼ同時）に「編集」ボタンを押し、内容が正しく表示されることを確認。
2. **フォルダ切り替え時の挙動**:
   - 別のフォルダに移動し、そこにあるカードを選択して編集モードに入った際、前のカードの内容が残っていないこと、および新しいカードの内容が正しく表示されることを確認。
3. **新規作成時の挙動**:
   - 「新規カードを作成」をクリックした際は、正しく空の状態で編集が開始されることを確認。

### 自動検証
- `npm run build` で型エラーが発生しないことを確認。
