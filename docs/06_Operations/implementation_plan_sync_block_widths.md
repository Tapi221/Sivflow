# 閲覧画面でのテキスト・コードブロック幅固定化の実装計画

## 概要
ユーザー要望「閲覧画面（Viewer）のカード表示において、テキストウィンドウとコードウィンドウの幅を、編集画面（Editor）と同じように固定する」に対応します。
編集画面はデスクトップで2カラムレイアウト（約650px）を採用していますが、閲覧画面はカード幅いっぱいに広がるため、可読性や見た目の統一感が損なわれています。
閲覧画面の該当ブロックに対して最大幅（`max-w-2xl` ≒ 672px）を設定し、編集画面のウィンドウ幅感を再現します。

## 変更内容

### Components/card/Flashcard.tsx

`renderBlocks` 関数内のレンダリングロジックを変更します。

1.  **テキストブロック (TextBlock / MathRenderer)**
    - 現状: `MathRenderer` が直接配置されている（親の flex 設定により中央揃え、幅制限なし）。
    - 変更後: `div.w-full.max-w-2xl.mx-auto.text-left` でラップし、最大幅を制限して中央に配置します。テキスト自体は左揃えにします（長文の可読性向上とエディタとの整合性のため）。

2.  **コードブロック (CodeBlock / CodeRenderer)**
    - 現状: `div.text-left.w-full` でラップされている。
    - 変更後: `div.text-left.w-full.max-w-2xl.mx-auto` に変更し、最大幅を制限します。

### 補足
- `max-w-2xl` (42rem = 672px) は、編集画面の `max-w-7xl` (1280px) の半分のカラム幅（padding考慮）に近似しており、一般的な読みやすい行長（Prose）としても適切です。
- モバイル（画面幅が 672px 未満）の場合は `w-full` が優先されるため、レイアウト崩れは起きません。

```tsx
// 変更イメージ (Flashcard.tsx)

// Text Block
<div className="w-full max-w-2xl mx-auto text-left"> {/* Wrapper追加 */}
  <MathRenderer ... />
</div>

// Code Block
<div className="text-left w-full max-w-2xl mx-auto"> {/* クラス変更 */}
  <CodeRenderer ... />
</div>
```

## 検証計画

### 手動検証
1. **デスクトップ表示 (幅広画面)**
   - カード編集画面（2カラム）でテキストとコードを入力し、見た目（改行位置など）を確認。
   - フォルダ詳細画面へ戻り、該当カードをクリックして閲覧モードにする。
   - テキストとコードのブロックがカード全幅に広がらず、編集画面と同程度の幅（中央寄せ）で表示されることを確認する。
   - `max-w-2xl` 以上の幅を持つ画像を貼り付けた場合など、他のブロックとのバランスも確認（画像は今回の変更対象外だが、レイアウトへの影響を見る）。

2. **モバイル表示**
   - 画面幅を狭めた際、`max-w-2xl` の影響を受けず、画面幅に合わせてリサイズされることを確認する。
