# カード閲覧時のテキスト選択防止機能の実装

## 概要
カード閲覧モード（学習画面およびプレビュー画面）において、テキスト選択を無効化する修正を行いました。これにより、スワイプ操作などの誤操作を減らし、アプリライクなUXを提供します。

## 変更内容
- **ファイル:** `src/Components/card/Flashcard.tsx`
- **変更:**
    - ルート要素に `select-none` クラスを追加し、カード全体を選択不可にしました。
    - 個別の要素（タグ、回答・質問エリア）に設定されていた `select-text` クラスを削除しました。

## 確認方法
1. **アプリを起動**
   ```bash
   npm run dev
   ```
2. **学習モードでの確認**
    - 任意のフォルダの「学習を開始」ボタンを押す。
    - カードが表示されたら、テキスト（質問文、数式など）を長押しまたはドラッグして選択を試みる。
    - **期待結果:** テキストが選択されないこと。
3. **プレビューでの確認（編集画面）**
    - カード編集画面を開く。
    - 右下の「プレビュー」ボタン（または目のアイコン）をクリック。
    - プレビューダイアログ内のカードでテキスト選択を試みる。
    - **期待結果:** テキストが選択されないこと。
4. **メモ・コードブロックの確認**
    - メモやコードが含まれるカードを表示し、それらも選択できないことを確認する。

## 注意点
- リンク（Reference）が含まれる場合、クリックは可能ですが、テキスト選択はできないようになっています（`ReferencePopup` はポータル表示のため影響を受けない可能性がありますが、カード本体のリンクボタンは選択不可です）。
- 編集画面（入力フォーム）は `Flashcard` コンポーネントを使用していないため、通常通りテキスト選択・編集が可能です。
