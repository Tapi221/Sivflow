# 耐性スコア分布のデザイン刷新計画

耐性スコア分布チャートの美観を向上させ、アプリ全体のモダンでプレミアムなデザインシステムに適合させます。

## 現状の課題
- 彩度の高い原色に近い色が使用されており、近年の洗練されたUI（パステルやニュートラルなトーン）と乖離している。
- グラフのラベルや軸がやや煩雑。
- 下部のフェーズバーのデザインが少し「詰め込みすぎ」な印象。

## 提案する変更

### [Component] StatsCharts.tsx

#### [MODIFY] [StatsCharts.tsx](file:///c:/FlashcardMaster/src/Components/stats/StatsCharts.tsx)
- **カラーパレットの刷新**:
    - 原色系から、プライマリカラー（#689A98）を基調とした、透明感のある洗練されたカラー（セージ、ミント、ティール、スレートなど）に変更。
- **グラフ（Recharts）のスタイル調整**:
    - `Bar` の角丸（radius）を調整し、よりソフトな印象に。
    - `ReferenceLine`（平均スコア）のデザインを、単純な線からアイコンやバッジを含むカスタムマーカーにアップグレード。
    - `Tooltip` をグラスモーフィズム（backdrop-blur）を適用したフローティングスタイルに変更。
- **フェーズバーの洗練**:
    - 境界線をなくし、滑らかなグラデーション背景を採用。
    - テキストを小さめに整え、余裕のあるパディングを確保。
    - ホバーエフェクトを追加。

## デバイス別の調整
- **モバイル**: フォントサイズを小さく保ちつつ、タップ可能な領域（ツールチップの反応など）を確保。
- **デスクトップ**: グラフのアニメーション（duration）を最適化し、ホバー時のインタラクションを豊かに。

## 検証計画

### 自動テスト
- 現状、UIコンポーネントのビジュアルテストはないため、ビルドエラーが発生しないことを確認します。
  ```ps1
  npm run build
  ```

### 手動確認
1. 統計画面にアクセスし、耐性スコア分布チャートが表示されることを確認。
2. グラフの各バーにホバーした際、洗練されたツールチップが表示されるか確認。
3. 平均スコアのラインがデザインに馴染んでいるか確認。
4. ダークモード（もし有効なら）やライトモードでの見え方を確認。
