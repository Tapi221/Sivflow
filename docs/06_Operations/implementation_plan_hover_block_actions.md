# ブロックアクションボタンの表示制御改善計画

## 目標
一問一答モード（およびカード編集全般）において、ブロックのアクションボタン（移動、複製、削除）が常に表示されて重なってしまう問題を解決するため、マウスホバー時のみ表示されるように修正する。

## 現状の課題
- `BlockWrapper.tsx` において、モバイルファーストのアプローチで `opacity-100` がデフォルト（常に表示）となっている。
- デスクトップ (`md`) では `md:opacity-0 md:group-hover:opacity-100` となっているはずだが、ユーザー環境（または特定のモード）で意図通りに機能していない、あるいはモバイル表示時の「重なり」が問題視されている可能性がある。
- 特にブロックの高さが低い場合（1行テキストなど）、右側にフローティングしているボタンが下のブロックに重なる問題が発生している。

## 変更内容

### `src/Components/card/blocks/BlockWrapper.tsx`

ボタンコンテナのクラスを修正し、ホバー時の表示制御を強化する。

**変更前:**
```tsx
className="absolute -top-3 -right-1 md:-right-12 md:top-4 flex md:flex-col gap-1.5 opacity-100 md:opacity-0 md:group-hover:opacity-100 transition-all z-20"
```

**変更後案:**
デフォルトで非表示 (`opacity-0`) とし、グループホバー時 (`group-hover:opacity-100`) に表示する形に統一する。
ただし、タッチデバイス（モバイル）でのユーザビリティを考慮し、以下のように調整する。

1.  **基本方針**: `opacity-0 group-hover:opacity-100` をベースにする。
2.  **モバイル対応**: タッチデバイスでは「ホバー」がないため、タップでフォーカスした際や、あるいは常に表示するが位置を調整する等の考慮が必要だが、今回の要望は「重なって全て表示されるのを防ぐ」ことであるため、**原則ホバー（PC）およびインタラクション時のみ表示**とするのが安全。
3.  **フォーカス時対応**: `group-focus-within:opacity-100` を追加することで、テキスト入力中などはボタンを表示させる（利便性のため）。これにより、モバイルでも入力しようとタップすればボタンが出る。

**修正後のクラス構成:**
```tsx
className={cn(
  "absolute flex transition-all z-20",
  // 位置設定 (既存維持)
  "-top-3 -right-1 md:-right-12 md:top-4 md:flex-col gap-1.5",
  // 表示制御 (変更)
  "opacity-0 group-hover:opacity-100 group-focus-within:opacity-100", 
  // モバイルで常に表示されていたのを廃止し、フォーカス時またはホバー時に限定
)}
```

**注意点**:
モバイルで `group-hover` は「タップした瞬間」に機能することが多いが、直感的でない場合がある。`focus-within` を入れることで「編集しようと触ったブロック」のボタンが出るようになるため、UXとしては自然になるはず。

## 検証計画

### 手動検証
1.  **デスクトップ表示**:
    - 複数のブロック（短いテキストブロック等）を追加する。
    - マウスを動かしていない状態でボタンが非表示であることを確認する。
    - マウスをブロックに乗せた時だけボタンが表示されることを確認する。
    - ブロックにフォーカス（入力）した時にボタンが表示されることを確認する。
2.  **モバイル表示（レスポンシブエミュレーション）**:
    - 画面幅を狭めた状態で確認。
    - デフォルトでボタンが非表示であることを確認。
    - ブロックをクリック/タップ（フォーカス）した時にボタンが表示されるか確認する。

## 実行確認
変更箇所は `BlockWrapper.tsx` のみ。
テストコードの修正は不要。
