# ブロックボタンコンテナの背景サイズ最適化

## 概要
カード編集画面のブロック追加ボタン群において、アイコン表示時（またはボタン数が少ない時）に背景のグレーの領域が必要以上に広がってしまう問題を修正します。コンテナの幅をコンテンツに合わせて自動調整（`w-fit`）するように変更します。

## 変更内容

### Components/card/BlockEditor.tsx

- ブロック追加ボタンを囲む `div` コンテナに `w-fit` クラスを追加します。
- これにより、Flexboxの親要素（特にモバイル表示時の `flex-col`）内でコンテナが全幅にストレッチされるのを防ぎます。

```tsx
// 変更前
<div className="flex flex-nowrap items-center gap-1.5 md:gap-2 bg-slate-100/50 ... max-w-full overflow-x-auto ...">

// 変更後
<div className="flex flex-nowrap items-center gap-1.5 md:gap-2 bg-slate-100/50 ... max-w-full w-fit overflow-x-auto ...">
```

## 検証計画

### 自動テスト
- 現状、UIの見た目に関する厳密な自動テストケースはないため、手動検証を行います。

### 手動検証
1. **モバイル表示 / ウィンドウ幅を縮小**
   - ブラウザのウィンドウ幅を狭めてモバイルレイアウト（縦並び）にします。
   - ブロックボタンの表示設定でラベルを非表示にする、または画面幅によりラベルが消える状態にします。
   - ボタン群の背景グレー領域が、ボタンの並び幅に合わせて短くなっていることを確認します（以前は右端まで伸びていた）。
   - スクロールが必要なほどボタンが多い場合（画面幅より広い場合）は、`max-w-full` が効いて画面幅に収まり、横スクロールできることを確認します。

2. **デスクトップ表示**
   - ウィンドウ幅を広げてデスクトップレイアウト（横並び）にします。
   - レイアウトが崩れていないことを確認します。
