# 未使用.tsファイル特定計画

プロジェクト内の`.ts`ファイルの中で、他のファイルから一切参照されていないものを特定し、安全に削除できる可能性のある候補リストを作成します。

## 調査対象
- `src` および `functions` ディレクトリ配下の `.ts` ファイル（および関連する `.tsx`, `.js`, `.jsx` からの参照）
- ただし、以下のファイルは除外します：
    - `*.d.ts` (型定義ファイル)
    - `*.spec.ts`, `*.test.ts` (テストファイル)
    - ファイル内に `// @keep`, `// preserved`, `// @no-delete` などの明示的に残す旨のコメントがあるもの

## 調査手順

1. **環境設定の確認**:
   - `tsconfig.json` (src配下) および `functions/tsconfig.json` の `paths` 設定を確認し、エイリアス（例: `@/*`, `~/*`）を抽出します。

2. **対象ファイルの抽出**:
   - `find_by_name` を使用して、プロジェクト内の `.ts` ファイルをリストアップし、除外条件に当てはまるものをフィルタリングします。

3. **参照の確認 (多角的な検索)**:
   - 各ファイルについて、以下の方法でプロジェクト全体から参照されているかを確認します。
   - **ケースインセンシティブ** (`-i`) で検索を実行。
   - **インポートパスの解決**:
     - 相対パス (`../`, `./`)
     - `tsconfig.json` のエイリアスパス
     - 拡張子なしのインポート
     - `index.ts` 経由の参照（ディレクトリ名のみの指定）
   - **動的インポート**: `import()` や `require()` 内の文字列。
   - **コメント精査**: コメント内のファイル名のみの出現は「参考情報」として扱い、機能的な参照と区別します。

4. **連鎖的な未使用状態の特定 ( daisy-chain )**:
   - 「AをインポートしているBも未使用」というケースを再帰的に特定し、削除候補を拡張します。

5. **エントリポイントからの追跡**:
   - `src/main.tsx`, `src/App.tsx`, `functions/src/index.ts` などの主要なエントリポイントから到達不可能なファイルを「高リスク候補」としてマークします。

6. **結果の整理と出力**:
   - 以下の情報を付与した報告書（JSON/CSV的な形式でも提示）を作成します：
     - `Path`: ファイルパス
     - `Type`: 参照なし / 動的参照の可能性あり / index経由のみ
     - `Risk`: 高（エントリポイントから到達不可） / 中 / 低
     - `Details`: 検出された参照箇所（あれば）

## 注意事項
- **ビルドによる検証**: 削除候補を除外した状態で `npm run build` を実行し、型エラーや依存エラーが発生しないかを確認します。
- **Gitブランチの使用**: 作業は専用のブランチ（または一連のコミット）で行い、容易にロールバックできるようにします。
- **動的インポートの注釈**: grepで完全に特定できない動的・動的なパス生成によるインポートについては注釈を付けます。

## 注意事項
- このタスクでは**ファイルの削除は行いません**。あくまで候補リストの作成までを目的とします。
- 動的なインポートや、外部ツールから間接的に参照されているファイル（設定ファイル等）については慎重に判断します。

## 検証計画
### 手動検証
- リストアップされたファイルが本当にどこからも参照されていないか、上位のファイルから順に再確認します。
