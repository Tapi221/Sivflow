# 新規フォルダ作成画面のモバイル表示最適化 実施報告

## 変更概要
モバイルデバイス（特にiOS）において、キーボード表示時にダイアログが隠れてしまう問題を修正しました。`window.visualViewport` APIを使用してキーボードの開閉を検知し、ダイアログの位置を自動的に調整するようにしました。

## 実施内容

### 1. `FolderDialog.tsx` の修正
- **「クラウド同期」オプションの削除**:
  - 新規作成画面からはトグルを削除し、設定画面で一括または個別管理するように変更。
- **「キャンセル」ボタンの削除と「✕」ボタンへの統合**:
  - フッターのキャンセルボタンを削除。右上の「✕」マークで閉じる操作に統一しました。
- **`visualViewport` による位置調整**:
  - 引き続き、キーボード表示時に画面上端へスライドする機能を維持しています。
- **デザインの微調整**:
  - クラウド同期トグル部分の余白や背景色を調整し、提供された画像のレイアウトに近づけました。

```tsx
// 修正箇所の抜粋

useEffect(() => {
  if (!window.visualViewport) return;
  const handleResize = () => {

// DialogContentへのスタイル適用
```

## 確認方法
1. モバイル端末（またはモバイルシミュレーション）で「新規フォルダ作成」ダイアログを開く。
2. 入力フォームをタップしてキーボードを表示させる。
3. ダイアログがアニメーションして上に移動し、入力欄が隠れずに表示されることを確認する。
4. キャンセルボタンが左、作成ボタンが右にあることを確認する。
5. キーボードを閉じると、元の位置に戻ることを確認する。
