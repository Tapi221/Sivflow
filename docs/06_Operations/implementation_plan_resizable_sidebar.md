# 実装計画: リサイズ可能なサイドバーの実装

## ゴール
1. サイドバーとメインコンテンツの境界線をドラッグして、サイドバーの幅を自由に調整できるようにする。
2. 境界線は「一本の線」として表現し、視覚的にシンプルに保つ。
3. 調整した幅は永続化（LocalStorage保存）し、次回アクセス時も維持する。

## User Review Required
> [!NOTE]
> 最小幅は現在のアイコン表示が崩れない `72px`、最大幅は `400px` 程度に制限する予定です。

## Proposed Changes

### レイアウト修正
#### [MODIFY] [Layout.tsx](file:///c:/FlashcardMaster/src/Layout.tsx)
- **State追加**:
  - `sidebarWidth`: サイドバーの幅を管理（初期値 `72px`、LocalStorageから読み込み）。
  - `isResizing`: ドラッグ中かどうかを管理。
- **リサイズロジック追加**:
  - `startResizing`: マウスダウン時にリサイズ開始。
  - `resize`: マウス移動時に幅を更新。
  - `stopResizing`: マウスアップ時にリサイズ終了＆保存。
  - `useEffect` で `mousemove`, `mouseup` をウィンドウ全体で監視。
- **サイドバー(`aside`)のスタイル変更**:
  - `w-[72px]` 固定を廃止し、`style={{ width: sidebarWidth }}` で動的に制御。
  - 右端にリサイズ用のボーダーライン（兼ドラッグハンドル）を追加。
- **メインコンテンツ(`main`)のスタイル変更**:
  - `md:ml-[72px]` 固定を廃止し、`style={{ marginLeft: isSidebarOpen ? sidebarWidth : 0 }}` で動的に制御。

### スタイル詳細
- リサイズハンドルは `w-[1px]` の線として表示し、ホバー時やドラッグ時に色を変更して操作可能であることを示す（例: `hover:bg-primary-600`）。
- カーソルは `col-resize` (または `ew-resize`) を使用。

## Verification Plan

### Manual Verification
1. **リサイズ操作確認**:
   - サイドバーの右境界線にカーソルを合わせ、リサイズカーソルが表示されるか確認。
   - ドラッグしてサイドバーの幅が変わるか確認。
   - メインコンテンツの開始位置がサイドバーの幅に合わせてスムーズに追従するか確認。
2. **最小・最大幅確認**:
   - 最小幅（72px）以下にならないこと。
   - 最大幅（設定値）以上にならないこと。
3. **永続化確認**:
   - 幅を変更した後、ページをリロードして幅が維持されているか確認。
4. **トグル動作確認**:
   - サイドバーを閉じた後、再度開いたときに最後に設定した幅で開くか確認。
