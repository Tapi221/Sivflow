# 実装計画：フォルダー移動の階層制限

フォルダーツリーのドラッグ＆ドロップ操作において、階層の深さを制限し、ルートフォルダーの誤操作を防ぎます。

## 課題

- 現在、フォルダーを無制限に深い階層に移動できるため、管理が複雑になる可能性がある
- ルートフォルダー（最上位のフォルダー）を誤って他のフォルダーの子にしてしまう可能性がある

## 要件

1. **最大階層の深さ制限**: 現在最も深い階層のフォルダーは、それ以上深い階層に移動できないようにする
2. **ルートフォルダーの保護**: 親フォルダーがないフォルダー（ルートフォルダー）は、他のフォルダーの子にできないようにする

## 変更内容

### [FolderTree](file:///c:/FlashcardMaster/src/Components/folder/FolderTree.tsx)

`handleDragEnd` 関数内に以下のチェックロジックを追加します：

1. **現在の最大階層深度を計算する関数を追加**
   - 全フォルダーを走査して、最も深い階層レベルを特定

2. **ドラッグ中のフォルダーの現在の階層レベルを計算**
   - ルートから対象フォルダーまでの親をたどって階層レベルを取得

3. **移動先の階層レベルを計算**
   - 移動先の親フォルダーの階層レベル + 1

4. **移動制限のチェック**
   - ドラッグ中のフォルダーが現在最大階層にある場合、それ以上深い階層への移動を禁止
   - ドラッグ中のフォルダーが現在ルートレベル（親なし）の場合、他のフォルダーの子への移動を禁止

5. **制限に引っかかった場合の処理**
   - 早期リターンで移動をキャンセル
   - 必要に応じてユーザーへの通知（トーストなど）

## 検証計画

- [ ] ルートフォルダーを他のフォルダーの子にドラッグしても移動できないことを確認
- [ ] 制限に引っかからない通常の移動は正常に動作することを確認
- [ ] 複数の階層が存在する場合でも正しく動作することを確認
