# プレビュー表示不具合の修正計画

## 概要
カード作成画面のプレビューにおいて、内容が表示されない不具合を修正します。`Flashcard` コンポーネントがブロック形式のデータを考慮せずにレガシーなフィールドのみをレンダリングしていることが原因です。

## Proposed Changes

### 1. 正規化ロジックの統合
#### [MODIFY] [Flashcard.tsx](file:///c:/FlashcardMaster/src/Components/card/Flashcard.tsx)
- `normalizeCard` をインポートし、コンポーネント内でデータを正規化します。これにより、ブロックが存在する場合に `questionText` や `answerText` が自動的に補完されます。
- また、可能であればブロックを順番に描画するロジックを追加し、より正確なプレビューを実現します。

### 2. コンテンツ存在確認の修正
#### [MODIFY] [Flashcard.tsx](file:///c:/FlashcardMaster/src/Components/card/Flashcard.tsx)
- `hasQuestionContent` および `hasAnswerContent` の判定ロジックに `questionBlocks` / `answerBlocks` のチェックを追加します。

## Verification Plan

### 手動確認
1. 一問一答モードで適当なテキストを問題セクションに入力する。
2. 「プレビュー」ボタンを押し、入力した内容が正しく表示されることを確認する。
3. コードブロックや画像を追加し、それらもプレビューに反映されることを確認する。
4. 表裏切り替えが正常に機能することを確認する。

### 自動ビルド確認
- `npm run build` でコンパイルエラーがないことを確認。
