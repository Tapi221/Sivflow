# KaTeXプレビュー表示不具合の修正

数式ブロックにおいて、一度不適切なLaTeX式を入力するとプレビューが表示されなくなる問題を修正します。

## 原因分析
`MathRenderer.tsx` において、`katex.render` がエラーを投げると `error` ステートが更新されます。その後のレンダリングで `error` が真の場合にコンポーネントが `null` を返して全消去されていたため、`containerRef` が DOM から外れ、`useEffect` 内での再レンダリング（エラー解消）ができなくなっていました。

## 解決策
エラー発生時でも `containerRef` を持つ `div` 要素をレンダリングし続けるように変更します。これにより、ユーザーが正しい数式を入力し直した際に `useEffect` が正常に動作し、プレビューが復帰します。

## 実施内容

### [MathRenderer.tsx](file:///c:/FlashcardMaster/src/Components/card/blocks/MathRenderer.tsx)
- `error` が原因で `null` を返すガード条件（56-58行目付近）を削除します。
- `containerRef` を常に DOM に残し、正常な入力への復帰をサポートします。

## 検証計画
1. 数式ブロックに正常な数式（例: `x^2`）を入力し、プレビューが表示されることを確認。
2. 不正な数式（例: `\frac{`）を入力し、プレビューが消える（または不備のある状態になる）が、DOM要素自体は残っていることを確認。
3. 再度正常な数式を入力し、プレビューが即座に復帰することを確認。
