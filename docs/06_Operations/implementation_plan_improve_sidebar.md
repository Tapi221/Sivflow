# サイドバー挙動改善計画

## 課題
ユーザーより「サイドバーの動きが気に入らない」との指摘あり。
現状 (`Layout.tsx`) の実装を分析した結果、以下の改善点が見込まれる。

1.  **アニメーションの質感**: 標準の `ease-in-out` は少し機械的に感じる場合がある。
2.  **パフォーマンス**: `transition-all` を使用しているため、不要なプロパティまで監視されている可能性がある。
3.  **コンテンツの追従**: メインコンテンツの `margin-left` トランジションとサイドバーの `transform` が同期して動く際の滑らかさ。

## 改善方針
より「アプリらしい」自然で高品質なモーション（Spring系やカスタムBezier）を適用し、操作感を向上させる。

## 具体的な変更内容

### [src/Layout.tsx](file:///c:/FlashcardMaster/src/Layout.tsx)

1.  **トランジションの最適化**
    - `transition-all` を廃止し、`transition-[transform,margin,opacity]` など必要なプロパティに限定する。
    - タイミング関数を `cubic-bezier(0.2, 0.0, 0.2, 1)` (Material Design Standard) または `cubic-bezier(0.4, 0, 0.2, 1)` に変更し、より自然な加減速を実現する。
    - デュレーションを `300ms` から微調整（例: `duration-500` で `ease-out` 強めなど、キビキビしつつ余韻のある動きへ）。

2.  **トグルボタンの挙動**
    - サイドバーの開閉に合わせてボタンのアイコンが変化する、あるいは微細なアニメーションを追加する（例: メニューアイコンが開閉時に回転するなど）ことで、対話性を高める。

3.  **（オプション）オーバーレイ動作の検討**
    - コンテンツを押し出す（Push）挙動が気に入らない可能性も考慮し、実装自体はPushのままだが、検証時に「オーバーレイの方が良いか」を確認できるように準備する。今回はまず動きの質（Easing）を改善する。

## 検証計画
1.  **動作確認**
    - デスクトップ表示 (`md` 以上) でサイドバーの開閉を行い、カクつきがないか確認する。
    - 開閉時のコンテンツ幅のリサイズがスムーズに行われるか確認する。
2.  **ブラウザテスト**
    - Chrome/Edge の DevTools でアニメーションFPSを確認する。

## 補足
もし「動き」が「アニメーション」ではなく「いつ消えるか（自動開閉）」などを指している場合は、別途ヒアリングが必要だが、まずは UI の Transition を最高品質にする。
