# カード作成画面の余白調整計画

## 概要
カード作成画面、特に一問一答モードにおいて、左右の余白が大きく、広いディスプレイ環境で情報密度が低く感じられる問題を改善します。

## Proposed Changes

### 一問一答モード (OneQAMode)
#### [MODIFY] [OneQAMode.jsx](file:///c:/FlashcardMaster/src/Pages/OneQAMode.jsx)
- コンテナの最大幅 `max-w-5xl` (1024px) を `max-w-[1400px]` または `max-w-7xl` (1280px) に拡張します。
- `CardEdit.jsx` との整合性を取るため、`max-w-[1400px]` を採用します。

### カードエディタ (CardEditor)
#### [MODIFY] [CardEditor.tsx](file:///c:/FlashcardMaster/src/Components/card/CardEditor.tsx)
- 問題と解答の間の `gap-12` を `gap-8` 程度に微調整し、横幅を広げた際の間延びを防ぎます。
- 必要に応じて、全体の `px-4` を `px-6` や `px-8` に増やし、端の余白のバランスを取ります。

## Verification Plan
- PCの広い画面で一問一答モードを開き、カードが横に広く表示され、左右のグレーの余白が減っていることを確認。
- モバイル表示でレイアウトが崩れていないことを確認（`md:` や `lg:` プレフィックスによる調整）。
- `CardEdit` 画面（通常の1枚作成画面）ともデザインに違和感がないか確認。
