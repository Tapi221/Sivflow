# カード編集画面モバイル最適化（ブロックツールバー）

カード編集画面において、ブロック追加用のツールバーが画面外にはみ出したり、レイアウトが崩れたりする問題を解決します。

## 問題の現状
- `BlockEditor.tsx` のツールバーが `overflow-x-auto` を使用しているが、モバイル端末の狭い幅では十分に機能せず、親要素（CardEditorのグリッド）を押し広げてしまっている可能性がある。
- ボタンが横一列に並んでいるため、端末幅によっては右側が隠れてしまう、または視認性が著しく低下している。

## 提案する変更

### [Component: Card Editor]

#### [MODIFY] [BlockEditor.tsx](file:///c:/FlashcardMaster/src/Components/card/BlockEditor.tsx)
- ツールバーのレイアウトを `flex-wrap` に変更し、狭い画面では自動的に複数行に折り返されるようにします。
- モバイル表示時（`md` 未満）のボタン内テキストを非表示にするか、ショートカット的なレイアウトを検討します。
- `no-scrollbar` によるスクロール対応よりも、一目で全オプションが見える「グリッド/マルチライン」形式を優先します。

#### [MODIFY] [CardEditor.tsx](file:///c:/FlashcardMaster/src/Components/card/CardEditor.tsx)
- グリッドの `gap` や `padding` を調整し、モバイル端末での余白を最適化します。
- 親コンテナに `max-width: 100%` と `overflow-hidden` を確実に適用し、横揺れを防止します。

## 検証プラン

- 現状、UIコンポーネントのビジュアルテストツールは導入されていない想定ですが、ビルドが通ることを確認します。
  - `npm run build`

### 手動検証
1. **ブラウザ開発者ツールでのデバイスシミュレート**
   - iPhone SE (320px) ～ iPhone 14 Pro Max (430px) の幅で、ツールバーが折り返され、画面外にはみ出さないことを確認。
   - ブロック追加ボタンがすべてタップ可能な位置にあることを確認。
2. **実機検証（ユーザーへ依頼）**
   - 実際のモバイル端末（iOS/Android）での表示を確認していただきます。
