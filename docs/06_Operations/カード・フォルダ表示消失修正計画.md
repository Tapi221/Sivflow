# 作成したカード・フォルダが表示されない問題の修正計画

## 概要
ユーザーが作成したカードやフォルダが表示されない（または同期されない）問題の根本原因を特定しました。
主な原因は、UIが使用する LocalDB インスタンスと、同期サービス (Sync V2) が監視しているインスタンスが別々であるため、保存イベントが正しく通知されていないことにあります。
また、hooksにおいて LocalDB の低レベル API を直接呼び出しているため、同期キューへの投入がスキップされている箇所があります。

## 修正内容

### 1. LocalDB インスタンスの一本化

- `SyncServiceFactory.ts` において、`new LocalDB(userId)` で新しいインスタンスを生成せず、`localDB.ts` の `db` シングルトンを使用するように修正します。

### 2. hooks における DB 操作の一貫性確保

- `useFolders.ts`, `useUserSettings.ts` において、`localDB.folders.put` や `update` を直接呼ぶのではなく、`LocalDB` クラスの `addItem`, `updateItem`, `upsert` を使用するように変更します。これにより、変更が確実に `SyncQueue` に追加され、同期がトリガーされるようになります。

### 3. 不要な Firestore 直接操作の削除と不整合修正

- `useFolders.ts` および `Folders.jsx` において、Firestore を直接 (`updateDoc`, `setDoc`) 更新している箇所を削除または `LocalDB` 経由に統一します。
- `Folders.jsx` の `handleSaveFolder` でフォルダ更新時に LocalDB が更新されていない不具合を修正します。

### 4. LocalDB クラスの微調整

- `localDB.ts` 内のバージョン定義の順序を整理し、コードの健全性を高めます。

## 影響範囲

- カード・フォルダの作成、更新、削除、並び替えの全フロー
- ユーザー設定の更新フロー
- 同期 (Sync V2) の実行タイミング

## 検証計画

### マニュアル検証

1. フォルダを新規作成し、即座に一覧に表示されることを確認。
2. 作成したフォルダ内にカードを作成し、即座に表示されることを確認。
3. フォルダ名を変更し、即座に反映されることを確認。
4. 設定（アクセントカラーなど）を変更し、保存されることを確認。
5. ブラウザのコンソールで `[SyncServiceV2] Sync started` などのログが出力され、バックグラウンド同期が走っていることを確認。
6. リロード後、データが保持されていることを確認。

### 自動テスト

- 既存の `src/services/__tests__/SyncServiceV2.test.ts` (あれば) または関連テストを実行。
- `npm run test` による全体的な壊れがないかの確認。
