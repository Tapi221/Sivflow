# カード保存機能の不具合修正計画

カードを新規作成した際に、ブロックエディタの内容が消失する、または一覧に表示されない問題を修正します。

## 根本原因の特定

1.  **データ消失**: `src/hooks/useCards.ts` の `createCard` 関数において、新規カードオブジェクトを作成する際に `questionBlocks` および `answerBlocks` フィールドがコピーされていません。その結果、ブロックエディタで入力した内容が localDB に保存されず、読み込み時に内容が消えて見えます。
2.  **表示されない不具合**: 内容が空（`questionText` が空）の状態で保存されると、一覧で「無題」や空のカードとして表示され、ユーザーが保存に失敗したと誤解する可能性があります。また、`folderId` の不整合がある場合、フィルタリングにより表示されません。

## 提案する変更

### [Component] Hooks

#### [MODIFY] [useCards.ts](file:///c:/FlashcardMaster/src/hooks/useCards.ts)
- `createCard` 関数を修正し、`cardData` から `questionBlocks` と `answerBlocks` も確実に引き継ぐようにします。
- オブジェクトの構築をスプレッド演算子を用いて簡潔にしつつ、必要な初期値を保証します。

### [Component] Services

#### [MODIFY] [localDB.ts](file:///c:/FlashcardMaster/src/services/localDB.ts)
- `addItem` および `updateItem` に詳細なデバッグログを追加し、保存されるデータの構造（特にブロックの有無）を確認できるようにします。

## 検証計画

### 自動テスト
- 検証用のスクリプトを作成し、`createCard` を呼び出した後の IndexedDB の状態を確認します。

### 手動確認
1.  新規カード作成画面で、テキストブロック、コードブロック、画像ブロックを追加して保存する。
2.  フォルダビューに戻り、カードが正しく追加されていることを確認する。
3.  追加されたカードを再度開き、ブロックの内容がそのまま残っていることを確認する。
4.  コンソールログを確認し、`localDb.addItem` に `questionBlocks` が含まれていることを確認する。

## 懸念事項
- ユーザーが言及した `window.storage.set()` がコード内に見あたらない点については、必要に応じてグローバル検索の結果を報告し、確認を求めます。
