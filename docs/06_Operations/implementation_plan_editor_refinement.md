# カード編集画面のUI洗練と「無題」カード問題の修正計画

カード編集画面の余白を調整して画面スペースを有効活用し、タイトル未入力時に自動的に「無題のカード」という名前がついてしまう挙動を修正します。

## ユーサレビューが必要な項目
- タイトルが完全に空の場合、カード一覧等でどのように表示されるべきか（現状はタイトルがない場合は自動補完されていますが、これを完全に停止すると一覧で何も表示されない可能性があります。ただし、ユーザーの要望は「勝手にタイトルを付けないでほしい」ということなので、保存時の自動補完を停止します）。

## 提案される変更点

### UI改善：余白の削減

#### [MODIFY] [CardEdit.jsx](file:///c:/FlashcardMaster/src/Pages/CardEdit.jsx)
- 外側のコンテナのパディング `md:px-14` を `md:px-4` または `md:px-6` に縮小し、左右の余白を削減します。

#### [MODIFY] [CardEditor.tsx](file:///c:/FlashcardMaster/src/Components/card/CardEditor.tsx)
- `BlockEditor` を囲むコンテナのパディング `md:px-6` を `md:px-2` に調整し、より広い編集エリアを確保します。

---

### 不具合修正：タイトルの自動補完停止

#### [MODIFY] [localDB.ts](file:///c:/FlashcardMaster/src/services/localDB.ts)
- `denormalizeCardForStorage` 内で、タイトルが空の際に `extractedQ` から補完したり、`無題のカード` と判定したりするロジックを削除または調整します。

#### [MODIFY] [utils/index.ts](file:///c:/FlashcardMaster/src/utils/index.ts)
- `normalizeCard` 内で、タイトルがない場合に `無題のカード` を設定するフォールバック処理を削除します。

## 検証プラン

### 自動テスト
- なし

### 手動確認
- カード編集画面を開き、左右の余白が以前より狭くなっていることを確認。
- タイトルを空のままカードを保存し、再び編集画面を開いた際にタイトルが「無題のカード」にならず、空のままであることを確認。
- タイトルが空のカードが一覧画面でどのように見えるか確認。
