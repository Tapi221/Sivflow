# 実装計画：Liquid Glass V6 構造的クリーンアップ

「操作時だけLiquid」戦略を完遂し、CSSの負債（@applyの乱用、不正なブレンドモード、古いエイリアス）を一掃します。

## ユーザーレビューが必要な項目
> [!IMPORTANT]
> - リスト行（Folders）の通常時が「完全にフラット（背景のみ）」になります。
> - ホバー／選択時のみ、ブラーと反射、影が発生する「操作フィードバック」としてのガラスUIに統一しました。

## 変更内容

### 1. index.css の整合性改善
- **specular（反射）の改善**: `mix-blend-mode: overlay` を `normal` に変更。暗い背景での「濁り」を防止。
- **@apply の廃止**: カスタムクラス（`.liquid-glass-header`）内での `@apply liquid-glass` を廃止し、プロパティを直接記述。Tailwindのコンパイル挙動による不安定さを排除。
- **z-index の固定**: ガラスの擬似要素の上にコンテンツが確実に載るよう、`.liquid-glass > *` に `z-index: 3` を強制。

---

### 2. コンポーネントのクラス刷新（Folders.jsx）
- **古いエイリアスの置換**: `glass`, `btn-glass` といった古い名称を `liquid-glass`, `liquid-glass-chip` に統一。
- **非表示フォルダセクションのデザイン適用**: 以前は bg-white だったセクションに `liquid-glass-header` を適用し、アプリ全体の統一感を持たせました。

---

## 検証計画

### 自動テスト
- `npm run build` を実行し、CSS構文エラーがないことを確認。

### 手動検証（ブラウザ）
1. **フォルダ一覧の視認性**:
   - 通常状態で文字がクッキリ見え、背景が「モヤ」っとしていないか確認。
2. **インタラクションの挙動**:
   - マウスを乗せた時だけ、ブラーと反射が「ヌルッ」と現れるか確認。
   - 選択（Selected）状態で行がガラス化したまま維持されるか確認。
3. **ヘッダーパネルの質感**:
   - 上部のツールバーや「非表示フォルダ」ボタンが、適切な深み（ブラー16px）を持っているか確認。
