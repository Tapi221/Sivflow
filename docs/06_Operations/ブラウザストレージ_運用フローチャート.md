# ブラウザストレージ - 5分で判断できる運用フローチャート

## 想定読者

- **レベル1**: 非エンジニア運用（ブラウザ操作のみ）
- **レベル2**: フロントエンドエンジニア
- **レベル3**: Firebase 管理者

---

## 🚨 症状別フローチャート

### A. アプリが起動しない

```
起動しない
  ↓
コンソールに [Auth] Fatal setup error: が出ている？
  ├─ YES → IndexedDB が無効化されている
  │         ├─ Chrome設定 > プライバシーとセキュリティ > Cookie を確認
  │         ├─ シークレットモードで試す
  │         └─ ダメなら → 別ブラウザで試す
  │                       └─ それでもダメ → 【レベル3】Firebase 確認
  │
  └─ NO → ローディング画面で止まっている？
            ├─ YES → 5分待つ
            │         ├─ 進んだ → 再構築中（正常）
            │         └─ 進まない → F5 でリロード
            │                       └─ ダメなら → 【諦める】今日は触らない
            │
            └─ NO → 白い画面？
                      └─ キャッシュクリア → F5
                          └─ ダメなら → 【レベル2】開発者に連絡
```

**判断ライン**:
- ✅ 5分待ってダメなら諦める
- ✅ 別ブラウザでダメなら Firebase の問題
- ✅ 白い画面はキャッシュクリアで解決しなければ開発者案件

**ユーザーへの説明**:
> 「現在、データを初期化しています。数分お待ちください。」

---

### B. 書き込みができない

```
書き込みができない
  ↓
コンソールに [Storage] Write blocked (READ_ONLY) が出ている？
  ├─ YES → ストレージ容量が枯渇している
  │         ├─ Chrome設定 > ストレージ管理 を開く
  │         ├─ 不要なサイトのデータを削除
  │         ├─ F5 でリロード
  │         └─ ダメなら → 【諦める】容量を空けてから再度試す
  │
  └─ NO → 同期エラー？
            ├─ コンソールに [Sync] が出ている？
            │   ├─ YES → ネットワーク確認
            │   │         └─ オンラインなら → 【レベル2】同期設定確認
            │   │
            │   └─ NO → 【レベル2】開発者に連絡
            │
            └─ 何も出ていない → F5 でリロード
                                └─ ダメなら → 【レベル2】開発者に連絡
```

**判断ライン**:
- ✅ READ_ONLY は容量問題。容量を空けるまで諦める
- ✅ 同期エラーはネットワーク確認で解決しなければ開発者案件
- ✅ ログが何も出ていなければ開発者案件

**ユーザーへの説明**:
> 「ストレージ容量が不足しています。ブラウザの設定から不要なデータを削除してください。」

---

### C. データが消えた

```
データが消えた
  ↓
コンソールに [AppInit] Rebuilding IndexedDB... が出ている？
  ├─ YES → 再構築が実行された（正常動作）
  │         ├─ 5分待つ
  │         ├─ [AppInit] Initialization complete が出た？
  │         │   ├─ YES → クラウド同期を待つ（さらに5分）
  │         │   │         ├─ データが戻った → 解決
  │         │   │         └─ 戻らない → 【レベル3】Firestore 確認
  │         │   │
  │         │   └─ NO → [AppInit] Health check failed: rebuild_loop が出た？
  │         │             ├─ YES → 【諦める】今日は触らない
  │         │             │         → 【レベル3】metadata リセットが必要
  │         │             │
  │         │             └─ NO → さらに5分待つ
  │         │                       └─ ダメなら → 【諦める】今日は触らない
  │         │
  │         └─ 何も出ない → F5 でリロード
  │                         └─ ダメなら → 【レベル2】開発者に連絡
  │
  └─ NO → 手動で削除した？
            ├─ YES → ゴミ箱から復元
            │         └─ ゴミ箱にもない → 【諦める】復元不可
            │
            └─ NO → 【レベル2】開発者に連絡
```

**判断ライン**:
- ✅ 再構築は正常動作。10分待ってダメなら諦める
- ✅ rebuild_loop は開発者案件。触らない
- ✅ 手動削除は復元不可

**ユーザーへの説明**:
> 「ローカルデータを再構築しました。クラウドから同期中です。数分お待ちください。」

---

## 📋 レベル別対応表

### レベル1: 非エンジニア運用（ブラウザ操作のみ）

**できること**:
- コンソールを開く（F12）
- ログを確認する
- リロードする（F5）
- キャッシュをクリアする
- 別ブラウザで試す
- 5分待つ

**できないこと**:
- IndexedDB を直接操作
- Firestore を確認
- metadata をリセット

**諦めるライン**:
- 5分待ってダメ
- 別ブラウザでダメ
- rebuild_loop が出た

---

### レベル2: フロントエンドエンジニア

**できること**:
- レベル1 の全て
- IndexedDB を確認（DevTools）
- metadata の状態を確認
- ログの詳細を読む
- 同期設定を確認

**できないこと**:
- Firestore を直接操作
- metadata を手動でリセット

**諦めるライン**:
- metadata が壊れている
- Firestore が壊れている
- rebuild_loop が出た

---

### レベル3: Firebase 管理者

**できること**:
- レベル2 の全て
- Firestore を確認
- Firestore のデータを削除
- metadata を手動でリセット
- Snapshot を復元

**できないこと**:
- コードを修正

**諦めるライン**:
- Firestore が完全に壊れている
- Snapshot が全て壊れている
- コードにバグがある

---

## 🔥 緊急時の判断基準

### 今日は諦めていい条件

以下のいずれかが出たら、**それ以上は触るな**:

1. `rebuild_loop` が出た
2. 5分待っても進まない
3. 別ブラウザでも起動しない
4. Firestore が落ちている
5. metadata の `rebuildCount > 3`

**理由**: これ以上触ると悪化する可能性がある

**次の行動**:
- ログを保存
- スクリーンショットを撮る
- 開発者に連絡
- 明日まで待つ

---

### すぐに対処すべき条件

以下のいずれかが出たら、**即座に対処**:

1. `READ_ONLY` モード
2. ストレージ容量不足
3. ネットワークエラー

**理由**: ユーザー操作で解決できる

**次の行動**:
- 容量を空ける
- ネットワークを確認
- リロード

---

## 📞 エスカレーションフロー

```
レベル1（非エンジニア）
  ↓ 5分待ってダメ
レベル2（フロントエンドエンジニア）
  ↓ IndexedDB/metadata 確認してダメ
レベル3（Firebase 管理者）
  ↓ Firestore 確認してダメ
開発者（コード修正）
```

**各レベルの制限時間**:
- レベル1: 5分
- レベル2: 15分
- レベル3: 30分

**制限時間を超えたら**: 次のレベルにエスカレーション

---

## まとめ

このフローチャートは「技術を知らない人が夜中に読んで判断できる」ことを目的としています。

**重要な原則**:
1. **5分で判断する**
2. **諦めるラインを明確にする**
3. **ユーザーへの説明を用意する**
4. **エスカレーションを躊躇しない**
