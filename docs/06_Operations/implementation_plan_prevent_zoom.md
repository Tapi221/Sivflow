# テキストブロックの入力最適化とUI調整の実装計画

## 概要
テキストブロックへの入力時にモバイルデバイス等で文字が拡大される（ズームインする）問題を解消し、あわせてブロックの右側に表示されるツールボタンのサイズを縮小してUIを改善します。

## 現状の課題
1. **入力時の文字拡大**: `TextBlock.tsx`内のフォントサイズが `text-sm` (14px) に設定されているため、iOS等のモバイルブラウザでフォーカス時に自動ズームが発生してしまう。
2. **ツールボタンのサイズ**: 右側に表示される操作ボタン（削除、複製、ドラッグ）が大きく、視覚的な主張が強い。

## 変更内容

### 1. テキスト入力のズーム防止 (`src/Components/card/blocks/TextBlock.tsx`)
- **変更前**: `text-sm` (14px)
- **変更後**: モバイル表示では `text-base` (16px) を適用し、PC表示では `sm:text-sm` (14px以上で14px) とするレスポンシブ対応を行う。これにより、モバイルでの自動ズームを防ぎつつ、PCでは既存のデザインを維持または適切なサイズにする。
  - クラス変更: `font-sans text-sm` -> `font-sans text-base sm:text-sm`

### 2. ツールボタンのサイズ縮小 (`src/Components/card/blocks/BlockWrapper.tsx`)
- 操作メニュー内のボタンのパディングを削減して全体のサイズを小さくする。
- **変更前**: `p-1.5 md:p-1.5`
- **変更後**: `p-1 md:p-1` (またはより適切な小さいサイズ)
- アイコンサイズは `w-3.5 h-3.5` のまま維持し、クリック領域（パディング）のみを調整してコンパクトにする。

## 検証計画
### 自動テスト
- 現状、UIの視覚的な変更に対する自動テストはないため、コンポーネントがエラーなくレンダリングされることを確認する。

### 手動検証
- **モバイルデバイス（またはエミュレータ）**: テキストエリアをタップしても画面がズームインしないことを確認。
- **UI確認**: ブロック右側のツールボタンが以前より小さく、かつ操作可能なサイズであることを確認。
- **レスポンシブ確認**: PCサイズでもレイアウトが崩れていないことを確認。
