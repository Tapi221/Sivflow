# プレビュー表示不具合修正 実装レポート

## 概要
カード作成画面のプレビューにおいて内容が表示されない不具合を、`Flashcard` コンポーネントのブロック対応化によって修正しました。

## 修正内容

### 1. Flashcard コンポーネントのブロック対応
- `Flashcard.tsx` 内で `normalizeCard` を使用し、レガシーフィールドが空であってもブロックデータから内容を抽出・補完するようにしました。
- プレビューおよび学習画面において、各ブロック（テキスト、コード、画像、音声、メモ）を登録された順番通りにレンダリングする `renderBlocks` ロジックを実装しました。

### 2. コンテンツ存在判定の強化
- 問題文や解答が空であるという誤判定を防ぐため、`questionBlocks` や `answerBlocks` にデータがある場合も「コンテンツあり」と見なすように修正しました。

## 確認事項
- [x] カード作成画面でテキストやブロックを入力した際、プレビューに即座に反映される。
- [x] テキスト以外のコードブロックや画像もプレビューに表示される。
- [x] 表裏の切り替え後も正しく内容が表示される。
