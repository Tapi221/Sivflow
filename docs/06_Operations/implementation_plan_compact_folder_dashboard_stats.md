# フォルダダッシュボード統計表示のコンパクト化の実装計画

フォルダダッシュボードの「耐性スコア分布」などの統計情報の表示が大きすぎるため、よりコンパクトで洗練された表示に改善します。

## 変更内容の概要

1.  **`StatsCharts.tsx` の改善**:
    *   `StabilityDistributionChart` に `compact` プロパティを追加。
    *   `compact` モード時、以下の調整を行う：
        *   カード全体のパディングを削減（`p-4` 程度）。
        *   ヘッダー（タイトル部分）の余白（`mb-8` など）を大幅に削減。
        *   タイトルのフォントサイズ、アイコンサイズを縮小。
        *   チャート本体の高さ（`height: 260`）を削減（`height: 180` 程度）。
        *   フェーズ表示バーの余白・サイズを縮小。
2.  **`FolderDashboard.tsx` の調整**:
    *   `StabilityDistributionChart` に `compact` プロパティを渡す。
    *   現在適用されている `scale-90`, `-translate-x-4`, `w-[110%]` などの不自然なスタイル調整を削除し、コンポジットなレイアウトに変更。

## 影響範囲

*   `StatsCharts.tsx`: `StabilityDistributionChart` を使用している他の箇所（あれば）への影響を確認。現在は主にフォルダダッシュボードで使用。
*   `FolderDashboard.tsx`: フォルダダッシュボードの表示。

## 確認事項

*   [ ] モバイル表示でレイアウトが崩れないか。
*   [ ] チャートのラベル（フェーズ名など）が小さくなりすぎて読めなくならないか。

## 検証計画

### 自動テスト
*   本変更はUIの見た目に関する修正であるため、既存の機能テストに影響がないことを確認（`npm run typecheck`）。

### 手動確認
1.  **ブラウザ表示確認**:
    *   フォルダダッシュボードを開き、統計表示が以前よりコンパクトに、かつ綺麗に表示されていることを確認する。
    *   チャートの棒グラフやホバー時のツールチップが正しく動作することを確認する。
2.  **レスポンシブ確認**:
    *   画面幅を狭めた際、統計表示が画面端からはみ出したり、重なったりしていないか確認する。
