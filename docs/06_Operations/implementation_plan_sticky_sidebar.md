# 実装計画: サイドバーの固定表示（Sticky）対応

## ゴール
1. **サイドバーの固定化**: ページがスクロールされても、フォルダツリー（サイドバー）が画面上に留まり続けるようにする。
2. **ページスクロールの許可**: カード編集エリアの内容が長い場合、ページ全体がスクロールするように変更する（内部スクロールの廃止）。
3. **リサイズ機能の維持**: サイドバーの境界線は引き続きページ最下部まで伸び、リサイズ可能であることを維持する。

## Proposed Changes

### 1. [Layout Structure] TreeViewLayout.tsx
- **Main Container**:
  - 固定高さ `h-[calc(100vh-140px)]` を削除し、コンテンツに合わせて伸長するように変更。
  - `overflow-hidden` を削除（Stickyを有効にするため）。
  - 代わりに `min-h-[calc(100vh-140px)]` を設定。
- **Left Pane (Wrapper)**:
  - `relative` で配置（リサイズハンドルの親）。
  - 内部に新しい `div.sticky` を作成。
    - `sticky top-4`: 画面上部から少し空けて固定。
    - `h-[calc(100vh-2rem)]`: 画面高さいっぱいに表示。
    - `overflow-y-auto`: サイドバーの中身が多い場合はサイドバー内でスクロール。
- **Right Pane**:
  - `overflow-hidden` や `flex-col` を削除し、自然に高さを確保させる。

### 2. [Child Component] CardEditorPane.tsx
- コンテナの `h-full overflow-y-auto` を削除。
- 内部の `CardEditor` が自然に高さを取るように修正。
- プレースホルダーの `min-h-[400px]` は維持。

## Verification Plan

### Manual Verification
1. **スクロール動作**:
   - 右側のカード編集エリアに長いコンテンツがある場合、ブラウザのスクロールバーでページ全体がスクロールすること。
   - その際、左側のフォルダツリーが画面に固定され、一緒に上に流れてしまわないこと。
2. **リサイズ動作**:
   - ページ全体が長くても、境界線（リサイズハンドル）が上から下まで繋がっており、どこでもドラッグ可能であること。
3. **表示崩れ確認**:
   - コンテナの角丸（`rounded-xl`）からコンテンツがはみ出していないか確認（必要なら `rounded-l-xl` 等で対応）。
