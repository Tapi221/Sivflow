# 不具合修正：ドラッグハンドルのホバー効果削除

ブロックの並び替えボタン（ドラッグハンドル）を操作する際の視覚的なフィードバック（ホバー・アクティブ時の背景色や境界線の変化）を削除します。

## 修正内容

### [BlockWrapper.tsx](file:///c:/FlashcardMaster/src/Components/card/blocks/BlockWrapper.tsx)
- ドラッグハンドル（`showDragHandle` 部分）の `className` から以下のクラスを削除します。
  - `hover:text-primary-600`
  - `hover:border-primary-100`
  - `active:bg-primary-50`
  - `active:border-primary-200`
  - `active:cursor-grabbing` (もしユーザーが「掴んだ瞬間」のカーソル変化も不要と言っている場合は検討しますが、操作感のためにカーソルは維持する方が一般的です。今回はまず背景色/ボーダーのホバー効果に絞ります。)

## 検証計画

### 手動確認
1. **ホバー確認**: マウスをドラッグハンドルに乗せた際、アイコンの色や境界線の色が変わらないことを確認します。
2. **クリック・ドラッグ確認**: ボタンを押した際（アクティブ状態）、背景が青くならないことを確認します。
3. **機能確認**: 並び替え機能自体（クリックで1行下移動、ドラッグでの移動）が引き続き正常に動作することを確認します。

### 自動検証
- `npm run build` を実行し、型エラーやビルドエラーがないことを確認。
