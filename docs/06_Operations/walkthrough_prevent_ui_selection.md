# テキスト選択防止のグローバル適用 完了レポート

## 概要
アプリケーション全体で意図しないテキスト選択（ハイライト）を防ぐモバイルアプリのような操作感を実現するため、グローバルに `select-none` を適用しました。ユーザーがテキストをコピーする必要があるカード内容や、編集中の入力フィールドについては個別に選択を許可する設定を行いました。

## 変更内容

### 1. グローバル設定
- [index.css](file:///c:/FlashcardMaster/src/index.css) の `body` 要素に `select-none` を追加しました。これにより、ボタンのダブルタップやドラッグ操作時に周囲のテキストが選択されるのを防ぎます。

### 2. カード閲覧画面 (Flashcard)
- [Flashcard.tsx](file:///c:/FlashcardMaster/src/Components/card/Flashcard.tsx) のメインコンテンツエリア（質問、解答、メモ）に `select-text` を適用しました。
- 学習中に数式を詳しく見たり、テキストをコピーしたりすることが可能です。

### 3. カード編集画面 (BlockEditor)
- 各入力ブロックの共通ラッパーである [BlockWrapper.tsx](file:///c:/FlashcardMaster/src/Components/card/blocks/BlockWrapper.tsx) のコンテンツ表示エリアに `select-text` を適用しました。
- これにより、テキスト入力中のカーソル移動や範囲選択が正常に行えます。

## 検証結果

| 項目 | テキスト選択の可否 | 期待される動作 |
| :--- | :--- | :--- |
| **ログイン画面 / 設定画面** | 不可 | 意図しないハイライトを防ぎ、UIの質感を向上 |
| **ダッシュボード / フォルダ一覧** | 不可 | リストやタイルの操作時にストレスを与えない |
| **カード閲覧（コンテンツ）** | **可能** | ユーザーが情報をコピー・精査できる |
| **カード編集（入力エリア）** | **可能** | 通常どおりの編集操作を維持 |
| **編集画面のヘッダー・ボタン** | 不可 | 誤操作による選択を防止 |

以上の変更により、全体としてより洗練された操作感を実現しました。
