# 一問一答画面の機能拡張（削除・並び替え）実装計画

一問一答画面（`OneQAMode`）において、作成中のカードを個別に削除したり、ドラッグ＆ドロップで並び替えたりできるように機能を拡張します。

## ユーザーレビューが必要な項目

- **UIデザイン**: ドラッグハンドルと削除ボタンの配置について。現在はカードの右上に配置する予定です。
- **削除時の挙動**: 削除時に確認ダイアログを出すか、即座にトースト通知と共に削除するか。今回は利便性を重視し、即座に削除（トーストで通知）する方針です。

## 変更内容

### 1. ページコンポーネントの拡張

#### [MODIFY] [OneQAMode.jsx](file:///c:/FlashcardMaster/src/Pages/OneQAMode.jsx)

- `@hello-pangea/dnd` を導入し、エディタのリストを `DragDropContext`, `Droppable`, `Draggable` でラップします。
- 各カードの `CardEditor` をラップするコンテナに、ドラッグハンドルと削除ボタンを追加します。
- `handleDelete(editorId)` 関数を実装し、指定されたエディタを `editors` ステートから削除します。
- `onDragEnd` 関数を実装し、ドラッグ後の並び替えを `editors` ステートに反映させます。
- インデックス番号（Q1, Q2...）が並び替えや削除後も正しく表示されるように調整します。
- 保存済みカードの表示レイアウトを調整し、ドラッグハンドル等と干渉しないようにします。

### 2. スタイルの調整

#### [MODIFY] [index.css](file:///c:/FlashcardMaster/src/index.css) (または `OneQAMode.jsx` 内のインライン/Tailwind)

- ドラッグ中のカードの視覚効果（影や背景色）を定義します。

## 検証プラン

### 自動テスト
- 既存のテストが壊れていないことを確認します。
- `npm test`

### 手動検証
1. **並び替え検証**:
    - 複数の「一問一答」エディタが表示されている状態で、ドラッグハンドルを掴んで順序を入れ替える。
    - 保存ボタンを押した際、入れ替えた順序でカードが作成されることを確認する。
2. **削除検証**:
    - 特定のエディタの削除ボタン（ゴミ箱アイコン）をクリックする。
    - そのエディタがリストから消え、以降の番号が正しく振り直されることを確認する。
3. **継続作成との整合性**:
    - 「続けて作成」をクリックした際、常にリストの最後に追加され、フォーカスが当たることを確認する。
    - 並び替え後も「続けて作成」や「保存」ボタンが（最後のエディタに）正しく表示されることを確認する。
