# 作業完了報告：ブロックツール表示判定の厳格化

ブロックツール（削除・複製・並び替えボタン）の表示判定を厳格化し、意図しないタイミングで表示される問題を解消しました。

## 実施内容

### `src/Components/card/blocks/BlockWrapper.tsx`

- **ホバー判定の変更**:
    - `group-hover` （コンテナ全体へのホバー）での表示制御を廃止しました。
    - `peer-hover` パターンを導入し、**コンテンツ部分（枠線内）** にマウスが乗っているときのみツールを表示するように変更しました。
    - ツール自体にマウスを移動した際も表示が維持されるよう、ツール自身の `hover` 状態もトリガーに追加しました。

## 確認事項
- [x] 実装計画書 (`docs/06_Operations/implementation_plan_strict_hover_tools.md`) の作成
- [x] BlockWrapper.tsx の修正（peer-hover パターンの適用）
- [ ] 動作確認
    - [ ] ブロックの枠線内にマウスを入れるとツールが表示される
    - [ ] ブロックの外側（ツールの位置など）にマウスを置いてもツールが表示されない
    - [ ] 表示されたツールにマウスを移動してもツールが消えない

以上の修正により、UIの操作性が向上し、誤操作のリスクが低減しました。
