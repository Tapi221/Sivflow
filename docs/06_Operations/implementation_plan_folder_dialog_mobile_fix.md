# 新規フォルダ作成画面のモバイル表示最適化計画

## 概要
モバイルデバイス（特にiOS）において、キーボード表示時に「新規フォルダ作成」ダイアログの入力フィールドが隠れてしまう問題を修正します。`window.visualViewport` APIを使用してキーボードの開閉を検知し、ダイアログの位置を動的に調整します。

## ユーザーレビューが必要な事項
- 特になし（UI調整のみ）

## 変更内容

### Components
#### [MODIFY] [FolderDialog.tsx](file:///c:/FlashcardMaster/src/Components/folder/FolderDialog.tsx)
- 「クラウド同期」のトグルオプションを削除（設定画面で管理するため）
- 「キャンセル」ボタンを削除し、右上の✕マークでの閉鎖に統一
- ダイアログの位置調整ロジック（スライド）は維持

**実装詳細:**
1. `visualViewport` の `resize` および `scroll` イベントを監視
2. キーボードが表示された場合（`visualViewport.height < window.innerHeight`）、画面下部の不可視領域（キーボードの高さ）を計算
3. ダイアログ全体、またはフォーム部分を `transform: translateY(-キーボードの高さの一部)` などで上にずらす
   - あるいは、`visualViewport.height` に合わせてコンテナの高さを調整し、Flexboxでセンタリングを維持する

## 検証計画

### 動作確認
1. **iOS Safariでの確認**
   - iPhone（実機またはシミュレータ）で「新規フォルダ作成」を開く
   - フォルダ名入力欄をタップしてキーボードを表示
   - ダイアログが上にスライドし、入力欄と「作成する」ボタンが見える状態になるか確認
   - 画面外をタップまたは「完了」を押してキーボードを閉じる
   - ダイアログが元の位置にスムーズに戻るか確認

2. **Android Chromeでの確認**
   - Android端末で同様の操作を行い、挙動を確認

3. **デスクトップでの確認**
   - デスクトップブラウザで動作に影響がないことを確認（`visualViewport` の挙動が異なるため、条件分岐が正しく機能するか）
