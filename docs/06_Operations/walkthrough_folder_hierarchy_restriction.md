# 完了報告：フォルダー移動の階層制限実装

フォルダーツリーのドラッグ＆ドロップ操作に階層制限を追加し、誤操作を防ぐようにしました。

## 変更内容

### [FolderTree](file:///c:/FlashcardMaster/src/Components/folder/FolderTree.tsx)

`handleDragEnd` 関数内に以下の階層制限ロジックを追加しました：

1. **階層深度計算関数 `getDepth`**
   - フォルダーIDを受け取り、ルートからの階層レベルを計算
   - 循環参照を防ぐため、訪問済みセットを使用

2. **最大階層深度の計算**
   - 全フォルダーの中で最も深い階層レベルを特定

3. **移動制限チェック**
   - **ルートフォルダーの保護**: 親がないフォルダー（ルートフォルダー）を他のフォルダーの子に移動しようとした場合、移動をキャンセル
   - **最大階層制限**: 現在最も深い階層にあるフォルダーを、それ以上深い階層に移動しようとした場合、移動をキャンセル

## 動作

- ルートフォルダーを他のフォルダーにドラッグしても移動できません
- 最も深い階層のフォルダーを、さらに深い階層にドラッグしても移動できません
- 制限に引っかかった場合、コンソールにメッセージが表示されます
- 通常の移動（制限に引っかからない移動）は従来通り動作します

## 検証結果

- [x] ルートフォルダーを他のフォルダーの子に移動できないことを確認
- [x] 最大階層のフォルダーをそれ以上深い階層に移動できないことを確認
- [x] 通常の移動は正常に動作することを確認
