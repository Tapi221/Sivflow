# リスト表示レイアウト最適化の実装完了

## 概要
カード一覧のリスト表示（`viewMode === 'list'`）のレイアウトを最適化し、横幅を有効活用したコンパクトなデザインに変更しました。
以前の「縦積みカード」スタイルから、「横並び行」スタイルへ移行し、画面あたりの情報量を増やしつつスッキリと整理しました。

## 実装内容
`src/Components/card/CardList.tsx` のスタイルロジックを大幅に改修しました。

### 変更点
1.  **コンテナ幅の拡張**:
    *   以前: `max-w-3xl`に制限。
    *   **現在**: `w-full` (幅制限なし、親コンテナいっぱいまで拡張) に変更。これにより、広い画面で多くのテキストを表示できるようになります。

2.  **カードアイテムの構造変更**:
    *   グリッド表示: 従来通りのカード型（縦積み、固定高さ）。
    *   **リスト表示**: **横並び (`flex-row`)** に変更し、高さ制限（`h-[240px]`）を撤廃して `h-auto` に設定しました。

3.  **コンテンツ配置の調整 (リスト表示時)**:
    *   **左端**: チェックボックス、ステータス、バッジ類をまとめて配置。
    *   **中央**: 質問 (`Question`) と 答え (`Answer`) を横並び（SP版は縦並び）のグリッドレイアウト (`grid-cols-[1fr_2fr]`) に配置し、長文でも比較しやすくしました。
        *   答え (`Answer`) が空の場合は "No answer" と表示し、レイアウト崩れを防いでいます。
    *   **右端**: 編集・削除ボタンを配置（ホバー時のみ表示は維持しつつ、絶対配置から静的配置に変更して重なりを防止）。

4.  **余白の削減**:
    *   パディングを `p-5` から `p-3` に削減し、より密度のあるリスト感を実現しました。

## 検証方法
1.  フォルダ画面で、グリッド表示アイコン（田）の隣にある「リスト表示アイコン（三）」をクリックします。
2.  カードが横長の行スタイルで表示されることを確認します。
3.  ウィンドウ幅を広げると、カードの幅も追従して広がることを確認します。
4.  長いテキストが適切に表示され、無駄な余白が削減されていることを確認します。
