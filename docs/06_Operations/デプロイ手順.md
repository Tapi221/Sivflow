# 本番環境へのデプロイ手順

本プロジェクトは React + Vite + Firebase (Hosting/Firestore/Auth/Functions/Storage) の構成となっており、以下の手順で本番環境へ反映します。

## 1. ビルドの実行

Vite を利用して本番用の静的ファイルを生成します。

```bash
npm run build
```

成功するとプロジェクトルートの `dist` フォルダにビルド済みファイルが出力されます。

## 2. Firebase へのデプロイ

Firebase CLI を利用して、Hosting、Firestore のインデックス/ルール、Functions などをデプロイします。

```bash
# 全てのサービスをデプロイする場合
firebase deploy

# Hosting のみに限定する場合
firebase deploy --only hosting
```

> [!NOTE]
> `firebase` コマンドが使用できない場合は、`npx firebase deploy` を試してください。

## 3. デプロイ後の確認

デプロイ完了後、以下の点を確認してください。

- Firebase Hosting の URL にアクセスし、最新の UI が反映されているか。
- 設定画面で「同期」が正常に行われ、FirebaseError が発生していないか。
- ブラウザの DevTools (F12) の Console にエラーが出ていないか。

## 4. エミュレータの停止（開発時）

ローカル開発でエミュレータを使用している場合、本番ビルドには `.env.production` の設定が反映されますが、デプロイ前に `.env` の `VITE_USE_FIREBASE_EMULATOR` が `false` であること（または環境変数で正しく制御されていること）を確認してください。
