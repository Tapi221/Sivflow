# カラム表示でのドラッグアンドドロップ移動機能の実装計画

カラム表示（Finder スタイル）において、フォルダを別のフォルダに移動したり、並び替えたりできるようにするため、`@hello-pangea/dnd` を使用してドラッグアンドドロップ機能を実装します。

## 変更内容の概要

1. `ColumnNavigator.tsx` に `DragDropContext` を配置し、ドラッグ終了時の移動ロジックを実装します。
2. `FolderColumn.tsx` の各フォルダアイテムを `Draggable` でラップします。
3. `FolderColumn.tsx` の各フォルダアイテム自体を `Droppable` でラップし、「フォルダへのドロップ」を可能にします。
4. `FolderColumn.tsx` のカラム全体を `Droppable` でラップし、並び替えや階層へのドロップを可能にします。

## ユーザーレビューが必要な事項
- 特になし。既存のツリー表示（`FolderTree.tsx`）と同様の挙動を目指します。

## 提案される変更

### フォルダコンポーネント

#### [MODIFY] [ColumnNavigator.tsx](file:///c:/FlashcardMaster/src/Components/folder/ColumnNavigator.tsx)
- `DragDropContext` をインポートし、カラムリストをラップします。
- `onDragEnd` ハンドラを実装します。
- `FolderTree.tsx` から階層制限チェック（ルート移動制限、最大深度制限）のロジックを移植または一部流用します。
- `onReorder` プロップを使用して、データの更新を通知します。

#### [MODIFY] [FolderColumn.tsx](file:///c:/FlashcardMaster/src/Components/folder/FolderColumn.tsx)
- `Draggable` と `Droppable` を `@hello-pangea/dnd` からインポートします。
- 各フォルダアイテムを `Draggable` でラップします。
- 各フォルダアイテムの UI を `Droppable` (`droppableId="drop-${fId}"`) でラップし、そのフォルダを親にする移動を可能にします。
- カラムのリスト全体を `Droppable` (`droppableId="column-${parentId}"`) でラップし、並び替えを可能にします。

## 検証計画

### 自動テスト
- 既存のテストスイートがあれば実行します。

### 手動確認
1. **並び替え**: 同じカラム内でフォルダを上下にドラッグして、並びが変わることを確認します。
2. **移動（別階層）**: あるカラムのフォルダを、別のカラムのフォルダの上にドロップして、そのフォルダのサブフォルダになることを確認します。
3. **移動（別階層のリスト）**: あるカラムのフォルダを、別の階層のカラムの余白にドロップして、その階層のルート（直下）に移動することを確認します。
4. **制限事項の確認**:
    - ルートフォルダを他のフォルダの中に移動できないこと（仕様通りであれば）。
    - 循環参照（自分自身やその子孫へ移動）ができないこと。
    - 最大深度制限（もしあれば）が守られていること。

## 考慮事項
- カラム表示では通常、名前順などで自動ソートされることが多いですが、ユーザーの「ドラッグアンドドロップで移動」という要望に応えるため、自由な並び替えまたは親の変更をサポートします。
