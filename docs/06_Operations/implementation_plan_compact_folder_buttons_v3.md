# フォルダ一覧のボタン間隔縮小 (第3回調整)

## 概要
修正を重ねてきましたが、提供された画像では依然としてグリップアイコンと折りたたみ矢印の間に目に見える「間」が残っています。ブラウザのデフォルトスタイルや、ボタン要素の暗黙的な余白が原因の可能性があるため、今回は `gap` を完全に廃止し、より強力なネガティブマージンとパディングの強制ゼロ化によって、視覚的に完全に隣接（あるいは微小な重なり）させます。

## 変更内容

### Components/folder

#### [MODIFY] [FolderTree.tsx](file:///c:/FlashcardMaster/src/Components/folder/FolderTree.tsx)

- **ボタンコンテナの強制密着**:
  - `flex` コンテナの `gap-0` を維持しつつ、さらに以下の調整を行います。
  - グリップアイコン（またはチェックボックス）を包む要素に `p-0` を強制。
  - 折りたたみボタン (`button`) に `p-0`, `m-0`, `border-none` を強制し、さらに `ml-[-4px]` 程度のネガティブマージンを適用してグリップ側に潜り込ませます。
  - アイコン自体の外側に余白を持たないよう、`GripVerticalIcon` と `Chevron` のクラスを再度精査します。

- **視覚的な「一体化」**:
  - 二つの要素を一つの「操作ユニット」として扱うため、ホバー時の背景色を共有するようなスタイルも検討しましたが、まずは物理的な距離をゼロ（またはマイナス）にします。
