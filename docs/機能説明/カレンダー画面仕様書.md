# カレンダー画面 (Calendar) 仕様書

## 概要
月ごとの復習スケジュールを可視化し、学習負荷の分散状況を確認するための画面です。
**責務分離の原則**に基づき、カレンダーは「スケジューリング結果」と「カードの成熟度（耐性）」のみを表示し、学習者の短期的な定着度（Retention）評価は行いません。

## 対象ファイル
- `src/Pages/Calendar.jsx`

---

## 1. 責務と使用指標

本画面では以下の指標を厳密に使い分けます。

| UI要素 | 使用指標 | 目的・理由 |
| :--- | :--- | :--- |
| **表示日** | `nextReviewDate` | スケジュールされた「いつやるか」の事実のみを表示。 |
| **ドット数** | `Card Count` | 単純な学習量（枚数）の可視化。 |
| **ドット色** | **`Resistance Score`** | **成熟度の表現**。「形成中（赤）」か「安定（青）」かを示し、静的な指標として扱う。 |
| **今日やるか** | `Retention` (内部判定) | 自動繰越ロジックなどで使用するが、UIには直接表示しない。 |

---

## 2. データ集計ロジック

### カード抽出
全カードリストから、以下のフィルタと加工を行います。

1. **基本フィルタ**:
   - `isDraft` (下書き) 除外。
   - `isSilent` (通知OFF) **除外**。ユーザーが意識すべきタスクのみを表示。
   - `isHidden` (非表示フォルダ) 除外。

2. **自動繰越 (Auto Carry Over)**:
   - 設定 `autoCarryOver` が有効な場合、`ScheduledDate < Today` のカードを全て「今日」に加算。
   - これらは `is_overdue = true` となり、UI上で警告色（赤字）で強調されます。

### サイドバー詳細 (Resistance Logic)
日付選択時に右側に表示されるカードリストには、カード個別の成熟度を示すインジケーターが表示されます。

- **指標**: `calculateResistanceScore(interval)`
- **判定ロジック**:
  - **形成中 (Developing)**: Score < 30% → `bg-[#FF5A65]` (Red)
  - **安定 (Stable)**: 30% ≤ Score < 70% → `bg-[#689A98]` (Teal)
  - **高耐性 (Solid)**: Score ≥ 70% → `bg-emerald-500` (Emerald)

---

## 3. UI/UX仕様詳細

### カレンダーグリッド
- **インテンシティ (負荷)**:
  - その日のカード枚数に応じてドット（1〜5個）を表示。
  - 期限切れカードが1枚でも混ざっていれば赤色 (`#FF5A65`) で警告。それ以外はティール色 (`#689A98`)。

### 凡例 (Legend)
画面下部には、変動する「定着度」ではなく、静的な「成熟度」の定義を示します。

- **OVERDUE**: 期限切れ（要対応）
- **SCHEDULED**: 予定あり
- **INTENSITY**: 学習量
- **RESISTANCE**: 成熟度（形成中 / 安定 / 高耐性）

### キーボードショートカット
矢印キーによる日付移動をサポートします（前月・翌月への自動遷移含む）。
