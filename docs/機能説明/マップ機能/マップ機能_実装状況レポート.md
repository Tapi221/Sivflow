## 1. 概要

現在のコードベース（`src/Components/map`, `src/Pages/WorldMap.tsx`）を調査し、仕様書との整合性を確認しました。
結論として、**実装は仕様書の約95%を網羅しており、主要な機能は設計通りに実装されています**。

特に、React + SVG + d3-forceを用いたアーキテクチャ、データモデル、およびHandTrayのフォルダ階層構造への対応は仕様に忠実です。
一部、仕様書に記載があるものの実装が見当たらない機能（自動整列ボタン）や、仕様よりも簡易化されている点（局所地図のレイアウト）が存在します。

## 3. 未実装・相違点 (Discrepancies)

調査により判明した主な相違点は以下の通りです。

### 3.1 自動整列ボタンの欠如 (Missing Feature)

- **仕様書**: 「機能詳細 4. 力学モデル」にて「自動整列ボタンで右→左に矢印を揃える」との記載があります。
- **現状**: `WorldMap.tsx` のUIには「戻る」「設定」「カードを戻すモード」「保存」ボタンしか存在しません。
- **影響**: ユーザーは手動でノードをドラッグして整理する必要があります。d3-forceの物理演算がある程度自然な配置を行いますが、強制的な整列機能はありません。

### 3.2 局所地図 (LocalMap) のレイアウト制限

- **仕様書**: 線の方向について「上下／左右で『深さ／抽象度』を表現」と柔軟な記載があります。
- **現状**: `LocalMap.tsx` のロジックでは、前提関係(Prerequisite)は上、派生関係(Derivation)は下に配置するようハードコードされており、左右配置のロジックはコメントアウトまたは記述されていません。
- **影響**: 幅広い階層構造を持つデータを局所地図で見る際、縦に長くなりすぎる可能性がありますが、現状のモバイル表示等を考慮すると意図的な単純化とも取れます。

### 3.3 AI Pickの実装深度

- **現状**: `HandTray.tsx` に「AI Pick」タブは存在しますが、バックエンドの複雑な推論ロジックと連携しているかはコードの表面上からは簡易的なフィルタリングに見受けられます（`utils/recommendation` の中身次第ですが、フェーズ分けとして簡易実装である可能性が高いです）。

## 4. 結論と推奨アクション

開発チームによる実装は非常に高品質で、仕様書の意図を正確に汲み取っています。特に複雑なSVG操作とReactの連携部分、および再帰的なフォルダ構造のデータハンドリングは堅牢に実装されています。

**非開発者（ユーザー・管理者）向けの推奨事項:**

1. **自動整列ボタンについて**:
   もし「ワンクリックできれいに並べる機能」が必須であれば、開発リストに追加する必要があります。現状でも物理演算によりある程度自然に広がるため、必須機能でないなら仕様書側を修正（削除）しても良いでしょう。
2. **局所地図のレイアウト**:
   現状の「上下配置」はスマートフォン等の縦長画面で見やすいため、この挙動で問題ないか確認してください。
3. **完成度**:
   機能としてはほぼ完成しており、すぐに利用可能な状態です。
