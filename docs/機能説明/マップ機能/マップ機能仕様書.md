# 暗記カードアプリ：マップ機能仕様書 (機能・技術統合版)

**統合日**: 2026-02-02
**対象バージョン**: Phase 3 Implemented
**元データ**: ユーザー提供メモ (memo.txt) および 旧技術仕様書

## 1. 概要 (Overview)

マップ機能は、単なるリスト形式では見えにくい「カード間の関係性」を視覚化し、直感的な操作で知識を体系化するための機能です。

### 1.1 3つの主要ビュー

- **局所地図 (Local Map)**: カード詳細画面などで、そのカードを中心とした直近のネットワークのみを表示する補助マップ。
- **世界地図 (World Map)**: フォルダ全体の構造を無限キャンバス上で俯瞰し、整理・編集するためのメインマップ。
- **手元カードトレイ (Hand Tray)**: マップ未配置のカードや、一時的にマップから退避させたカードを管理するドック領域。

### 1.2 目的

- 関係性を直感的に把握 (Visual Understanding)
- 学習フローを壊さず理解層向け探索体験を提供 (Seamless Exploration)
- 心理的安全性の確保（未作成カードや未接続カードも安心）

## 2. アーキテクチャと技術構成 (Architecture)

### 2.1 技術スタック

- **Drawing Engine**: `React + SVG`
  - DOMイベントとの親和性とカスタマイズ性を重視し、CanvasではなくSVGを採用。
- **Physics Engine**: `d3-force`
  - ノード間の引力・斥力、衝突判定、マウス操作のドラッグ挙動計算に使用。
- **Interaction**: `d3-zoom`, `d3-selection`
  - WorldMapにおけるパン・ズーム操作、およびSVG要素の直接操作に使用。

### 2.2 ディレクトリ構造

```text
src/
  Components/
    map/
      LocalMap.tsx       # 局所地図（学習/詳細画面用）
      HandTray.tsx       # 手元カードトレイ（DnD元）
  Pages/
    WorldMap.tsx         # 世界地図（俯瞰/編集用）
  hooks/
    useGraphSimulation.ts # 力学計算ロジック（d3-force ラッパー）
    useProjectMap.ts      # 配置データのCRUD
    useCardRelations.ts   # 関係性データの取得
```

## 3. 機能仕様詳細 (Feature Details)

### 3.1 局所地図 (Local Map)

- **配置**: カード詳細画面などの下部に埋め込み表示。
- **挙動**: 現在閲覧中のカードを中心(Center Node)とし、直接接続されている前提・派生カードを周囲に展開。
- **方向性**: 上下／左右で「深さ／抽象度」を視覚的に表現（例：前提は上、派生は下）。
- **未接続表現**: 関係性を持たないカードは薄く点滅またはグレーアウト表示。
- **操作**: `HandTray` とのDnD連携が可能。

### 3.2 世界地図 (World Map) / 俯瞰モード

- **概要**: フォルダ単位で作成可能なカスタムマップ。1つのフォルダに複数のマップ作成が可能。
- **操作**:
  - **Zoom/Pan**: `d3-zoom` による自由な視点移動。
  - **配置**: ノード(カード)をドラッグ＆ドロップで任意の位置に配置。
  - **保存**: 現在のノード座標、関係線、接続タイプなどをデータベースに永続化 (`pinned` フラグ)。
- **AI補助**: 関連性の高いカードの配置候補提示（Phase 2以降）。

### 3.3 手元カードトレイ (Hand Tray)

- **概要**: マップ上のノードとしては配置されていないカードをリスト管理するUI。
- **表示スコープ**:
  - 現在のマップが属するルートフォルダ、およびその**全子孫フォルダ**内のカード。
  - マップに既に配置済みのカードはリストから除外される。
- **機能**:
  - **フォルダ絞り込み**: ルート配下の特定サブフォルダのみに絞り込み可能。
  - **検索**: キーワードによるリアルタイムフィルタリング。
  - **ALLタブ**: 全件表示。
  - **AI PICKタブ**: 文脈に基づく推薦表示。
- **インタラクション (Drag & Drop)**:
  - **Tray → Map**: カードをドラッグしてマップ上の任意位置へ配置。
  - **Map → Tray**: マップ上のノードをトレイ領域へドロップすることで、マップから削除しトレイに戻す ("Return to Tray")。

### 3.4 関係線 (Relations)

- **線の種類**:
  - `straight`: 直線（デフォルト）
  - `wave`: 波線（補助関係）
  - `zigzag`: 山線（複雑派生）
  - `double`: 二重線（強い関連）
- **操作**: 線をタップして種類を変更可能。
- **アニメーション**:
  - **Snap**: `snapDistance` 以内に近づくと自動吸着し、接続アニメーション（バウンド）再生。
  - **Detach**: 離脱時は滑らかに線が消滅。

### 3.5 空UI / 初回体験 (Onboarding)

- **初回チュートリアル**:
  - `flashcard_map_tutorial_completed` フラグ管理により、初回アクセス時にガイドを表示。
  - ステップ: Welcome → 配置(Tray) → 接続(Link) → 保存(Save)。
- **空状態 (Empty State)**:
  - 1枚目のカードを配置した際の演出、2枚目の誘導など、白紙の恐怖（Blank Canvas Syndrome）を軽減するマイクロインタラクション。

## 4. データモデル (Data Model)

本機能はDexie.js (IndexedDB) 上の以下のテーブルおよびTypeScriptインターフェースにより管理されます。

### 4.1 永続化データ (Tables)

#### `cardRelations` テーブル

カード間のリンク情報を管理します。

| Field | Type | Description |
| :--- | :--- | :--- |
| `id` | string (UUID) | PK |
| `fromCardId` | string | 接続元カードID |
| `toCardId` | string | 接続先カードID |
| `type` | string | 'related' |
| `lineType` | string | 'straight' \| 'wave' \| 'zigzag' |
| `createdAt` | number | 作成日時 |

#### `projectMaps` テーブル

WorldMapにおけるカードの座標情報を永続化します。

| Field | Type | Description |
| :--- | :--- | :--- |
| `id` | string (UUID) | PK |
| `folderId` | string | 対象フォルダID |
| `nodes` | Array | `MapNodePosition` の配列 |

### 4.2 ランタイム型定義 (Interfaces)

```typescript
// カードエンティティ（既存Card型の拡張）
interface Card {
    id: string;
    title: string;
    tags: Tag[]; // 色分けに使用
    // ...
}

// マップ上のノード座標（projectMaps.nodes の要素）
interface MapNodePosition {
  cardId: string;
  x: number;
  y: number;
  pinned: boolean; // trueの場合、力学モデルの影響を受けず位置固定
}

// 関係性のランタイム表現
interface CardRelation {
    from: string; 
    to: string;   
    lineType: "straight" | "wave" | "zigzag" | "double";
    // 以下はランタイム計算用プロパティ（DBには保存しない場合あり）
    isConnected: boolean;
    idealDistance: number;
    // ...
}
```

## 5. UI/UX基本ルール

### 5.1 カード表示

- **ポップアップ**: ノードタップで詳細ポップアップ表示。裏表切替可能。
- **タグ配色**: カードノードの背景色には、カードの先頭タグの色を適用。

### 5.2 力学モデル設定

- **反発力**: カード同士は重ならないよう反発。
- **吸引力**: 孤立ノードが離れすぎないよう中心付近へ緩やかに吸引。
- **擬似無重力**: ユーザーがドラッグ中は周囲のカードも連動して動き、有機的なつながりを感じさせる。

### 5.3 UX注意点

- 局所地図は「学習フローの補助」、世界地図は「探索・理解」という役割分担を明確にする。
- 「戻すモード (Return Mode)」または「トレイへのドロップ」により、配置ミスからの復帰を容易にする。
- 線の長さは `idealDistance` (推奨距離) に基づき、力学モデルのバネ力で自動調整される。

## 6. 機能制限と課題

### 6.1 機能制限

- **マップ作成条件**: 対象フォルダ（サブ含む）にカードが**2枚以上**あること。単一カードでは関係性が定義できないため。
- **トレイの表示スコープ**: マップのルートフォルダ配下のみ。無関係な別プロジェクトのカードは混入させない。

### 6.2 今後の課題

- **パフォーマンス**: ノード数が数百を超えた場合のSVG描画負荷の最適化（Canvasへの移行検討）。
- **リアルタイム同期**: 複数人編集やデバイス間同期の高頻度更新への対応。
- **自動配置アルゴリズム**: タグやフォルダ構造に基づいた初期配置の自動クラスタリング。
