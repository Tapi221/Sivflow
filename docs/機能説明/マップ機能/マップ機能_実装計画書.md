# マップ機能統合 (World & Local Map) 実装計画

**作成日**: 2026-02-01
**対象**: Phase 3 (Map Feature Integration)
**関連**: `docs/03_Specifications/MapFeature_UserSpec.md`

## 1. 概要
ユーザーがカード間の関係性を視覚的・直感的に操作できる「マップ機能」を実装する。
従来の「局所地図」と大規模な「世界地図（俯瞰モード）」を統合し、手元カードトレイ（HandTray）を用いたスムーズなカード配置体験を提供する。

## 2. 技術選定とアーキテクチャ

### 2.1 レンダリング & インタラクション
- **Simulation Engine**: `d3-force`
    - 理由: 独自の物理挙動（snapDistanceでの吸着、反発力、手元トレイとの連携）を細かく制御するため。純粋な計算エンジンとして使用。
- **Rendering**: `React + SVG`
    - 理由: DOM/SVGイベントハンドラを活用し、Reactのエコシステム（Context, State）と密結合させるため。Canvasベース（react-force-graph等）はカスタマイズ難易度が高く、トレイからのD&D連携が複雑になるため不採用。
- **Drag & Drop**: `react-use-gesture` or `Mouse/Touch Events` wrapper
    - 理由: マップ外（トレイ）からマップ内へのドラッグ、およびマップ内での移動をシームレスに扱うため、低レイヤーの制御が必要。

### 2.2 データ永続化 (Dexie.js + Firestore)
LocalDB (Dexie) を正とし、Firestoreへ同期する。

## 3. データベース設計

### 3.1 新規テーブル: `cardRelations`
カード間の有向エッジを管理する。

```typescript
interface CardRelation {
    id: string;             // UUID
    fromCardId: string;     // FK
    toCardId: string;       // FK
    type: "premise" | "derivative" | "related"; // 前提, 派生, 関連
    lineType: "straight" | "wave" | "zigzag" | "double";
    reasonTag?: string;     // 接続理由（オプション）
    createdAt: number;
    updatedAt: number;
}
```

### 3.2 新規テーブル: `projectMaps` (世界地図用・将来拡張)
特定のコンテキスト（プロジェクト）ごとの配置情報を保存する。局所地図は動的に生成されるため保存不要だが、世界地図は座標保存が必要。

```typescript
interface MapNodePosition {
    cardId: string;
    x: number;
    y: number;
    pinned: boolean; // 自動配置の影響を受けないか
}

interface ProjectMap {
    id: string;
    folderId: string;    // 所属フォルダID（必須）
    name: string;
    nodes: MapNodePosition[];
    createdAt: number;
    updatedAt: number;
}
```
