# 学習モード (Study Mode) 仕様書

## 概要
FSRSアルゴリズムに基づき、カードの復習を行う中核機能です。
ユーザーの自己評価（Recall）に基づいて次回の復習スケジュールを決定します。

## 対象ファイル
- `src/Pages/StudyMode.jsx` (ロジック・状態管理)
- `src/Components/study/StudyCard.tsx` (UI表示・インタラクション)

---

## 1. 状態管理とライフサイクル

### セッション状態 (`StudyMode.jsx`)
| State変数 | 初期値 | 説明 |
| :--- | :--- | :--- |
| `currentIndex` | `0` | 現在学習中のカードインデックス。 |
| `studyComplete` | `false` | 全カード終了時に `true` となり、完了画面へ遷移します。 |
| `results` | `{0:0, 1:0...}` | 各評価（Forgot/Hard/Good/Easy）の選択回数を集計。 |
| `studyCards` | Memo | フィルタリング・ソート済みの学習対象キュー（詳細は後述）。 |

### カード状態 (`StudyCard.tsx`)
| State変数 | 値域 | 説明 |
| :--- | :--- | :--- |
| `studyPhase` | `'timing' / 'answer'` | 表面（出題中）か裏面（回答中）かを管理します。 |
| `elapsedTime` | `0...` | 「答えを見る」を押すまでの思考時間を計測 (秒)。次回スケジュール計算には影響しませんが、ログ(`responseTimeMs`)として記録されます。 |

---

## 2. インタラクション詳細

### 出題フェーズ (`timing`)
- **表示要素**:
  - 問題文 (`question_text`): 数式(`MathRenderer`)を含むテキスト。
  - 画像 (`question_images`): サムネイル表示。全画面表示対応。
  - 音声 (`question_audios`): 再生ボタン (`AudioPlayer`)。**自動再生はされません**。
  - ヒント: "Click card to reveal answer"。
- **操作**:
  - カード全体がクリッカブル領域となっており、クリックで「回答フェーズ」へ移行します。

### 回答フェーズ (`answer`)
- **表示要素**:
  - 問題文（そのまま維持）。
  - **正解 (`answer_text`)**: 水平線 (`hr`) の下に表示されます。
  - **評価ボタン**: 画面下部に4つの選択肢が出現。
    - 忘れた (0): ピンク/赤。復習間隔リセット。
    - あいまい (1): 黄/琥珀。設定 `showReviewHard` でON/OFF可。
    - 覚えた (2): 青。標準的な進捗。
    - 余裕 (3): 緑。設定 `showReviewEasy` でON/OFF可。
- **操作**:
  - 評価ボタンをクリックすると、`handleResult` が実行され、次のカードへ進みます。
  - カード右上の "Back to Question" ボタンで出題フェーズに戻ることも可能です（誤操作防止）。

※ **キーボードショートカット**: 現行実装には含まれていません（将来実装予定）。

## 3. ロジックフロー

### 学習完了時 (`handleResult`)
1. **DB更新 (Parallel Execution)**:
   - `LocalDB.cards`: `memoryStability`, `nextReviewDate`, `reviewCount` 等を更新。
   - `LocalDB.studyLogs`: 学習履歴を追加（オフラインストリーク維持のため重要）。
   - `Firestore`: 学習ログを非同期送信（失敗しても学習は継続するFail-safe設計）。
2. **集計**: `results` ステートを更新。
3. **進行**: `currentIndex` を進める。もし最後のカードなら `studyComplete = true`。

### 完了画面 (Completion)
1. **エフェクト**:
   - `canvas-confetti` による紙吹雪エフェクトが発火。
   - トロフィーアイコンのアニメーション。
2. **スタンプラリー (`StampRally`)**:
   - 本日の学習継続日数（Streak）を表示。
   - `LocalDB.userStats` または `loginHistory` からストリーク情報を再取得して表示。

## 4. UIコンポーネント仕様

### プログレスバー
- **位置**: ヘッダー直下。
- **計算式**: `(currentIndex / totalCards) * 100` (%)。
- **アニメーション**: CSS `transition-all duration-500` により滑らかに推移します。

### 画像/音声 (`MediaUploader` 連携)
- **画像**: `ImageGallery` コンポーネント。クリックでライトボックス（全画面黒背景）表示。読み込みエラー時はプレースホルダーを表示。
- **音声**: HTML5 `<audio>` タグのラッパー。複数音声がある場合は並列してコントロールを表示。排他再生制御あり（Aを再生するとBは停止）。

### カード抽出ロジック（再掲・厳密版）
1. **除外**: `isDraft`(下書き), `isDeleted`(ゴミ箱), `folder.isHidden`(非表示)。
2. **フォルダ整合性**: 所属フォルダIDが現在有効なフォルダリストに存在すること。
3. **日付判定 (Auto Carry Over)**:
   - **ON**: `NextReview <= Today` (過去分も全て含む)
   - **OFF**: `NextReview === Today` (今日分のみ)
