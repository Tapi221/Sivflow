# データ管理・その他機能仕様書

## 概要
データのバックアップや移行、および閲覧専用の補助的な画面機能に関する仕様です。

## 対象ファイル
- `src/Components/export/ExportDialog.tsx`
- `src/Components/export/ImportDialog.tsx`
- `src/Pages/CardView.jsx`
- `src/Components/UserNotRegisteredError.tsx`

---

## 1. データエクスポート (`ExportDialog`)

設定または機能メニューから呼び出され、アプリ内のデータをJSON形式で出力します。

### エクスポート範囲
1. **すべてのデータ**:
   - 全フォルダ、全カード、学習ログ等を含むフルバックアップ (`AppSnapshot`)。
2. **フォルダを選択**:
   - 指定した特定のフォルダとその配下のカードのみをエクスポート。

### 出力フォーマット
- JSONファイル。
- 構造: `{ metadata: {...}, data: { cards: [...], folders: [...] } }`

---

## 2. データインポート (`ImportDialog`)

JSONファイルからデータを復元・取り込みます。

### インポートプロセス
1. **ファイル選択**: ローカルのJSONファイルを読み込み。
2. **プレビューと検証**:
   - ファイル内のカード数・フォルダ数を表示。
   - 既存データとの「世代（Generation）」比較を行い、インポートデータが新しいか古いかを表示（警告）。
3. **アクション選択**:
   - **上書き (Replace)**: 既存データを**すべて削除**し、インポートデータで置き換えます。※危険操作のため警告表示。
   - **キャンセル**: 何もせずに終了。
   - ※マージ機能（既存を残して追加）は現状のUIでは「上書き」か「キャンセル」の二択実装となっています（コード上はUpsertのループ処理ですが、UIフローは破壊的変更を示唆）。

---

## 3. カード詳細閲覧画面 (`CardView`)

`StudyMode`（学習）や `CardEdit`（編集）とは異なり、カードの内容を静的に確認するための閲覧専用画面です。

### 機能
- **閲覧**: 表面、裏面、補足情報、タグなどを一覧表示。
- **音声再生**: 音声ファイルが含まれる場合、再生ボタンを表示。
- **画像拡大**: 画像クリックで拡大表示。
- **遷移**:
  - `Edit`: カード編集画面へ。
  - `Study`: このカードを含むフォルダの学習モードへ。
- **ステータス表示**: 現在の定着度（Memory Stability）を可視化。

---

## 4. エラー・制限画面

### `UserNotRegisteredError`
- **概要**: 認証済みだが、アプリ利用権限がないユーザーに表示されるアクセス制限画面。
- **表示条件**: ログインは成功したが、Firestore上のユーザー登録チェック等で弾かれた場合（想定される管理者制御）。
