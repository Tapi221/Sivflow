# 学習統計画面 (Statistics) 仕様書

## 概要
ユーザーの学習進捗、記憶の定着度、および将来の予測を可視化する分析画面です。
単なるグラフ表示だけでなく、「FSRS理論（忘却曲線）」に基づいて、「なぜ今復習が必要なのか」をユーザーに直感的に伝える役割を持ちます。

## 対象ファイル
- `src/Pages/Statistics.jsx`
- `src/Components/stats/StatsCharts.tsx`
- `src/Components/stats/LevelHistory.tsx`
- `src/utils/statistics.ts`

## 1. フィルタリング仕様

統計データは、画面上部の「フォルダ選択」プルダウンによって動的に再計算されます。

### 階層集計ロジック (`getDescendants`)
フォルダ構造は親子関係を持つため、親フォルダを選択した際には**その子孫フォルダ全てのカード**を集計対象とします。

1. **ツリー展開**: 選択された `folderId` をルートとして、再帰的に全ての子フォルダID (`child.parentFolderId === currentId`) を収集します。
2. **カード抽出**: 収集された全てのフォルダIDに所属するカード (`filteredCards`) を抽出します。

### 除外カード
以下のカードは、どのフォルダが選択されていても常に集計から除外されます。
- **ドラフト**: `isDraft === true` (作成中)
- **ゴミ箱**: `isDeleted === true` (論理削除済み)
- **非表示フォルダ**: 親フォルダ設定等で `isHidden === true` となっているカード。

## 2. 定着度分布 (Retention Distribution)

現在の学習セットに含まれるカードが、どの程度の「忘れにくさ（定着度）」を持っているかを分布図で示します。

### 定義: 定着度 (Retention Probability)
- **計算式**: $P = e^{ - \frac{I}{S \times 100} }$
- **意味**: 次の復習予定日において、記憶が保持されている確率（推定値）。復習直後は100%に近く、時間が経つにつれて減少します。

### 分布チャートのロジック
全対象カードの定着度 ($P$) を計算し、以下のフェーズごとの割合を円グラフまたは棒グラフで表示します。

| フェーズ | 範囲 ($P$) | 色設定 (Tealテーマ) | 意味 |
| :--- | :--- | :--- | :--- |
| **要復習 (Unstable)** | $0\% \le P < 20\%$ | 淡紅 ([#FCA5A5]) | 危険域。直ちに復習が必要。 |
| **覚えかけ (Fragile)** | $20\% \le P < 40\%$ | 淡橙 ([#FDBA74]) | 初期段階。頻繁なケアが必要。 |
| **定着途上 (Growing)** | $40\% \le P < 65\%$ | 淡黄 ([#FDE047]) | 順調に育っている段階。 |
| **安定 (Stable)** | $65\% \le P < 85\%$ | 淡碧 ([#99F6E4]) | 安心して放置できる期間が増加。 |
| **長期保持 (Solid)** | $85\% \le P \le 100\%$ | ブランドTeal ([#689A98]) | 非常に強い記憶。王冠マーク対象。 |

### チャートのレスポンシブ・各種表現
- **X軸目盛り**: 画面幅 768px 以上の場合は **5%刻み**、それ未満（モバイル）は **10%刻み** で表示します。
- **リファレンスライン**: 
    - 「安定ライン (65%)」「王冠ライン (85%)」を表示。
    - ラベルはグラフ上部エリアに整列配置され、グラフ内の点線と **点線コネクタ** で視覚的に接続されます。
- **フェーズバー**: 
    - グラフ下部に重なり合う矢印状のバーを表示。
    - 主張を抑えるため高さ **h-5 (20px)** に最適化されています。
    - 重なり（ネガティブマージン）を考慮し、合計幅がコンテナの100%に適合するよう再計算されています。

## 3. レベル履歴 (Level History)

過去の学習によって、各カードの「レベル/安定度」がどのように推移してきたかを時系列グラフ（エリアチャート）で表示します。

### データソース
- **Firestore**: `levelHistories` コレクション（日次スナップショット）。
- **LocalDB**: `levelHistories` テーブル（オフライン対応用）。

### 集計ロジック
直近7日間、30日間、または全期間の履歴データを取得し、各日時点での「フェーズごとのカード枚数」を積み上げグラフとして描画します。
これにより、学習を継続することで「赤（要復習）」が減り、「緑（安定）」が増えていく様子を視覚化します。

---

## 4. 忘却曲線シミュレーション (Recall Probability)

FSRS理論教育用のパネルです。数式に基づいた理想的な曲線をプロットします。

- **X軸**: 経過日数 (Days)
- **Y軸**: 記憶保持率 (Retention %)
- **曲線**: $y = e^{ - \frac{x}{S \times 100} }$
  - $S=0.1$ (Unstable) から $S=1.0$ (Solid) までの代表的な値を複数プロットし、安定度が高まるにつれて曲線が緩やかになる（忘れにくくなる）様子を示します。

## 5. UI/UX詳細

### ローディング状態
- 集計中は `Skeleton` コンポーネントを表示し、レイアウトシフトを防ぎます。
- 特に `LevelHistory` の取得は重い処理（数万レコードの可能性）になるため、`useQuery` でキャッシュ制御を行います。

### 空の状態 (Empty State)
- フィルタ結果が0件の場合、「学習データがありません」というメッセージと共に、学習モードへの誘導ボタンを表示します。
