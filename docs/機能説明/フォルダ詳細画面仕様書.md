# フォルダ詳細画面 (Folder View) 仕様書

**最終更新日時**: 2026-02-04 03:22 (JST)

## 概要
特定のフォルダ内にある学習カードやサブフォルダを管理する画面です。
カードリストの閲覧・編集だけでなく、学習資料（PDF）の管理やフォルダメモの記述も行えます。

## 対象ファイル
- `src/Pages/FolderView.jsx` (メイン画面)
- `src/hooks/useFolders.ts` (フォルダ操作フック)
- `src/hooks/useCards.ts` (カード操作フック)
- `src/services/localDB.ts` (データ永続化・修復)

---

## 1. 画面構成とタブナビゲーション

以下の4つのタブで構成されています。

| タブ名 | キー | 機能概要 |
| :--- | :--- | :--- |
| **CARDS** | `cards` | カードリストの表示・ソート・フィルタリング・編集。 |
| **MEMO** | `memo` | フォルダに関するメモ書き。Markdown形式で保存されます。 |
| **NOTES** | `notes` | 学習資料（PDF）のアップロードと管理。 |
| **MAP** | `map` | (開発中) カード間の関連性を可視化するマップビュー。 |

---

## 2. CARDS タブ仕様

### フィルタリング (Filter Mode)
- **All (標準)**: すべてのカードを表示。
- **王冠あり**: 定着度（Stability）が一定以上（デフォルト85%）のカード。
- **王冠なし**: 定着度が85%未満のカード。

### ソート機能 (Sort Mode)
- **カスタム (標準)**: ドラッグ＆ドロップによる自由な並べ替え。`orderIndex` で管理。
- **定着度**: 記憶の定着度順。
- **作成日**: カードの作成日時順。

### 同期インジケーター
- フォルダ内の変更が発生すると、ヘッダー付近に同期ステータス（同期中/完了）が表示されます。
- `SyncServiceV2` と連携し、オフライン時は「待機中」としてキューに蓄積されます。

---

## 3. サブフォルダ管理 (Sub Folders)
- **階層構造**: 親フォルダ ID (`parentId`) を持つことでツリー構造を形成します。
- **移動**: カードをサブフォルダのタイルにドラッグ＆ドロップすることで、フォルダ間の移動が可能です。
- **作成/削除**: `useFolders` フックを介して `LocalDB` に反映され、即座に同期キューへ追加されます。

---

## 4. NOTES タブ (資料管理)
- **PDFアップロード**: `Firebase Storage` の `users/{uid}/notes/{folderId}/` へ保存。
- **メタデータ管理**: アップロード完了後、フォルダオブジェクトの `notePdfs` フィールドにURLやファイル名が記録されます。
- **閲覧**: ブラウザの標準PDFビューアで表示。

---

## 5. MEMO タブ
- **機能**: フォルダ固有の Markdown メモ。
- **保存**: 入力内容は `folders` テーブルの `memo` フィールドに保存されます。

---

## 6. 同期とデータ整合性管理

### 同期ロジック (Sync v2)
- フォルダ詳細画面でのあらゆる変更（名前変更、移動、削除、ソート順変更）は、`SyncServiceV2` の「Pull-then-Push」フローによって同期されます。
- **不具合対策**: Firestore の `updatedAt` (Timestamp) 比較を正しく行うことで、他デバイスでのフォルダ構成変更を漏らさず反映します。

### データ修復 (`repairDataIntegrity`)
フォルダ情報の整合性が崩れた場合（例：親フォルダが削除された孤立フォルダなど）、`localDB.ts` の修復ロジックが自動的に以下の処理を行います。
- 親フォルダが見つからない場合はルート（未分類）へ移動。
- 名前が空のフォルダに「名称未設定フォルダ」を割り当て。
- 重複する `orderIndex` の再採番。

### データ復旧
デバイスからデータが消失した場合、`IndexedDBRebuildOrchestrator` がクラウドから全てのフォルダ・カード情報を再ダウンロードし、この画面の表示内容を復元します。
