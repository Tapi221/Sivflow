この# 設定画面仕様書

現在のアプリケーションにおける設定画面（SettingsDialog）の機能と仕様を詳細にまとめます。

## 概要
設定画面はモーダルダイアログとして実装され、左側のサイドバーでカテゴリを切り替えるタブ形式を採用しています。

## タブ一覧

| ID | ラベル | アイコン | 説明 |
| :--- | :--- | :--- | :--- |
| `account` | アカウント | User | プロフィール、ユーザー名、アップロードファイル管理 |
| `study` | 学習設定 | BookOpen | 学習ロジック、表示設定、スケジュール設定 |
| `theme` | テーマ | Moon | 外観モード、カレンダー設定 |
| `tags` | タグ管理 | Tag | タグの色変更、削除 |
| `voice` | 音声設定 | Volume2 | TTS（読み上げ）の設定 |
| `sync` | 同期設定 | RefreshCw | クラウド同期、デバイス管理、容量確認 |
| `data` | データ | Database | データのバックアップと修復 |

---

## 各タブの詳細仕様

### 1. アカウント (Account)
ユーザーの基本情報と、アップロード済みファイルの個別管理を行います。

#### プロフィール設定
- **プロフィール画像**:
    - **アップロード**: JPEG, PNG, HEIC形式に対応（HEICは自動的にJPEG変換）。最大10MB。
    - **表示**:
        - オンライン時: Cloud StorageのURLを使用。
        - オフライン時/失敗時: ローカルURLまたはフォールバックアイコンを表示し、再試行が可能。
        - ステータスバッジ: ローカルのみ保存されている場合は「Database」アイコンを表示。
- **ユーザー名**:
    - 表示・編集が可能。編集時はインライン入力フィールドに切り替わります。
    - ログアウトボタンをサイドバー下部に配置。

#### ストレージ管理 (`StorageManager`)
Cloud Storage上の個別のファイルを管理します（同期クォータとは別視点）。
- **使用量表示**:
    - 合計使用量とファイル総数を表示。
- **ファイルリスト**:
    - アップロード済みのファイルを時系列順に表示（最新のもの）。
    - 各ファイルのファイル名、サイズ、MIMEタイプアイコンを表示。
    - **削除機能**: 個別のファイルを削除可能（FirestoreメタデータとStorage実体の両方を削除）。
- **クリーンアップ**:
    - 「不要ファイルを整理」ボタン: 24時間以上経過した「失敗（failed）」ステータスのアップロード記録を一括削除します。

---

### 2. 学習設定 (Study)
学習アルゴリズム（FSRSなど）の動作や、学習画面のUIをカスタマイズします。

#### マスター判定 (王冠マーク)
- **王冠の閾値**:
    - 定着度（Stabilityなどに基づく算出値）が何%を超えたら王冠を表示するか設定。
    - 範囲: 0% ～ 100% (5%刻み)。デフォルト: 85%。
- **王冠カードの復習を継続**:
    - ON: 王冠が付いても通常のスケジュールに従って復習に出現させる。
    - OFF: 王冠が付いたら復習スケジュールから除外する（仕様実装依存）。

#### レビューボタン表示設定
学習画面下部の評価ボタンの表示制御。
- **忘れた (Forgot)**: 常に表示。復習間隔リセット。
- **あいまい (Hard)**: ON/OFF可能。復習間隔を微増（1.2倍程度など）。
- **覚えた (Good)**: 常に表示。基本の復習間隔。
- **余裕 (Easy)**: ON/OFF可能。復習間隔を大幅増（ボーナス係数適用）。

#### スケジュール設定
- **未消化カードの自動繰越**:
    - ON: 当日消化しきれなかったカードを翌日の「今日の復習」に自動で含める。
- **遅延ボーナス**:
    - ON: 予定日より遅れて復習し、かつ正解した場合に、遅れた期間分を加味して次回の間隔を伸ばす（FSRSのロジックに影響）。
- **復習開始日**:
    - **当日**: カード作成直後から復習対象にする。
    - **翌日**: 作成当日は復習に出さず、翌日から開始する。
    - ※ この設定を変更すると、本日作成された未学習（レビュー回数0）のカードの `nextReviewDate` が即時に一括更新されます。

---

### 3. テーマ (Theme)
- **外観モード**: ライト / ダーク / システム設定。
- **カレンダー設定**:
    - **週の始まり**: 日曜始まり / 月曜始まり の切り替え。

---

### 4. タグ管理 (Tags)
タグ作成はフォルダ画面等で行いますが、ここでは管理を行います。
- **ルートフォルダフィルタ**: 選択したルートフォルダに属するタグのみを表示。
- **インクリメンタル検索**: タグ名でリアルタイム検索。
- **カラー変更**: プリセットパレット（パステルカラー中心）から選択して変更。
- **削除**: タグの使用を停止・削除（紐付いているカードからは解除されます）。

---

### 5. 音声設定 (Voice)
TTS（Text-to-Speech）エンジンを使用した自動読み上げ設定。
- **自動再生トリガー**:
    - 問題表示時（カードの表面が出た瞬間）。
    - 解答表示時（裏面を開いた瞬間）。
- **音声モデル選択**:
    - OpenAI / Gemini 系のTTSモデル（Kore, Puck, Charon, Fenrir, Zephyr）から選択可能。

---

### 6. 同期設定 (Sync)
クラウド同期とデバイス連携の核心部分です。

#### 同期ステータス
- **現在の状態**: 同期中 / エラー / 同期済み / オフライン。
- **最終同期時刻**: 最後に成功した日時を表示。
- **手動同期**: 「今すぐ同期」ボタンで強制同期トリガー。

#### 基本設定
- **同期間隔**: 5分 / 15分 / 30分 / 60分。
- **WiFi接続時のみ同期**: モバイル通信量を節約する設定。
- **フォルダごとの同期**: ルートフォルダ単位でクラウド同期の有効/無効を切り替え。

#### デバイス・ストレージ管理 (`DeviceSyncSettings`)
- **ストレージ容量 (Quota)**:
    - 上限: **500MB** (無料プラン想定)。
    - 使用量の内訳バー: 「高解像度画像」と「サムネイル」の比率を可視化。
    - 警告: 使用率90%超で赤色警告を表示。
- **デバイス管理**:

    - **デバイスリスト**:
        - デバイス名、ID、ステータス（ACTIVE/DISCONNECTED）、最終同期時刻を表示。
        - **解除機能**: 他の端末または現在の端末の登録を解除（Disconnect）可能。
- **画像同期ポリシーの注意書き**:
    - デフォルトではサムネイルのみを全端末に同期し、高解像度画像は学習などで必要になった際にオンデマンドで取得する「スマート同期」方式を採用している旨の記載。

---

### 7. データ (Data)
トラブルシューティングとデータ復旧のための高度な機能です（`DataRescuePanel`）。

#### 概要表示
現在のデータベース内のフォルダ数・カード数のリアルタイムカウント。

#### バックアップからの復元
ブラウザの IndexedDB 内にある、過去のデータベースバージョンやバックアップをスキャンします。
- **スキャン**: `LocalDB.listDatabases()` を使用してブラウザ内のDBを一覧化。Firebase内部DBなどは除外。
- **リスト表示**: 検出されたバックアップのカード数・フォルダ数を表示。
- **復元実行**: 選択したバックアップの内容で現在のデータを上書き（インポート）します。完了後にリロードが必要。

#### 高度な機能 (Advanced Tools)
- **データの正規化 (Normalization)**:
    - `repairDataIntegrity`: 整合性が取れていないデータ（親がいないフォルダ、重複データなど）を検出し、修復・整理します。
- **詳細スキャン (Forensic Audit)**:
    - 開発者向けツール。ブラウザのストレージ領域を詳細にスキャンし、コンソールにログを出力します。
