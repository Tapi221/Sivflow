# 復習アルゴリズム (Review Algorithm) 仕様書

## 概要
`src/services/reviewAlgorithm.ts` に実装されている、FSRS (Free Spaced Repetition Scheduler) ベースのスケジューリングロジックの厳密な仕様です。

## 定数定義
| 定数名 | 値 | 説明 |
| :--- | :--- | :--- |
| `MIN_STABILITY` | `0.01` | 定着度の下限値 (ゼロ除算防止)。 |
| `MAX_STABILITY` | `1.0` | 定着度の上限値 (100%)。 |
| `MAX_INTERVAL_DAYS` | `90` | 復習間隔の最大日数。これ以上の間隔は空けない。 |

## ロジックフロー (内部計算)

```mermaid
graph TD
    A[学習開始] --> B{初回?}
    B -- Yes --> C[S = 0.30]
    B -- No --> D[S = DB保存値]
    
    C --> E[ユーザー評価入力]
    D --> E
    
    E --> F{評価は?}
    F -- 0:忘れた (Failed) --> G[S = S * 0.5]
    F -- 1:あいまい (Hard) --> H[S = S * 0.8]
    F -- 2:覚えた (Good) --> I[S = S + (1-S)*0.1]
    F -- 3:余裕 (Easy) --> J[S = S + (1-S)*0.25]
    
    G --> K{Delay Bonus?}
    H --> K
    I --> K
    J --> K
    
    K -- Yes --> L[Sにボーナス加算]
    K -- No --> M[S固定]
    
    L --> M
    
    M --> S[更新データ保存]
```

### 1. 初期化と正規化 (Initialization)
カードデータには新旧フォーマットが混在する可能性があるため、計算前に `memoryStability` (0.01 - 1.0) へ正規化します。

- **新規カード**: デフォルト値 `0.30` (30%)。
- **正当な値 (0-1)**: そのまま使用（`clamp` で 0.01-1.0 に収める）。
- **Legacy値 (5-100)**: `(val - 5) / 95` で 0-1 に変換。
- **Legacy Level (0-5)**: `0.1 + (level / 5) * 0.8` で変換。

### 2. 定着度更新 (Update Stability)
ユーザー評価 ($G$) に応じて、現在の安定度 ($S$) を更新します。

$$ S_{new} = \text{clamp}( f(S, G) ) $$

#### 評価別係数
1. **忘れた (Forgot / Score 0)**:
   $$ S_{new} = S \times 0.5 $$
   (記憶喪失として50%減衰させる)
2. **あいまい (Hard / Score 1)**:
   $$ S_{new} = S \times 0.8 $$
   (不安定さを示すため20%減衰)
3. **覚えた (Good / Score 2)**:
   $$ S_{new} = S + (1 - S) \times 0.1 $$
   (「未習得領域」の10%分だけ成長)
4. **余裕 (Easy / Score 3)**:
   $$ S_{new} = S + (1 - S) \times 0.25 $$
   (「未習得領域」の25%分を一気に成長)

### 3. 遅延ボーナス (Delay Bonus)
設定 `delayBonusEnabled` が有効、かつ復習予定日を過ぎている場合、遅れても正解できたことに対するボーナスを与えます。

$$ \text{DelayDays} (D) = \max(0, \text{Today} - \text{ScheduledDate}) $$
$$ S_{final} = S_{new} \times (1 + \log_2(1 + D)) $$

※ $D > 0$ の場合のみ適用。

### 4. 間隔計算 (Interval Calculation)
最終的な安定度 $S$ ($0.01 \le S \le 1.0$) から、次回までの日数 $I$ を決定します。

$$ I = \text{round}( 1 + 100 \times S^{2.5} ) $$

---

## UI Metrics Definition (表示指標)

内部の `memoryStability` は直接ユーザーには見せず、以下の2つの指標に分離して表示します。

### 1. 定着度 (Retention Probability)
「次の復習日まで記憶を保持できる確率（推定値）」を表します。

- **計算式**:
  $$ P = e^{- \frac{I}{S \times 100}} $$
  ($I$: 次回までの日数, $S$: Stability)
- **意味**: 時間経過とともに減少する確率論的な値です。
- **注意**: ※ 本確率は学習状態の目安であり、実際の想起成功を保証するものではありません。

#### フェーズ分類 (Retention Phase)
定着度 ($P$) に基づいて、カードの状態を分類します。

| Retention ($P$) | フェーズ名 | 状態 |
| :--- | :--- | :--- |
| **< 20%** | **要復習** | 危険域。直ちに復習が必要。 |
| **20 - 40%** | **覚えかけ** | 初期段階。頻繁なケアが必要。 |
| **40 - 65%** | **定着途上** | 順調に育っている段階。 |
| **65 - 85%** | **安定** | 安心して放置できる期間が増加。 |
| **> 85%** | **長期保持** | 非常に強い記憶。王冠マーク対象。 |

### 2. 耐性スコア (Resistance Score)
「どれくらい長い復習間隔に耐えられているか」を表します。

- **計算式**:
  $$ R = \frac{\log_{10}(1 + I)}{\log_{10}(1 + \text{MAX\_INTERVAL})} \times 100 $$
- **意味**: スケジュールされた期間の長さに基づく、積み上げ型のスコアです。
- **UI**: 補助情報として表示（ホバー時や詳細画面など）。
