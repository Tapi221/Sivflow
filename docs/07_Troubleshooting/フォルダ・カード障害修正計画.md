# フォルダ/カード保存・同期障害の修正計画

## 問題の概要
1.  **Firebase初期化のハング**: `src/services/firebase.ts` の `catch` ブロック内で `await import` を使用しており、Firestoreの初期化に失敗（二重初期化など）した場合にモジュールの評価が停止し、依存するコンポーネントがハングしている。
2.  **Firestoreインスタンスの不在**: `firestoreDb` が `null` のままエクスポートされる可能性があり、`collection(db, ...)` などの呼び出しで `FirebaseError` が発生している。
3.  **UIのエラーハンドリング不足**: 保存処理中にエラーが発生した場合、`isLoading` が `false` に戻らず、「保存中」の状態が継続してしまう。

## 提案される変更点

### 1. Firebase初期化の改善 [src/services/firebase.ts](file:///c:/FlashcardMaster/src/services/firebase.ts)
- `getFirestore` を静的にインポートする。
- `catch` ブロック内の `await import` を削除し、同期的な `getFirestore(app)` に置き換える。
- `_firestoreDb` が確実に設定されるようにロジックを簡素化する。

### 2. UIのエラーハンドリングの追加 [src/Components/folder/FolderDialog.tsx](file:///c:/FlashcardMaster/src/Components/folder/FolderDialog.tsx)
- `handleSubmit` に `try...finally` を追加し、保存の成功・失敗にかかわらず `setIsLoading(false)` が呼ばれるようにする。

### 3. 同期アダプターのガード強化 [src/services/logic/CloudSyncAdapter.ts](file:///c:/FlashcardMaster/src/services/logic/CloudSyncAdapter.ts)
- 各メソッドの冒頭で `firestoreDb` の存在チェックを徹底し、未初期化時の予期せぬエラー（`collection(null, ...)`）を防止する。

## 検証計画

### 自動テスト
- 現状、Firestoreの初期化を直接テストするユニットテストがないため、手動検証を中心に実施する。

### 手動検証
1.  アプリケーションを起動し、コンソールに Firebase 関連のエラーが出ていないか確認する。
2.  新規フォルダ作成を試み、「保存中」で止まらずに完了することを確認する。
3.  過去のカードが表示されるか（Dexieへの同期が走るか）を確認する。
4.  LocalStorage/IndexedDB をクリアして再ログインし、フル同期が正常に完了することを確認する。
