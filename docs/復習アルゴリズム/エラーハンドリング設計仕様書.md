# エラーハンドリング・信頼性設計仕様書

本プロジェクトでは、オフラインファーストおよびデータの永続性を極めて重視し、単純な例外処理を超えた多層的な信頼性確保メカニズムを実装しています。

## 1. データの整合性と自己修復機能

単なるエラー回避ではなく、データが「壊れていないか」をチェックし、必要に応じて自動修復します。

### データの整合性スキャン (`DataIntegrityService.ts`)
- **プロアクティブ・チェック**: 起動時に全カードとフォルダの親子関係、必須フィールドの有無をスキャンします。
- **孤立資産の隔離 (Quarantine)**: 親フォルダが存在しない「孤立カード」を検出し、ゴミ箱または専用の復旧用フォルダへ退避させ、データロストを防止します。

### 自動整合性修復 (`localDB.ts - repairDataIntegrity`)
- **所有権の統一**: ログイン時にローカルデータの所有者IDを現ユーザーへ自動的に付け替え、同期対象から漏れるのを防ぎます。
- **フィールド補完**: 名前がないフォルダへの自動命名、削除フラグの論理整合性を自動修正します。
- **Blob URL の永続化防止**: データベース保存直前に temporary な `blob:` URLを自動除去し、次回起動時の画像読み取り失敗を防止します。

## 2. 高度なデータ復旧メカニズム

壊滅的なデータの不整合が発生した際、手動または自動でデータをサルベージする手段を提供します。

### データ・フォレンジック調査 (`LocalDB.fullOriginForensicAudit`)
- **全オリジンスキャン**: ブラウザ内の全IndexedDBを低レベルで走査し、データの断片をコンソールに出力して救出します。

### 内容のハイドレーション (Content Hydration)
- **生データ解析**: `_rescueRaw` に保持されている Firestore の生データ（JSON）を解析し、フィールドが欠落していても中身を再構築します。
- **Firestore SDK キャッシュ抽出**: SDK 内部の `remoteDocuments` テーブルから直接ドキュメントを吸い出す特殊ロジックを備えています。

## 3. 同期システムの分散制御と信頼性 (`syncService.ts`)

複数端末での同時使用や、頻繁なオフライン状態に耐える設計です。

### 分散同期ロック (Distributed Sync Lock)
- **競合防止**: Firestore 上の `sync_locks` ドキュメントを利用し、複数端末が同時に書き込みを行うのを防止します。
- **タイムアウト**: ロックは5分で自動失効（expiresAt）し、異常終了時も同期が永久にロックされるのを防ぎます。


### オフラインキュー (`SyncQueue`)
- オフライン中の操作をFIFOキューに保存し、オンライン復帰時に優先的に処理します。

## 4. ファイル管理の信頼性 (`imageSyncService.ts` / `useReliableFileUpload.ts`)

画像のアップロードは最もエラーが発生しやすい箇所であるため、特に堅牢な設計を施しています。

### 重複排除と再開 (Deduplication & Resume)
- **SHA-256 チェックサム**: 全ファイルのハッシュ値を計算。すでにクラウドに同じハッシュの画像が存在する場合は、転送をスキップして再利用します。
- **容量制限 (500MB Quota)**: ストレージ使用量をローカルでリアルタイム監視し、制限を超えたアップロードを未然に防ぎます。

### エラー耐性ロジック
- **指数バックオフ (Exponential Backoff)**: アップロード失敗時、2^n 秒の待ち時間を設けて最大5回まで自動リトライします。
- **ローカル・ハイブリッド保持**: アップロードが最終的に失敗しても、Base64 形式でローカル DB に保存し、学習を中断させません。

## 5. クラウド同期の安全性 (`cloudProvider.ts` / `imageUtils.ts`)

### データ・シリアライゼーションの厳格化
- **Firestore undefined 徹底排除**: `stripUndefined: true` 設定による正規化により、Firestoreでエラーとなる `undefined` 値が混入するのを通信直前で100%遮断します。
- **キャメルケース変換**: 通信パラメーターの不一致を防ぐためのネーミングコンベンション強制適用。

## 6. アプリケーション実行基盤の保護

### チャンク読み取りエラーの自動復旧 (`main.tsx`)
- デプロイ直後に発生する「古いJSファイルの欠落エラー (ChunkLoadError)」を監視し、検知時に自動でページをリフレッシュして最新版を読み込ませます。

### バックアップと世代管理 (`AutoBackupService.ts`)
- **日次自動バックアップ**: 1日1回、スナップショットを自動作成。
- **世代ローテーション**: 最大7世代（7日分）のバックアップを保持し、`generationCounter` による厳密なバージョン管理を行います。

## 7. UI/UXにおけるエラー提示

- **DataRescuePanel**: 上記の高度な復旧機能をユーザーが手動で実行できるインターフェースを提供しています。
- **決済エラーハンドリング**: Stripe Checkout へのリダイレクト失敗時などの外部サービス連携エラーを検知し、ログ出力とユーザー通知を行います。
