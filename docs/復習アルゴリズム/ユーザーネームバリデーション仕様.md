# ユーザーネームバリデーション仕様（バックエンド・スキーマ）

フロントエンドの実装（`userValidation.ts`）に対応する、バックエンドおよびデータベース層でのバリデーション仕様です。

## 1. データベース・スキーマ定義

### Firestore スキーマ
ユーザー設定（`userSettings`）コレクション内の `displayName` フィールドとして保存されます。

- **コレクション**: `userSettings`
- **ドキュメントID**: `{userId}` (Firebase Auth の UID)
- **フィールド**:
  - `displayName` (string): ユーザーの表示名
  - `updatedAt` (timestamp): 最終更新日時

```typescript
// Firestore ドキュメント構造例
{
  "displayName": "アカリ_ユーザー",
  "userId": "auth_uid_123",
  "updatedAt": Timestamp(2026-01-29T05:00:00Z)
}
```

## 2. バックエンド・バリデーション関数 (Node.js / TypeScript)

Firebase Cloud Functions で実装する場合の例です。フロントエンドと同じ正規表現と文字カウントロジックを使用します。

```typescript
/**
 * ユーザーネームのバックエンドバリデーション
 * 使用言語: Node.js (TypeScript)
 */
import * as functions from 'firebase-functions';

// 使用可能文字の正規表現（フロントエンドと同一）
const ALLOWED_CHARS_REGEX = /^[a-zA-Z0-9\s\-_ \u3040-\u309F\u30A0-\u30FF\u4E00-\u9FAF\u3400-\u4DBF]+$/;

export const validateUsernameBackend = (username: string): { isValid: boolean; message: string } => {
  const trimmedName = username.trim();

  // 1. 空白・必須チェック
  if (!trimmedName) {
    return { isValid: false, message: "ユーザーネームを入力してください。" };
  }

  // 2. 文字数チェック (Unicode単位でのカウント)
  // Intl.Segmenter を使用するとより正確に書記素単位でカウント可能
  const segmenter = new Intl.Segmenter('ja-JP', { granularity: 'grapheme' });
  const charCount = Array.from(segmenter.segment(trimmedName)).length;

  if (charCount < 1 || charCount > 20) {
    return { isValid: false, message: "ユーザーネームは1文字以上20文字以内で入力してください。" };
  }

  // 3. 文字種チェック
  if (!ALLOWED_CHARS_REGEX.test(trimmedName)) {
    return { isValid: false, message: "使用できない文字が含まれています。" };
  }

  return { isValid: true, message: "" };
};

// 使用例: Firestoreの書き込みトリガーなど
// export const onUpdateSettings = functions.firestore.document('userSettings/{uid}').onWrite((change, context) => { ... });
```

## 3. Firestore セキュリティルール

データベース層での最終防衛ラインとして、セキュリティルールを設定します。

```javascript
service cloud.firestore {
  match /databases/{database}/documents {
    match /userSettings/{userId} {
      allow read: if request.auth != null && request.auth.uid == userId;
      allow write: if request.auth != null && request.auth.uid == userId
        && request.resource.data.displayName is string
        && request.resource.data.displayName.trim().size() >= 1
        && request.resource.data.displayName.trim().size() <= 60 // size() はバイト数または文字列長（ルール上は単純長）
        // 複雑なUnicode正規表現はセキュリティルールでは限定的なため、詳細はFunctions側で担保
        && request.resource.data.displayName.matches('^[\\p{L}\\p{N}\\s\\-_]+$'); 
    }
  }
}
```

## 4. エラーメッセージの例文

ユーザーに表示するメッセージの一覧です。

| 状況 | エラーメッセージ |
| :--- | :--- |
| 空白のみ / 未入力 | ユーザーネームを入力してください。（空白のみは不可） |
| 21文字以上 | ユーザーネームは20文字以内で入力してください。 |
| 使用不可な記号 | ユーザーネームに使用できない文字が含まれています。（日本語、英数字、スペース、ハイフン、アンダースコアが使用可能です） |
| システムエラー | ユーザーネームの保存に失敗しました。時間をおいて再度お試しください。 |
