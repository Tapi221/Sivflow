# デザイン思想記録：定着度の視覚化 (Physics to Feeling)

## 1. 課題：数字の冷たさと認知負荷

FSRSアルゴリズムのコアパラメータである `Memory Stability (S)` は、**「記憶が10%の確率で想起できなくなるまでの日数」**という物理学的な定義を持ちます。
しかし、これをそのままUIに表示することには問題がありました。

* **直感的でない**: 「安定度: 25.5」と言われても、ユーザーは喜んでいいのか警戒すべきか分からない。
* **行動に繋がらない**: 数字を見ても「今復習すべきか？」「あと3日放置していいか？」の判断ができない。

私たちは、**「物理量 (Physics)」を「感情・感覚 (Feeling)」に変換するレイヤー**を挟むことにしました。

---

## 2. 変換ロジックの物語

### なぜ "Stability" を隠したか

Stabilityは指数関数的な性質を持ちます（日数計算に使われる）。ユーザーにとって重要なのは「何日持つか」ではなく、**「今、私の頭の中に記憶は残っているのか？」**という現在進行系の不安です。
そのため、Stability（日数）ではなく、そこから逆算された **Retention Probability（確率）** をメイン指標に採用しました。

### メタファー：信号機 (Traffic Light)

複雑な確率論を、誰もが知っている「信号機」のアナロジーに落とし込みました。

| 確率 (Retention) | 色 | 意味 (User Feeling) | 行動 (Action) |
| :--- | :--- | :--- | :--- |
| **85% - 100%** | **Solid (#689A98)** | **「長期保持」** - 安心、誇り | 放置してOK。王冠マーク誇示。 |
| **65% - 85%** | **Stable (#99F6E4)** | **「安定」** - 順調 | 学習は順調に進んでいる。 |
| **40% - 65%** | **Growing (#FDE047)** | **「定着途上」** - 成長期 | 油断せず復習を継続。 |
| **20% - 40%** | **Fragile (#FDBA74)** | **「覚えかけ」** - 注意 | 記憶の定着が始まる重要な時期。 |
| **< 20%** | **Unstable (#FCA5A5)** | **「要復習」** - 緊急 | **今すぐ復習せよ！** 記憶が消えかけている。 |

このカラーコーディングにより、従来の原色（信号機色）よりもアプリ全体の Teal/Slate テーマに溶け込みつつ、**「画面全体が寒色に近づくほど安全、暖色に近づくほど危険」**という直感的な色の温度差で判断できるようになっています。
さらに、ラベルとグラフ線を「点線コネクタ」で繋ぐことで、データとラベルの一体感を高め、認知負荷を低減しています。

## 3. もう一つの軸：Resistance Score

「確率」だけでは表現できない達成感があります。それは **「積み上げ」** です。
確率（青信号）は、復習直後は誰でも100%になります。これでは、今日始めたばかりの初心者と、1年間学習を続けたベテランの区別がつきません。

そこで導入したのが **Resistance Score (R)** です。

* **概念**: 「記憶の強靭さ」。風雪にどれだけ耐えられるか。
* **計算**: 復習間隔（Interval）の対数をとる。
* **目的**: ゲーミフィケーション。「レベル」のように、長く続ければ続けるほど育っていく数値。

### 分離の決断

カレンダー画面のリファクタリング（202x-01）において、私たちは当初混同されがちだったこれら2つを明確に分離しました。

* **カレンダーの色**: **Resistance** (積み上げ) を反映。
  * 理由: カレンダーは「過去の努力の可視化」であるため、積み上げ量で色付けするのが自然。
* **カードリストの色**: **Retention** (現状の確率) を反映。
  * 理由: リストは「これからのタスク」であるため、緊急度で色付けするのが自然。

この使い分けにより、ユーザーは **「過去を誇り（Resistance）、未来に備える（Retention）」** という健全なメンタルモデルを構築できます。
