この文書は本アプリケーション内で表示に用いられる「耐性スコア（Resistance Score）」に関する詳細仕様をまとめる。

目的:

- ユーザーに「どれだけ長い間隔を耐えているか」を直感的に示すための指標を定義する。
- `memoryStability`（内部定着度）とは別の、UI向け補助指標として運用する。


## 1. 概要

- 名前: 耐性スコア（Resistance Score、以下 R）
- 種別: 表示用メトリクス（副次的指標）。SRS の内部状態を変えない。
- 値域: 0〜100（%）
- 意味: 現在カードに設定されている復習間隔（日数）が「相対的にどれくらい長いか」を対数スケールで示す。

## 2. 計算式（実装参照: `src/utils/reviewMetrics.ts`）

- パラメータ:
  - `intervalDays` (I): 現在設定されている復習間隔（日数）。負値は 0 として扱う。
  - 定数: `MAX_INTERVAL_DAYS = 90`

- 手順:
  1. 正の領域へ調整: I = max(0, intervalDays)
  2. スコア化（対数比）:

     numerator = log10(1 + I)
     denominator = log10(1 + MAX_INTERVAL_DAYS)
     R = round( (numerator / denominator) * 100 )

  3. 上限処理: R = min(100, R)

- 解説:
  - 母関数は対数 (base10) を用い、短い間隔の差より長い間隔での差を視覚的に強調する。
  - I = 0 → R = 0
  - I = 1 → R ≈ (log10 2 / log10 91) * 100 ≈ 15
  - I = 7 → R ≈ 46
  - I = 30 → R ≈ 76
  - I = 90 → R = 100

## 3. 意図と設計判断

- なぜ対数スケールか:
  - 人間の感覚では「1日→2日」の増加より「30日→60日」の増加の方が重要性を感じやすいが、線形スケールでは後者が視認しづらい。
  - 対数スケールは「積み上がり感（長期間に耐えた）」を直感的に示すのに有効。

- `memoryStability`（S）との差分:
  - S は忘却曲線に基づく推定保持確率の元となる内部パラメータ（0.01–1.0）であり、復習の度に更新される。
  - R は S の代わりに UI の一部（カレンダー、カードビューの進捗バー、統計）で使われ、ユーザーに「どれだけ間隔を伸ばしてきたか」を示す。
  - R は S の可視化ではなく補助的な「成熟度」指標である。

## 4. UI での使い方

- 表示例（実装での利用箇所）:
  - カード詳細: `src/Pages/CardView.jsx` — プログレスバー幅に R% を用いる。
  - カレンダーのドット色: `src/Pages/Calendar.jsx` — R により色を分岐（例: R >= 70 → 緑/長期保持、R >= 30 → 青/安定、R < 30 → オレンジ〜赤/形成中）。
  - 統計パネル: R の分布を集計して「積み上がり」ビューを表示。

- 実装時の注意点:
  - R は UI 表示専用。復習アルゴリズム（次回日付や stability 更新）には影響しない。
  - 表示は round された整数値を用いる（コードの実装に準拠）。

## 5. 閾値例（推奨）

- 表示目的に応じたガイドライン:
  - R < 15: ほぼ新規、耐性が低い（赤系）
  - 15 ≤ R < 30: 形成中（オレンジ）
  - 30 ≤ R < 70: 安定（青系）
  - R ≥ 70: 長期保持（緑系）

これらはプロダクト内の現行カラー基準と合わせて調整すること。

## 6. 実例（計算サンプル）

- 関数呼び出し: `calculateResistanceScore(interval)`（参照実装）

- 例:
  - `I = 0` → numerator = log10(1) = 0 → R = 0
  - `I = 1` → numerator = log10(2) ≈ 0.3010、denominator = log10(91) ≈ 1.9590 → R ≈ 15
  - `I = 7` → numerator = log10(8) ≈ 0.9031 → R ≈ 46
  - `I = 30` → numerator = log10(31) ≈ 1.4914 → R ≈ 76
  - `I = 90` → numerator = log10(91) ≈ 1.9590 → R = 100

## 7. テスト/検証案

- 単体テスト（`src/tests/reviewMetrics.test.ts` 既存）に次を追加する:
  1. 境界テスト
     - `I = 0` → R === 0
     - `I = MAX_INTERVAL_DAYS` → R === 100
  2. 単調性テスト
     - `I1 < I2` のとき R(I1) ≤ R(I2)
  3. 代表値一致テスト（近似）
     - `I = 1` → R in [14,16]
     - `I = 7` → R in [44,48]
     - `I = 30` → R in [74,78]

- テスト実装例（擬似）:

  ```ts
  import { calculateResistanceScore } from '@/utils/reviewMetrics';

  test('Resistance monotonicity', () => {
    expect(calculateResistanceScore(1)).toBeLessThanOrEqual(calculateResistanceScore(10));
    expect(calculateResistanceScore(10)).toBeLessThanOrEqual(calculateResistanceScore(90));
  });
  ```

## 9. 改善案 / 将来検討

- スケールの見直し:
  - 現行は log10 を採用。利用データに基づいて基数（対数の底）や MAX_INTERVAL_DAYS を動的に調整する検討。

- 個別化:
  - ユーザーごとに「長い間隔を好む」か「短い間隔で学ぶ」かの嗜好がある場合、R の閾値や表示傾向をユーザー設定で調整可能にする。

- 表示のアクセシビリティ:
  - カラーだけで情報を伝達しない（パーセント表示やツールチップの説明を追加）

## 10. まとめ

- `Resistance Score` は復習アルゴリズムそのものではなく、ユーザー向け可視化のための補助指標である。
- 現行は対数スケール（log10(1+I) / log10(1+MAX)）により 0–100 のレンジで算出され、UI の色分けやバー表示に利用されている。
